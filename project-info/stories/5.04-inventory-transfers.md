# TR-23: Inventory - Transfer Orders

**Epic:** Inventory Management System
**Component:** Inventory Transfers Between Locations
**Version:** 1.0
**Last Updated:** October 29, 2025

---

## 1. Overview

This story covers all workflows for moving inventory between locations (central warehouse and technician containers). Transfers are the primary mechanism for distributing inventory to technicians, returning unused stock, and rebalancing inventory across locations.

**Key Objectives:**
- Support multi-directional transfers (warehouse ↔ container, container → warehouse)
- Barcode-driven transfer confirmation to ensure accuracy
- Two-step process: create transfer → confirm receipt
- Real-time inventory updates only after confirmation
- Complete audit trail of all movements
- Mobile-first experience for technicians
- Prevent inventory discrepancies

## 2. Transfer Types & Directions

### 2.1. Supported Transfer Flows

```
┌─────────────────────────────────────────────────────┐
│              CENTRAL WAREHOUSE                      │
│           (Main Stock Location)                     │
└──────────┬────────────────────────────┬─────────────┘
           │                            │
           │ Type A                     │ Type B
           │ Stock Out                  │ Return
           │ (Warehouse → Tech)         │ (Tech → Warehouse)
           ↓                            ↑
    ┌──────────────┐            ┌──────────────┐
    │  Container   │            │  Container   │
    │   Tech #1    │            │   Tech #2    │
    └──────────────┘            └──────────────┘
           │                            ↑
           │ Type C (Direct)            │
           │ Tech-to-Tech               │
           │ (With approval)            │
           └────────────────────────────┘
```

**Transfer Type A: Warehouse → Technician Container**
- **Purpose:** Stock technician van with needed inventory
- **Initiated by:** Logistics, Manager, Dispatcher, or Technician (request)
- **Approval:** Not required if initiated by Logistics
- **Common scenarios:**
  - New technician starter pack
  - Restock commonly used items
  - Specific items needed for scheduled job
  - Emergency job requires parts not in tech's van

**Transfer Type B: Technician Container → Warehouse**
- **Purpose:** Return unused or excess inventory
- **Initiated by:** Technician, Logistics, Manager
- **Approval:** Not required
- **Common scenarios:**
  - End of season (return winter-specific items)
  - Technician leaving company (return all inventory)
  - Excess stock in van (rebalance to warehouse)
  - Wrong items were transferred

**Transfer Type C: Technician → Technician (Direct)**
- **Purpose:** Move inventory directly between technicians
- **Initiated by:** Technician, Logistics, or Dispatcher
- **Approval:** Required if initiated by Technician or Dispatcher (approved by Logistics or Manager)
- **Delivery method:** Usually via mail/shipping
- **Process:**
  1. Create transfer request (if Tech/Dispatcher initiates)
  2. Logistics/Manager approves (if required)
  3. Transfer becomes PENDING
  4. Tech A ships items
  5. Tech B receives and scans items
  6. Transfer completes, inventory moves from Container A to Container B
- **Why direct?** More efficient, no need for warehouse middleman when both parties agree

## 3. Transfer Workflow Overview

### 3.1. Two-Step Process

All transfers follow a two-step confirmation process:

```
Step 1: CREATE TRANSFER
  Who: Sender (Logistics/Technician)
  Action: Select items, quantities, destination
  Result: Transfer order created
  Status: PENDING
  Inventory: No change yet (reserved at source)

Step 2: CONFIRM RECEIPT
  Who: Receiver (Technician/Logistics)
  Action: Scan items to confirm receipt
  Result: Transfer completed
  Status: COMPLETED
  Inventory: Deducted from source, added to destination
```

**Why two-step?**
- ✅ Prevents inventory loss (what if package lost in transit?)
- ✅ Confirms physical receipt before inventory adjustment
- ✅ Allows for discrepancy detection (missing items, damage)
- ✅ Creates accountability (both parties involved)
- ✅ Matches real-world logistics (ship → receive)

**Validation:**

```
System checks before creating transfer:
1. ✅ Source has sufficient quantity?

2. ✅ Destination can receive?
   - Container active?
   - Technician account active?

3. ✅ Items valid for transfer?
   - All items exist in catalog
   - No negative quantities
```

## 7. Transfer States & Status

### 7.1. Transfer Lifecycle

**Standard Transfers (Warehouse ↔ Tech):**
```
DRAFT
  ↓ [Create transfer]
PENDING
  ↓ [Receiver starts scanning]
RECEIVING (in progress)
  ↓ [Receiver completes scan]
COMPLETED
  OR
  ↓ [Issues detected]
COMPLETED_WITH_ISSUES
  OR
  ↓ [Cancelled before receipt]
CANCELLED
```

**Tech-to-Tech Transfers (with shipping):**
```
DRAFT
  ↓ [Create transfer or approve request]
PENDING
  ↓ [Sender marks as shipped]
SHIPPED
  ↓ [Receiver starts scanning]
RECEIVING (in progress)
  ↓ [Receiver completes scan]
COMPLETED
  OR
  ↓ [Issues detected]
COMPLETED_WITH_ISSUES
  OR
  ↓ [Cancelled before shipment]
CANCELLED
```

**Status Definitions:**

| Status | Description | Inventory Impact | Actions Available |
|--------|-------------|------------------|-------------------|
| **DRAFT** | Transfer being created | None | Edit, Cancel |
| **PENDING** | Awaiting pickup/shipment | Reserved at source | Confirm/Ship, Cancel, Edit |
| **SHIPPED** | Items shipped (tech-to-tech only) | Reserved at source | Track, Receive |
| **RECEIVING** | Receiver scanning items | Reserved at source | Continue, Pause |
| **COMPLETED** | All items confirmed | Moved source → dest | View only |
| **COMPLETED_WITH_ISSUES** | Completed with discrepancies | Partial move | View, Report |
| **CANCELLED** | Transfer cancelled | Reservation released | View only |

### 9.2. Transfer API

```php
// File: /local/api/inventory/transfers.php

namespace Local\Inventory;

class TransferManager {

    /**
     * Create transfer order
     * POST /api/inventory/transfers/create
     */
    public static function create($data) {
        global $DB;
        $userId = $GLOBALS['USER']->GetID();

        $transferNumber = self::generateTransferNumber($data['type']);

        $DB->Query("
            INSERT INTO b_inventory_transfer
            (TRANSFER_NUMBER, TRANSFER_TYPE, SOURCE_STORE_ID, DEST_STORE_ID,
             STATUS, REASON, EXPECTED_DATE, DELIVERY_METHOD, NOTES, CREATED_BY, DATE_CREATE)
            VALUES
            ('{$transferNumber}', '{$data['type']}', {$data['source_store_id']},
             {$data['dest_store_id']}, 'pending', '{$DB->ForSql($data['reason'])}',
             '{$data['expected_date']}', '{$data['delivery_method']}',
             '{$DB->ForSql($data['notes'])}', {$userId}, NOW())
        ");

        $transferId = $DB->LastID();

        // Add items
        foreach ($data['items'] as $item) {
            self::addItem($transferId, $item);
        }

        // Reserve inventory at source
        self::reserveInventory($transferId);

        // Send notifications
        self::sendTransferNotification($transferId, 'created');

        return [
            'transfer_id' => $transferId,
            'transfer_number' => $transferNumber
        ];
    }

    /**
     * Add item to transfer
     */
    public static function addItem($transferId, $itemData) {
        global $DB;

        // Get product cost
        $product = \CIBlockElement::GetByID($itemData['product_id'])->Fetch();
        $unitCost = $product['UF_COST_TECH'];
        $totalCost = $unitCost * $itemData['quantity'];

        // Check availability at source
        $transfer = $DB->Query("SELECT SOURCE_STORE_ID FROM b_inventory_transfer WHERE ID = {$transferId}")->Fetch();
        $available = self::getAvailableQuantity($transfer['SOURCE_STORE_ID'], $itemData['product_id']);

        if ($available < $itemData['quantity']) {
            throw new \Exception("Insufficient stock. Available: {$available}, Requested: {$itemData['quantity']}");
        }

        $DB->Query("
            INSERT INTO b_inventory_transfer_item
            (TRANSFER_ID, PRODUCT_ID, QUANTITY_EXPECTED, UNIT_COST, TOTAL_COST)
            VALUES
            ({$transferId}, {$itemData['product_id']}, {$itemData['quantity']}, {$unitCost}, {$totalCost})
        ");

        // Update transfer totals
        self::updateTransferTotals($transferId);

        return $DB->LastID();
    }

    /**
     * Scan item during receiving
     * POST /api/inventory/transfers/{id}/scan
     */
    public static function scanItem($transferId, $barcode) {
        global $DB;

        // Find product by barcode
        $product = self::findProductByBarcode($barcode);
        if (!$product) {
            throw new \Exception("Product not found for barcode: {$barcode}");
        }

        // Check if product in transfer
        $transferItem = $DB->Query("
            SELECT * FROM b_inventory_transfer_item
            WHERE TRANSFER_ID = {$transferId} AND PRODUCT_ID = {$product['ID']}
        ")->Fetch();

        if (!$transferItem) {
            throw new \Exception("Product not in transfer order");
        }

        // Check if already scanned all
        if ($transferItem['QUANTITY_SCANNED'] >= $transferItem['QUANTITY_EXPECTED']) {
            return [
                'status' => 'warning',
                'message' => 'Already scanned all expected quantity',
                'scanned' => $transferItem['QUANTITY_SCANNED'],
                'expected' => $transferItem['QUANTITY_EXPECTED']
            ];
        }

        // Increment scanned count
        $newScanned = $transferItem['QUANTITY_SCANNED'] + 1;
        $DB->Query("
            UPDATE b_inventory_transfer_item
            SET QUANTITY_SCANNED = {$newScanned},
                IS_SCANNED = TRUE,
                SCANNED_AT = NOW()
            WHERE ID = {$transferItem['ID']}
        ");

        // Update transfer scanned items count
        $DB->Query("
            UPDATE b_inventory_transfer
            SET SCANNED_ITEMS = SCANNED_ITEMS + 1,
                STATUS = 'receiving'
            WHERE ID = {$transferId}
        ");

        return [
            'status' => 'success',
            'item_id' => $transferItem['ID'],
            'product_name' => $product['NAME'],
            'scanned' => $newScanned,
            'expected' => $transferItem['QUANTITY_EXPECTED'],
            'progress' => self::getTransferProgress($transferId)
        ];
    }

    /**
     * Complete transfer (move inventory)
     * POST /api/inventory/transfers/{id}/complete
     */
    public static function complete($transferId, $data = []) {
        global $DB;

        // Get transfer details
        $transfer = $DB->Query("SELECT * FROM b_inventory_transfer WHERE ID = {$transferId}")->Fetch();

        if ($transfer['STATUS'] === 'completed') {
            throw new \Exception("Transfer already completed");
        }

        // Get all items
        $items = $DB->Query("
            SELECT * FROM b_inventory_transfer_item WHERE TRANSFER_ID = {$transferId}
        ")->fetchAll();

        $hasDiscrepancies = false;

        foreach ($items as $item) {
            $quantityToTransfer = $item['QUANTITY_SCANNED'] ?: $item['QUANTITY_EXPECTED'];

            // Check for discrepancies
            if ($quantityToTransfer != $item['QUANTITY_EXPECTED']) {
                $hasDiscrepancies = true;
            }

            // Move inventory
            self::moveInventory(
                $transfer['SOURCE_STORE_ID'],
                $transfer['DEST_STORE_ID'],
                $item['PRODUCT_ID'],
                $quantityToTransfer
            );
        }

        // Update transfer status
        $status = $hasDiscrepancies ? 'completed_with_issues' : 'completed';
        $DB->Query("
            UPDATE b_inventory_transfer
            SET STATUS = '{$status}',
                HAS_DISCREPANCIES = " . ($hasDiscrepancies ? 'TRUE' : 'FALSE') . ",
                CONFIRMED_BY = " . $GLOBALS['USER']->GetID() . ",
                DATE_CONFIRMED = NOW(),
                LOCATION_CONFIRMED = '{$DB->ForSql($data['location'])}'
            WHERE ID = {$transferId}
        ");

        // Release reservations
        self::releaseReservations($transferId);

        // Log movement
        self::logTransferMovement($transferId);

        // Send notifications
        self::sendTransferNotification($transferId, 'completed');

        // Generate PDF
        $pdfUrl = self::generateTransferPDF($transferId);

        return [
            'transfer_id' => $transferId,
            'transfer_number' => $transfer['TRANSFER_NUMBER'],
            'status' => $status,
            'has_discrepancies' => $hasDiscrepancies,
            'pdf_url' => $pdfUrl
        ];
    }

    /**
     * Helper: Move inventory between stores
     */
    private static function moveInventory($sourceStoreId, $destStoreId, $productId, $quantity) {
        // Deduct from source
        $sourceStock = \Bitrix\Catalog\StoreProductTable::getList([
            'filter' => ['STORE_ID' => $sourceStoreId, 'PRODUCT_ID' => $productId],
            'select' => ['ID', 'AMOUNT']
        ])->fetch();

        if ($sourceStock) {
            \Bitrix\Catalog\StoreProductTable::update($sourceStock['ID'], [
                'AMOUNT' => $sourceStock['AMOUNT'] - $quantity
            ]);
        }

        // Add to destination
        $destStock = \Bitrix\Catalog\StoreProductTable::getList([
            'filter' => ['STORE_ID' => $destStoreId, 'PRODUCT_ID' => $productId],
            'select' => ['ID', 'AMOUNT']
        ])->fetch();

        if ($destStock) {
            \Bitrix\Catalog\StoreProductTable::update($destStock['ID'], [
                'AMOUNT' => $destStock['AMOUNT'] + $quantity
            ]);
        } else {
            \Bitrix\Catalog\StoreProductTable::add([
                'STORE_ID' => $destStoreId,
                'PRODUCT_ID' => $productId,
                'AMOUNT' => $quantity
            ]);
        }
    }

     * Helper: Get available quantity (on hand - reserved)

    private static function getAvailableQuantity($storeId, $productId) {

    /**
     * Create tech-to-tech transfer request (requires approval)
     * POST /api/inventory/transfers/request-tech-to-tech
     */
    public static function createTechToTechRequest($data) {
        global $DB;
        $userId = $GLOBALS['USER']->GetID();

        // Check if user has permission
        $userRole = self::getUserRole($userId);
        $requiresApproval = !in_array($userRole, ['logistics', 'manager', 'owner']);

        if ($requiresApproval) {
            // Create as request (pending approval)
            $requestNumber = self::generateRequestNumber();

            $DB->Query("
                INSERT INTO b_inventory_transfer_request
                (REQUEST_NUMBER, TECHNICIAN_USER_ID, CONTAINER_ID, URGENCY, STATUS, REASON, REQUESTED_BY, DATE_CREATE)
                VALUES
                ('{$requestNumber}', {$data['from_tech_id']}, {$data['source_store_id']},
                 '{$data['urgency']}', 'pending', '{$DB->ForSql($data['reason'])}', {$userId}, NOW())
            ");

            $requestId = $DB->LastID();

            // Add items to request
            foreach ($data['items'] as $item) {
                $DB->Query("
                    INSERT INTO b_inventory_transfer_request_item
                    (REQUEST_ID, PRODUCT_ID, QUANTITY)
                    VALUES ({$requestId}, {$item['product_id']}, {$item['quantity']})
                ");
            }

            // Notify logistics/manager
            self::sendNotification('tech_to_tech_request_pending', $requestId);

            return [
                'request_id' => $requestId,
                'request_number' => $requestNumber,
                'status' => 'pending_approval'
            ];

        } else {
            // Logistics can create directly (no approval)
            $data['type'] = 'tech_to_tech';
            $data['requires_approval'] = false;
            return self::create($data);
        }
    }

    /**
     * Approve tech-to-tech transfer request
     * POST /api/inventory/transfers/requests/{id}/approve
     */
    public static function approveRequest($requestId, $data = []) {
        global $DB;
        $userId = $GLOBALS['USER']->GetID();

        // Get request
        $request = $DB->Query("SELECT * FROM b_inventory_transfer_request WHERE ID = {$requestId}")->Fetch();

        if (!$request || $request['STATUS'] !== 'pending') {
            throw new \Exception("Request not found or already processed");
        }

        // Get request items
        $items = $DB->Query("
            SELECT * FROM b_inventory_transfer_request_item WHERE REQUEST_ID = {$requestId}
        ")->fetchAll();

        // Get source and dest stores
        $sourceStore = $request['CONTAINER_ID'];
        $destStore = $data['dest_store_id'];

        // Create transfer order
        $transferData = [
            'type' => 'tech_to_tech',
            'source_store_id' => $sourceStore,
            'dest_store_id' => $destStore,
            'reason' => $request['REASON'],
            'delivery_method' => $data['delivery_method'] ?? 'mail',
            'items' => $items
        ];

        $transferResult = self::create($transferData);

        // Update request status
        $DB->Query("
            UPDATE b_inventory_transfer_request
            SET STATUS = 'approved',
                REVIEWED_BY = {$userId},
                DATE_REVIEWED = NOW(),
                TRANSFER_ID = {$transferResult['transfer_id']},
                REVIEW_NOTES = '{$DB->ForSql($data['notes'] ?? '')}'
            WHERE ID = {$requestId}
        ");

        // Notify requester and both techs
        self::sendNotification('tech_to_tech_request_approved', $requestId, $transferResult['transfer_id']);

        return [
            'request_id' => $requestId,
            'transfer_id' => $transferResult['transfer_id'],
            'transfer_number' => $transferResult['transfer_number'],
            'status' => 'approved'
        ];
    }

    /**
     * Helper: Generate transfer number
     */
    private static function generateTransferNumber($type) {
        $year = date('Y');
        $prefix = $type === 'return' ? 'TR-RTN' : 'TR';
        $nextNum = self::getNextTransferNumber($type);
        return sprintf('%s-%s-%06d', $prefix, $year, $nextNum);
    }

    /**
     * Helper: Generate request number
     */
    private static function generateRequestNumber() {
        $year = date('Y');
        $nextNum = self::getNextRequestNumber();
        return sprintf('TRQ-%s-%06d', $year, $nextNum);
    }
```
