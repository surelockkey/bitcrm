# TR-21: Inventory - Multi-Warehouse Configuration

**Epic:** Inventory Management System
**Component:** Warehouse Structure & Stock Locations
**Version:** 1.0
**Last Updated:** October 29, 2025

---

## 1. Overview

This story covers the setup and management of the multi-warehouse system in Bitrix24, which enables tracking inventory across multiple physical and virtual locations. The system supports one central warehouse and unlimited technician containers (virtual warehouses representing van inventory).

**Key Objectives:**
- Configure central warehouse for bulk inventory storage
- Create technician containers (one per technician) for van inventory
- Enable real-time stock visibility across all locations
- Support seamless transfers between locations
- Scale easily as new technicians are added

### 2.2. Location Types

**Central Warehouse (Type: Physical)**
- **Purpose:** Main inventory storage
- **Location:** Company office/warehouse (physical address)
- **Managed by:** Logistics Manager
- **Access:** Management, Logistics, Dispatchers (view only)
- **Capacity:** 2000+ unique SKUs
- **Receiving:** Bulk shipments from UHS, Amazon, AliExpress, etc.
- **Outgoing:** Transfers to technician containers

**Technician Container (Type: Virtual)**
- **Purpose:** Track inventory in technician's van
- **Location:** Virtual (follows technician)
- **Managed by:** Individual Technician (mobile app)
- **Access:** Technician (full access), Management (view/transfer)
- **Capacity:** 50-150 typical SKUs (varies by technician)
- **Receiving:** Transfers from central warehouse OR direct supplier shipments
- **Outgoing:** Used in jobs (automatic deduction when added to Deal)

**No Limit on Number of Containers:**
- System supports unlimited technician containers
- Each container is independent
- Performance optimized for 10-50 active technicians
- Typical inventory per company:
  - **Total unique SKUs:** 2000-3000 across entire company
  - **Per technician:** 50-150 SKUs (many shared between technicians)
  - **Performance:** System tested with 3000 SKUs × 50 containers = 150,000 potential stock records
  - **Reality:** Most SKUs appear in multiple containers (e.g., common keys, locks)

### 3.2. Warehouse Permissions

**Permission Definitions:**
- **View Stock:** Can see what inventory is in the warehouse and quantities
- **Receive Stock:** Can add new inventory when shipments arrive from suppliers
- **Transfer Out:** Can create transfer orders to move inventory from warehouse to technician containers
- **Edit Details:** Can modify warehouse settings (address, operating hours, etc.)
- **Stocktake:** Can perform physical inventory count and reconcile with system

**Central Warehouse Permissions:**

| Role | View Stock | Receive Stock | Transfer Out | Edit Details | Stocktake |
|------|------------|---------------|--------------|--------------|-----------|
| **Owner** | ✅ Yes | ✅ Yes | ✅ Yes | ✅ Yes | ✅ Yes |
| **Manager** | ✅ Yes | ✅ Yes | ✅ Yes | ✅ Yes | ✅ Yes |
| **Logistics** | ✅ Yes | ✅ Yes | ✅ Yes | ✅ Yes | ✅ Yes |
| **Dispatcher** | ✅ Yes | ❌ No | ✅ Yes | ❌ No | ❌ No |
| **Technician** | ❌ No* | ✅ Yes** | ✅ Yes** | ❌ No | ✅ Yes** |

**\*Technician Central Warehouse Visibility:**
- **Cannot** see full central warehouse inventory list
- **Can** see if specific item is "Available at warehouse" when viewing product details
- **Can** request transfers via mobile app
- This prevents confusion - technicians focus on their own container inventory

**\*\*Technician Warehouse Operations:**
- **Receive Stock:** Can receive direct deliveries to warehouse (when package arrives there instead of to technician)
- **Transfer Out:** Can create transfer requests from warehouse to their own container
- **Stocktake:** Can participate in warehouse physical counts if asked by management

**Why these permissions:**
- **All roles** can receive stock (everyone might receive packages)
- **All roles** can create transfers (including technicians requesting stock for their container)
- **Dispatchers** don't perform physical stocktakes (not their responsibility)
- **Technicians** can help with warehouse inventory counts when needed

### 4.2. Automatic Container Creation

**Trigger:** New user created with role "Technician"

**Implementation:**

```php
// File: /local/php_interface/include/user_events.php

use Bitrix\Main\EventManager;
use Bitrix\Catalog\StoreTable;

EventManager::getInstance()->addEventHandler(
    'main',
    'OnAfterUserAdd',
    'createTechnicianContainer'
);

EventManager::getInstance()->addEventHandler(
    'main',
    'OnAfterUserUpdate',
    'onUserRoleChange'
);

function createTechnicianContainer($arFields) {
    $userId = $arFields['ID'];

    // Check if user has Technician role
    $userGroups = CUser::GetUserGroup($userId);
    $isTechnician = in_array(TECHNICIAN_GROUP_ID, $userGroups);

    if (!$isTechnician) {
        return;
    }

    // Get user details
    $user = CUser::GetByID($userId)->Fetch();
    $fullName = trim($user['NAME'] . ' ' . $user['LAST_NAME']);

    // Get next container number
    $nextNumber = getNextTechnicianNumber();

    // Create container
    $containerCode = sprintf('TECH_%03d', $nextNumber);
    $containerTitle = sprintf('Tech #%d - %s', $nextNumber, $fullName);

    $storeId = StoreTable::add([
        'TITLE' => $containerTitle,
        'CODE' => $containerCode,
        'ACTIVE' => 'Y',
        'ADDRESS' => 'Mobile - Technician Van',
        'DESCRIPTION' => 'Virtual warehouse for technician van inventory',
        'USER_ID' => $userId,  // Link to technician
        'ISSUING_CENTER' => 'N',
        'SHIPPING_CENTER' => 'N',
        'SORT' => 100 + $nextNumber,
        'UF_STORE_TYPE' => 'container',  // Custom field
        'UF_TECH_NUMBER' => $nextNumber,  // Custom field
    ])->getId();

    // Send notification to technician
    sendTechnicianWelcomeEmail($userId, $containerTitle);

    // Log creation
    writeToLog("Container created: {$containerTitle} (ID: {$storeId}) for User ID: {$userId}");

    return $storeId;
}

function getNextTechnicianNumber() {
    $result = StoreTable::getList([
        'filter' => ['UF_STORE_TYPE' => 'container'],
        'select' => ['UF_TECH_NUMBER'],
        'order' => ['UF_TECH_NUMBER' => 'DESC'],
        'limit' => 1
    ])->fetch();

    return $result ? ($result['UF_TECH_NUMBER'] + 1) : 1;
}

function onUserRoleChange($arFields) {
    // Handle role changes (e.g., technician promoted to dispatcher)
    // May need to handle container deactivation or transfer
    $userId = $arFields['ID'];

    $wasTechnician = userWasTechnician($userId);
    $isTechnician = userIsTechnician($userId);

    if ($wasTechnician && !$isTechnician) {
        // Technician left or role changed
        // Deactivate container (don't delete - preserve history)
        deactivateTechnicianContainer($userId);
    } elseif (!$wasTechnician && $isTechnician) {
        // User became technician
        createTechnicianContainer(['ID' => $userId]);
    }
}

function deactivateTechnicianContainer($userId) {
    $container = StoreTable::getList([
        'filter' => [
            'USER_ID' => $userId,
            'UF_STORE_TYPE' => 'container',
            'ACTIVE' => 'Y'
        ]
    ])->fetch();

    if ($container) {
        // Check if container has inventory
        $hasStock = hasInventoryInStore($container['ID']);

        if ($hasStock) {
            // Alert management: container has stock, needs transfer
            sendAlert("Container {$container['TITLE']} has inventory. Transfer required before deactivation.");
        } else {
            // Safe to deactivate
            StoreTable::update($container['ID'], [
                'ACTIVE' => 'N',
                'TITLE' => $container['TITLE'] . ' (INACTIVE)'
            ]);
        }
    }
}
```

**Technician Container Permissions:**

| Role | View Stock | Receive Stock | Transfer Out | Use in Jobs | Stocktake | Request Transfer |
|------|------------|---------------|--------------|-------------|-----------|------------------|
| **Container Owner (Technician)** | ✅ Own only | ✅ Yes | ✅ Own only* | ✅ Yes | ✅ Own only | ✅ Yes |
| **Owner** | ✅ All | ✅ Yes | ✅ Yes | ✅ Yes** | ✅ All | ✅ Yes |
| **Manager** | ✅ All | ✅ Yes | ✅ Yes | ✅ Yes** | ✅ All | ✅ Yes |
| **Logistics** | ✅ All | ✅ Yes | ✅ Yes | ✅ Yes** | ✅ All | ✅ Yes |
| **Dispatcher** | ✅ All (read) | ❌ No | ✅ Yes | ✅ Yes** | ❌ No | ✅ Yes |

**\*\*Use in Jobs Permission:**
All roles can add items from technician containers to jobs/deals. This allows:
- **Technicians:** Add their own inventory to jobs
- **Management/Logistics/Dispatchers:** Add items on behalf of technicians when creating or modifying jobs remotely

**\*Technician Transfer Permissions:**
- **Can** create transfer requests FROM warehouse TO their container
- **Can** return items FROM their container TO warehouse
- **Cannot** directly transfer TO other technicians (must go through warehouse)
- This prevents unauthorized "borrowing" between technicians and maintains proper tracking

**Detailed Permissions Explained:**

**Container Owner (Technician):**
- ✅ View all inventory in **their own** container only
- ✅ Receive stock (confirm transfers from warehouse or direct deliveries)
- ✅ Create transfers FROM warehouse TO their container
- ✅ Return items FROM their container TO warehouse
- ✅ Use stock in jobs (scan barcode, add to deal → automatic deduction)
- ✅ Perform stocktakes on their own container
- ✅ View movement history for their container
- ❌ Cannot see other technicians' containers
- ❌ Cannot transfer directly TO other technicians (must go through warehouse)
- ❌ Cannot manually adjust quantities (except via stocktake)

**Management/Logistics:**
- ✅ View **all** containers across all technicians
- ✅ Receive stock into any container
- ✅ Create transfer orders to/from any container (including tech-to-tech via warehouse)
- ✅ Add items from any container to jobs/deals (when managing jobs remotely)
- ✅ Perform stocktakes on any container (audit capability)
- ✅ View all movement history
- ✅ Generate reports across all containers

**Dispatchers:**
- ✅ View all containers (read-only) for job assignment
- ✅ Check technician inventory before assigning jobs
- ✅ Add items from technician containers to jobs (when creating/modifying jobs)
- ✅ Create transfers for technicians (request stock from warehouse)
- ❌ Cannot receive stock (not their responsibility)
- ❌ Cannot perform stocktakes
- ❌ Cannot manually adjust inventory

CREATE TABLE IF NOT EXISTS b_inventory_starter_pack (
  ID INT PRIMARY KEY AUTO_INCREMENT,
  NAME VARCHAR(255) NOT NULL,
  CODE VARCHAR(50) NOT NULL UNIQUE,
  DESCRIPTION TEXT,
  SPECIALIZATION VARCHAR(100),  -- Lockout, KeyProgramming, Residential, Commercial
  ACTIVE ENUM('Y', 'N') DEFAULT 'Y',
  TOTAL_COST DECIMAL(10,2) NOT NULL,  -- Total cost (for tech)
  TOTAL_ITEMS INT NOT NULL,  -- Total quantity of all items
  CREATED_BY INT NOT NULL,
  DATE_CREATE DATETIME NOT NULL,
  DATE_UPDATE DATETIME,
  INDEX idx_specialization (SPECIALIZATION),
  INDEX idx_active (ACTIVE)
);

-- Items in each starter pack
CREATE TABLE IF NOT EXISTS b_inventory_starter_pack_item (
  ID INT PRIMARY KEY AUTO_INCREMENT,
  PACK_ID INT NOT NULL,
  PRODUCT_ID INT NOT NULL,
  QUANTITY DECIMAL(18,4) NOT NULL,
  SORT INT DEFAULT 500,
  FOREIGN KEY (PACK_ID) REFERENCES b_inventory_starter_pack(ID) ON DELETE CASCADE,
  FOREIGN KEY (PRODUCT_ID) REFERENCES b_iblock_element(ID) ON DELETE CASCADE,
  INDEX idx_pack (PACK_ID)
);

-- Track which packs were applied to which technicians
CREATE TABLE IF NOT EXISTS b_inventory_starter_pack_applied (
  ID INT PRIMARY KEY AUTO_INCREMENT,
  PACK_ID INT NOT NULL,
  CONTAINER_ID INT NOT NULL,
  USER_ID INT NOT NULL,  -- Technician
  TRANSFER_ID INT NULL,  -- Reference to transfer order that delivered the pack
  DATE_APPLIED DATETIME NOT NULL,
  APPLIED_BY INT NOT NULL,  -- Who triggered the application
  STATUS ENUM('pending', 'delivered', 'confirmed') DEFAULT 'pending',
  FOREIGN KEY (PACK_ID) REFERENCES b_inventory_starter_pack(ID),
  FOREIGN KEY (CONTAINER_ID) REFERENCES b_catalog_store(ID),
  INDEX idx_user (USER_ID),
  INDEX idx_container (CONTAINER_ID)

// File: /local/api/inventory/starter_packs.php

namespace Local\Inventory;

class StarterPackManager {

    /**
     * Create new starter pack
     */
    public static function createPack($data) {
        global $DB;

        $name = $DB->ForSql($data['name']);
        $code = $DB->ForSql($data['code']);
        $description = $DB->ForSql($data['description']);
        $specialization = $DB->ForSql($data['specialization']);

        // Calculate totals
        $totalCost = 0;
        $totalItems = 0;

        foreach ($data['items'] as $item) {
            $product = CIBlockElement::GetByID($item['product_id'])->Fetch();
            $totalCost += $product['UF_COST_TECH'] * $item['quantity'];
            $totalItems += $item['quantity'];
        }

        // Insert pack
        $DB->Query("
            INSERT INTO b_inventory_starter_pack
            (NAME, CODE, DESCRIPTION, SPECIALIZATION, TOTAL_COST, TOTAL_ITEMS, CREATED_BY, DATE_CREATE)
            VALUES
            ('{$name}', '{$code}', '{$description}', '{$specialization}', {$totalCost}, {$totalItems}, " . $GLOBALS['USER']->GetID() . ", NOW())
        ");

        $packId = $DB->LastID();

        // Insert items
        foreach ($data['items'] as $item) {
            $DB->Query("
                INSERT INTO b_inventory_starter_pack_item
                (PACK_ID, PRODUCT_ID, QUANTITY, SORT)
                VALUES
                ({$packId}, {$item['product_id']}, {$item['quantity']}, {$item['sort']})
            ");
        }

        return $packId;
    }

    /**
     * Apply starter pack to technician container
     */
    public static function applyPackToTechnician($packId, $containerId, $sourceWarehouseId = null) {
        global $DB;

        // Get pack details
        $pack = $DB->Query("
            SELECT * FROM b_inventory_starter_pack WHERE ID = {$packId}
        ")->Fetch();

        if (!$pack) {
            throw new \Exception("Starter pack not found");
        }

        // Get container details
        $container = StoreTable::getById($containerId)->fetch();
        if (!$container) {
            throw new \Exception("Container not found");
        }

        $userId = $container['USER_ID'];

        // Get pack items
        $items = $DB->Query("
            SELECT * FROM b_inventory_starter_pack_item WHERE PACK_ID = {$packId}
        ")->fetchAll();

        // Use central warehouse as default source
        if (!$sourceWarehouseId) {
            $sourceWarehouseId = self::getCentralWarehouseId();
        }

        // Check if source has enough stock
        foreach ($items as $item) {
            $stock = StoreProductTable::getList([
                'filter' => [
                    'STORE_ID' => $sourceWarehouseId,
                    'PRODUCT_ID' => $item['PRODUCT_ID']
                ],
                'select' => ['AMOUNT']
            ])->fetch();

            if (!$stock || $stock['AMOUNT'] < $item['QUANTITY']) {
                $product = CIBlockElement::GetByID($item['PRODUCT_ID'])->Fetch();
                throw new \Exception("Insufficient stock for {$product['NAME']}. Need: {$item['QUANTITY']}, Have: " . ($stock['AMOUNT'] ?? 0));
            }
        }

        // Create transfer order
        $transferId = TransferOrderManager::create([
            'source_store_id' => $sourceWarehouseId,
            'dest_store_id' => $containerId,
            'notes' => "Starter Pack: {$pack['NAME']}",
            'created_by' => $GLOBALS['USER']->GetID()
        ]);

        // Add items to transfer
        foreach ($items as $item) {
            TransferOrderManager::addItem($transferId, [
                'product_id' => $item['PRODUCT_ID'],
                'quantity' => $item['QUANTITY']
            ]);
        }

        // Record pack application
        $DB->Query("
            INSERT INTO b_inventory_starter_pack_applied
            (PACK_ID, CONTAINER_ID, USER_ID, TRANSFER_ID, DATE_APPLIED, APPLIED_BY, STATUS)
            VALUES
            ({$packId}, {$containerId}, {$userId}, {$transferId}, NOW(), " . $GLOBALS['USER']->GetID() . ", 'pending')
        ");

        $appliedId = $DB->LastID();

        // Send notification to technician
        self::notifyTechnicianAboutStarterPack($userId, $pack, $transferId);

        return [
            'applied_id' => $appliedId,
            'transfer_id' => $transferId,
            'pack_name' => $pack['NAME'],
            'total_items' => $pack['TOTAL_ITEMS'],
            'total_cost' => $pack['TOTAL_COST']
        ];
    }

    /**
     * Get starter pack analytics
     */
    public static function getPackAnalytics($packId, $days = 90) {
        global $DB;

        // Get technicians who have this pack
        $technicians = $DB->Query("
            SELECT USER_ID, CONTAINER_ID, DATE_APPLIED
            FROM b_inventory_starter_pack_applied
            WHERE PACK_ID = {$packId}
              AND STATUS = 'confirmed'
              AND DATE_APPLIED > DATE_SUB(NOW(), INTERVAL {$days} DAY)
        ")->fetchAll();

        $packItems = $DB->Query("
            SELECT PRODUCT_ID, QUANTITY
            FROM b_inventory_starter_pack_item
            WHERE PACK_ID = {$packId}
        ")->fetchAll();

        $analytics = [
            'pack_id' => $packId,
            'technicians_count' => count($technicians),
            'date_range_days' => $days,
            'items' => []
        ];

        // Analyze usage of each item
        foreach ($packItems as $packItem) {
            $productId = $packItem['PRODUCT_ID'];
            $product = CIBlockElement::GetByID($productId)->Fetch();

            $usageCount = 0;
            $techsUsing = 0;
            $totalUsed = 0;
            $reorderFrequency = [];

            foreach ($technicians as $tech) {
                // Check how many times this product was used in deals
                $used = $DB->Query("
                    SELECT COUNT(*) as cnt, SUM(QUANTITY) as total
                    FROM b_deal_product_usage
                    WHERE USER_ID = {$tech['USER_ID']}
                      AND PRODUCT_ID = {$productId}
                      AND DATE_USED > '{$tech['DATE_APPLIED']}'
                ")->Fetch();

                if ($used && $used['cnt'] > 0) {
                    $techsUsing++;
                    $totalUsed += $used['total'];
                    $usageCount += $used['cnt'];
                }

                // Check reorder frequency
                $reorders = $DB->Query("
                    SELECT DATE(DATE_CREATE) as reorder_date
                    FROM b_inventory_transfer_item ti
                    JOIN b_inventory_transfer t ON t.ID = ti.TRANSFER_ID
                    WHERE t.DEST_STORE_ID = {$tech['CONTAINER_ID']}
                      AND ti.PRODUCT_ID = {$productId}
                      AND t.DATE_CREATE > '{$tech['DATE_APPLIED']}'
                    ORDER BY t.DATE_CREATE
                ")->fetchAll();

                if (count($reorders) > 0) {
                    // Calculate days between starter pack and first reorder
                    $firstReorder = strtotime($reorders[0]['reorder_date']);
                    $packApplied = strtotime($tech['DATE_APPLIED']);
                    $daysToReorder = ($firstReorder - $packApplied) / 86400;
                    $reorderFrequency[] = $daysToReorder;
                }
            }

            $usageRate = count($technicians) > 0 ? ($techsUsing / count($technicians)) * 100 : 0;
            $avgReorderDays = count($reorderFrequency) > 0 ? array_sum($reorderFrequency) / count($reorderFrequency) : null;

            $analytics['items'][] = [
                'sku' => $product['CODE'],
                'name' => $product['NAME'],
                'pack_quantity' => $packItem['QUANTITY'],
                'usage_rate' => round($usageRate, 1),
                'techs_using' => $techsUsing,
                'total_times_used' => $usageCount,
                'total_quantity_used' => $totalUsed,
                'avg_reorder_days' => $avgReorderDays ? round($avgReorderDays) : null,
                'recommendation' => self::generateRecommendation($usageRate, $avgReorderDays, $packItem['QUANTITY'])
            ];
        }

        // Sort by usage rate
        usort($analytics['items'], fn($a, $b) => $b['usage_rate'] - $a['usage_rate']);

        return $analytics;
    }

    private static function generateRecommendation($usageRate, $avgReorderDays, $currentQty) {
        if ($usageRate < 20) {
            return "LOW_USAGE - Consider removing from pack";
        }

        if ($avgReorderDays && $avgReorderDays < 14) {
            return "INCREASE_QTY - Frequently reordered, increase pack quantity";
        }

        if ($usageRate > 80 && $avgReorderDays > 60) {
            return "OPTIMAL - Good balance";
        }

        return "MONITOR - Continue tracking";
    }

    private static function getCentralWarehouseId() {
        $warehouse = StoreTable::getList([
            'filter' => ['UF_STORE_TYPE' => 'warehouse', 'ACTIVE' => 'Y'],
            'select' => ['ID'],
            'limit' => 1
        ])->fetch();

        return $warehouse['ID'];
    }

    private static function notifyTechnicianAboutStarterPack($userId, $pack, $transferId) {
        // Send email/SMS notification
        \CEvent::Send('STARTER_PACK_APPLIED', 's1', [
            'USER_ID' => $userId,
            'PACK_NAME' => $pack['NAME'],
            'TOTAL_ITEMS' => $pack['TOTAL_ITEMS'],
            'TRANSFER_ID' => $transferId
        ]);
    }
}

/**
 * Get stock levels for a specific warehouse
 * GET /api/inventory/warehouses/{id}/stock
 */
function getWarehouseStock($storeId, $filters = []) {
    $query = [
        'filter' => ['STORE_ID' => $storeId],
        'select' => ['PRODUCT_ID', 'AMOUNT']
    ];

    if (isset($filters['category'])) {
        // Join with products to filter by category
        $query['filter']['PRODUCT.SECTION_ID'] = $filters['category'];
    }

    $stock = StoreProductTable::getList($query)->fetchAll();

    // Enrich with product details
    foreach ($stock as &$item) {
        $product = CIBlockElement::GetByID($item['PRODUCT_ID'])->Fetch();
        $item['sku'] = $product['CODE'];
        $item['name'] = $product['NAME'];
        $item['cost_tech'] = $product['UF_COST_TECH'];
        $item['price_client'] = $product['UF_PRICE_CLIENT'];

        // Calculate reserved quantity
        $item['reserved'] = getReservedQuantity($storeId, $item['PRODUCT_ID']);
        $item['available'] = $item['AMOUNT'] - $item['reserved'];
    }

    return $stock;
}

/**
 * Get multi-location stock for a product
 * GET /api/inventory/products/{id}/locations
 */
function getProductLocations($productId) {
    $stores = StoreProductTable::getList([
        'filter' => ['PRODUCT_ID' => $productId, '>AMOUNT' => 0],
        'select' => ['STORE_ID', 'AMOUNT']
    ])->fetchAll();

    $locations = [];
    foreach ($stores as $stock) {
        $store = StoreTable::getById($stock['STORE_ID'])->fetch();
        $reserved = getReservedQuantity($stock['STORE_ID'], $productId);

        $locations[] = [
            'store_id' => $stock['STORE_ID'],
            'store_name' => $store['TITLE'],
            'store_type' => $store['UF_STORE_TYPE'],
            'quantity' => $stock['AMOUNT'],
            'reserved' => $reserved,
            'available' => $stock['AMOUNT'] - $reserved
        ];
    }

    return $locations;
}

/**
 * Reserve stock for a deal
 * POST /api/inventory/reserve
 */
function reserveStock($storeId, $productId, $quantity, $dealId, $userId) {
    global $DB;

    // Check available stock
    $stock = StoreProductTable::getList([
        'filter' => ['STORE_ID' => $storeId, 'PRODUCT_ID' => $productId],
        'select' => ['AMOUNT']
    ])->fetch();

    $reserved = getReservedQuantity($storeId, $productId);
    $available = $stock['AMOUNT'] - $reserved;

    if ($available < $quantity) {
        throw new \Exception("Insufficient stock. Available: {$available}, Requested: {$quantity}");
    }

    // Create reservation
    $DB->Query("
        INSERT INTO b_catalog_store_reserved
        (STORE_ID, PRODUCT_ID, QUANTITY, DEAL_ID, RESERVED_BY, DATE_RESERVED, DATE_EXPIRES, STATUS)
        VALUES
        ({$storeId}, {$productId}, {$quantity}, {$dealId}, {$userId}, NOW(), DATE_ADD(NOW(), INTERVAL 24 HOUR), 'active')
    ");

    return $DB->LastID();
}

/**
 * Release reservation (deal cancelled or completed)
 * POST /api/inventory/release-reservation/{id}
 */
function releaseReservation($reservationId) {
    global $DB;

    $DB->Query("
        UPDATE b_catalog_store_reserved
        SET STATUS = 'released'
        WHERE ID = {$reservationId}
    ");
}

/**
 * Helper: Get reserved quantity
 */
function getReservedQuantity($storeId, $productId) {
    global $DB;

    $result = $DB->Query("
        SELECT SUM(QUANTITY) as total
        FROM b_catalog_store_reserved
        WHERE STORE_ID = {$storeId}
          AND PRODUCT_ID = {$productId}
          AND STATUS = 'active'
          AND (DATE_EXPIRES IS NULL OR DATE_EXPIRES > NOW())
    ")->Fetch();

    return $result['total'] ?? 0;
}
