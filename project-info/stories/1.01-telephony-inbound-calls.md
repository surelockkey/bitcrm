## 1. Call Routing

### 1.1. Business Logic

**Principle:** All available dispatchers see the incoming call simultaneously. The first one who answers gets the call.

**Available dispatcher:**
- ✅ Online in Bitrix24
- ✅ Has "clock-in" status (started work day)
- ✅ NOT on another call currently

**Behavior:**
```
Call arrives
  ↓
System checks: who is available?
  → Sarah: ✅ online, clocked-in, free
  → Mike: ✅ online, clocked-in, free
  → Tom: ❌ on another call (busy)
  → Lisa: ❌ offline
  ↓
Call shown to Sarah and Mike SIMULTANEOUSLY
(both hear ringing, see pop-up)
  ↓
Sarah clicks [Answer] first
  ↓
Result:
  - Sarah takes the call
  - Mike no longer sees this call (automatically disappears)
  - Sarah is now "busy", doesn't see new incoming calls
  - Mike remains "free", can accept the next call
```

### 1.2. If ALL Dispatchers Are Busy

**Scenario:**
All dispatchers on other calls → new call goes into queue.

**Behavior:**
```
Call arrives, but everyone is busy
  ↓
Call remains visible in shared Call Queue
  ↓
First dispatcher who finishes their call IMMEDIATELY sees:
  - Notification: "1 call waiting in queue"
  - Can accept manually from queue
  ↓
If NO ONE accepts within 120 seconds:
  → Forwarding (forwarding rule)
```

### 1.3. Timeout & Forwarding Rules

**Ring timeout:** 120 seconds

**After 120 seconds with no answer:**

**Option 1: Voicemail**
```
→ Twilio plays message:
  "Sorry we missed your call. Please leave a message after the beep."
→ Client leaves voicemail
→ Voicemail recording is saved
→ Bitrix receives notification + voicemail recording
→ Call Log entry created: STATUS = "Missed (Voicemail)"
```

**Option 2: Forward to Manager**
```
→ Call forwarded to Shift Manager
→ If Manager doesn't answer in 30 sec → Voicemail
```

**Configuration (chosen during implementation):**
```
Bitrix24 → Telephony → Queue Settings:
  Ring timeout: 120 seconds

  After timeout action:
    [▼] Select action:
        - Send to voicemail
        - Forward to user: [Shift Manager ▼]
        - Forward to external number: [+1 (555) 999-9999]
        - Play busy signal and hang up
```

## 2. Pop-up Card (Call Information Pop-up)

### 2.1. When It Appears

Pop-up appears **BEFORE the dispatcher accepts the call** — at the moment when the phone is "ringing".

The dispatcher sees information about the caller and can decide to:
- [Answer] — accept the call
- [Decline] — reject (call remains in queue for others)

### 2.2. What Is Displayed (2 Loading Stages)

#### Stage 1: Instant Load (0 seconds, immediate)

Information from Twilio (does not require database lookup):

```

│                                     │
```

### 2.3. Data Sources

| Field | Source | Load Time |
|------|---------|------------------|
| Phone number | Twilio | Instant (0 sec) |
| Job Source | Twilio webhook metadata | Instant (0 sec) |
| Caller Location (city, state) | Twilio Caller Lookup API | Instant (0 sec) |
| Client Name | Bitrix CRM search by phone | 1-2 sec |
| Client Address | Bitrix CRM | 1-2 sec |
| Last Job Type | Bitrix CRM (last Deal query) | 1-2 sec |
| Last Job Date | Bitrix CRM | 1-2 sec |

### 2.4. Twilio Caller Location

**How it works:**

Twilio Lookup API determines approximate location by area code.

**Example:**
```javascript
// Twilio webhook
const phoneNumber = "+14045551234";

// Twilio Lookup API
twilio.lookups.v1.phoneNumbers(phoneNumber)
  .fetch({ type: ['caller-name'] })
  .then(result => {
    // Area code: 404 → Atlanta, Georgia
    const location = {
      city: "Atlanta",
      state: "GA",
      country: "US"
    };

    // Send to Bitrix
    sendToBitrix({
      phone: phoneNumber,
      caller_city: "Atlanta",
      caller_state: "GA"
    });
  });
```

**Limitations:**
- This is NOT GPS location (not precise address)
- This is lookup by phone number area code
- Precision: city/state (not street/house)
- May be inaccurate if client moved but kept old number

### 2.5. Technical Implementation

#### Bitrix PHP (webhook handler)

```php
// File: /local/php_interface/include/telephony_inbound.php

use Bitrix\Main\EventManager;

EventManager::getInstance()->addEventHandler(
    'voximplant',
    'OnExternalCallStart',
    'showIncomingCallPopup'
);

function showIncomingCallPopup($arParams) {
    $phoneNumber = $arParams['PHONE_NUMBER'];
    $metadata = $arParams['PARAMS'] ?? [];

    // STAGE 1: Instantly show pop-up with basic info
    $popupData = [
        'phone' => $phoneNumber,
        'job_source' => $metadata['job_source'] ?? 'Unknown',
        'caller_city' => $metadata['caller_city'] ?? null,
        'caller_state' => $metadata['caller_state'] ?? null,
        'loading' => true  // show search indicator
    ];

    showPopupToDispatchers($popupData);

    // STAGE 2: Asynchronous CRM search (parallel, doesn't block display)
    searchClientAsync($phoneNumber, function($clientData) use ($arParams) {
        // When found - update pop-up
        updatePopupWithClientData($arParams['CALL_ID'], $clientData);
    });
}

function searchClientAsync($phoneNumber, $callback) {
    // Search Contact by phone number
    $contact = \Bitrix\Crm\ContactTable::getList([
        'filter' => ['PHONE' => $phoneNumber],
        'select' => ['ID', 'NAME', 'LAST_NAME', 'ADDRESS']
    ])->fetch();

    if ($contact) {
        // Find last Deal for this Contact
        $lastDeal = \Bitrix\Crm\DealTable::getList([
            'filter' => ['CONTACT_ID' => $contact['ID']],
            'order' => ['DATE_CREATE' => 'DESC'],
            'select' => ['UF_CRM_JOB_TYPE', 'DATE_CREATE'],
            'limit' => 1
        ])->fetch();

        $callback([
            'found' => true,
            'name' => $contact['NAME'] . ' ' . $contact['LAST_NAME'],
            'address' => $contact['ADDRESS'],
            'last_job_type' => $lastDeal['UF_CRM_JOB_TYPE'] ?? null,
            'last_job_date' => $lastDeal['DATE_CREATE'] ?? null,
            'contact_id' => $contact['ID']
        ]);
    } else {
        $callback([
            'found' => false
        ]);
    }
}
```

**Field Logic:**

**1. Client (in CRM) — NOT editable**
- If number found in CRM → shows "John Doe" (read-only, gray background)
- If new client → shows "New Client" (read-only)
- This field CANNOT be changed, it's for information only

**2. Contact Name for this Deal — OPTIONAL**
- Used when NOT the client himself calls, but someone on his behalf
- Example:
  ```

  - CRM has client "John Doe" (male)
  - His wife Jane calls
  - Dispatcher enters: Contact Name = "Jane Doe"

  Result:
  - Deal is created for client "John Doe" (Contact ID remains)
  - Deal records: "Contact Person: Jane Doe"
  - In Timeline: "Call from Jane Doe (on behalf of John Doe)"
  ```

**3. Address, Job Type, Notes — filled manually**
- Dispatcher listens to client and enters information during conversation

### 3.2. Auto-Save (automatic draft saving)

**Problem:**
Call may drop (connection lost, client hung up) before dispatcher filled all fields.

**Solution:**
System automatically saves draft every 5 seconds + when call ends.

**How it works:**

```
Dispatcher fills form:
  - Street: "123 Main St" ← entered
  - City: "Atlanta" ← entered
  - ZIP: (empty)
  ↓
After 5 seconds → Auto-save in background
  ↓
Call drops (client hung up)
  ↓
Bitrix saves draft:
  - Call ID: abc123
  - Phone: +1 (555) 123-4567
  - Client ID: 456 (John Doe)
  - Contact Name: "Jane Doe"
  - Street: "123 Main St"
  - City: "Atlanta"
  - ZIP: null (not filled)
  - Job Type: null
  - Notes: null
  - Status: DRAFT
  - Saved at: 2025-10-06 14:35:22
```

**Next call from same number:**

```
Client calls back after 10 minutes
  ↓
Bitrix searches: is there a draft for this number?
  → Found draft (from 10 min ago)
  ↓
Pop-up shows:
┌─────────────────────────────────────┐
│ 📞 +1 (555) 123-4567               │
│ Client: John Doe                    │
│                                     │
│ ⚠️ Draft found (10 min ago)        │
│ Street: 123 Main St, Atlanta       │
│                                     │
│ [Answer & Continue Draft]          │
│ [Answer & Start New]               │
└─────────────────────────────────────┘
```

**If dispatcher chooses [Answer & Continue Draft]:**
```
Deal Draft Interface opens with ALREADY FILLED fields:
  - Street: "123 Main St" ← saved
  - City: "Atlanta" ← saved
  - ZIP: [___________] ← empty, needs to be filled
  - Job Type: [___________] ← empty, needs to be filled

Dispatcher continues filling from where they stopped
```

**If dispatcher chooses [Answer & Start New]:**
```
Draft is ignored, clean form opens
Old draft remains in system (can be found in Call History)
```

### 3.4. Draft Storage

**Database structure:**

```sql
CREATE TABLE b_crm_deal_draft (
    ID INT PRIMARY KEY AUTO_INCREMENT,
    CALL_ID VARCHAR(255),           -- Twilio Call SID
    PHONE VARCHAR(50) NOT NULL,     -- Client phone
    CLIENT_ID INT,                   -- Contact ID (if found in CRM)
    CONTACT_NAME VARCHAR(255),       -- Contact person for this deal
    STREET VARCHAR(255),
    CITY VARCHAR(100),
    STATE VARCHAR(50),
    ZIP VARCHAR(20),
    JOB_TYPE VARCHAR(100),
    NOTES TEXT,
    JOB_SOURCE VARCHAR(50),          -- From Twilio metadata
    METRO_AREA VARCHAR(50),          -- From Twilio metadata
    CREATED_BY INT,                  -- Dispatcher user ID
    DATE_CREATE DATETIME,
    DATE_UPDATE DATETIME,            -- Last auto-save
    STATUS ENUM('DRAFT', 'CONVERTED', 'ABANDONED'),
    INDEX idx_phone (PHONE),
    INDEX idx_call_id (CALL_ID)
);
```

**Draft lifecycle:**

```
Draft created:
  → STATUS = 'DRAFT'
  → DATE_CREATE = now()
  ↓
Auto-save every 5 seconds:
  → DATE_UPDATE = now()
  ↓
Deal created:
  → STATUS = 'CONVERTED'
  → Draft no longer shown
  ↓
7 days passed without conversion:
  → STATUS = 'ABANDONED'
  → Deleted automatically (cleanup job)
```

```javascript
    };
}

// Sends to server
function saveDraftToServer() {
    const data = collectFormData();

    // Check if anything is filled
    const hasData = Object.values(data).some(val => val && val.trim() !== '');

    if (!hasData) {
        // Form is empty, don't save
        return;
    }

    // AJAX request to server
    BX.ajax.runAction('crm.deal.draft.save', {
        data: data
    }).then(
        response => {
            console.log('Draft saved:', response.data.draft_id);
            showNotification('Draft saved', 'success');
        },
        error => {
            console.error('Draft save failed:', error);
        }
    );
}

    clearInterval(draftAutoSaveInterval);
```

```php
class DealDraftHandler {

                    NOW(),
                    NOW(),
                    'DRAFT'
                )
            ");

            return $DB->LastID();
        }
    }

    /**
     * Find draft by phone number
     */
    public static function findDraftByPhone($phone) {
        global $DB;

        $phone = $DB->ForSql($phone);

        $result = $DB->Query("
            SELECT * FROM b_crm_deal_draft
            WHERE PHONE = '{$phone}'
              AND STATUS = 'DRAFT'
              AND DATE_CREATE > DATE_SUB(NOW(), INTERVAL 7 DAY)
            ORDER BY DATE_UPDATE DESC
            LIMIT 1
        ")->Fetch();

        return $result ?: null;
    }

    /**
     * Convert draft to Deal
     */
    public static function convertDraftToDeal($draftId) {
        global $DB;

        $draft = $DB->Query("
            SELECT * FROM b_crm_deal_draft WHERE ID = {$draftId}
        ")->Fetch();

        if (!$draft) {
            throw new \Exception('Draft not found');
        }

        // Create Deal
        $deal = new \CCrmDeal();
        $dealId = $deal->Add([
            'TITLE' => $draft['JOB_TYPE'] . ' for ' . $draft['CONTACT_NAME'],
            'CONTACT_ID' => $draft['CLIENT_ID'],
            'PHONE' => [['VALUE' => $draft['PHONE'], 'VALUE_TYPE' => 'WORK']],
            'UF_CONTACT_PERSON' => $draft['CONTACT_NAME'],
            'ADDRESS' => $draft['STREET'] . ', ' . $draft['CITY'] . ', ' . $draft['STATE'] . ' ' . $draft['ZIP'],
            'UF_CRM_JOB_TYPE' => $draft['JOB_TYPE'],
            'UF_CRM_JOB_SOURCE' => $draft['JOB_SOURCE'],
            'UF_CRM_METRO_AREA' => $draft['METRO_AREA'],
            'STAGE_ID' => 'SUBMITTED',
            'COMMENTS' => $draft['NOTES'],
            'ASSIGNED_BY_ID' => $draft['CREATED_BY']
        ]);

        if ($dealId) {
            // Mark draft as converted
            $DB->Query("
                UPDATE b_crm_deal_draft
                SET STATUS = 'CONVERTED'
                WHERE ID = {$draftId}
            ");

            return $dealId;
        }

        throw new \Exception('Failed to create deal');
    }
```

## 4. Acceptance Criteria

### 4.1. Call Routing

- ✅ Incoming call is shown to ALL available dispatchers simultaneously
- ✅ Dispatcher on another call does NOT see new calls
- ✅ First one who answers gets the call, others no longer see it
- ✅ If all are busy — call waits in queue for 120 seconds
- ✅ After 120 sec — voicemail or forwarding (per settings)

### 4.2. Pop-up Card

- ✅ Pop-up appears instantly (0 sec) with number, source, location
- ✅ CRM data (name, address, last job) is added after 1-2 sec
- ✅ If client is new — "New Client" is shown
- ✅ Twilio Caller Location is determined by area code

### 4.3. Deal Draft & Auto-Save

- ✅ Deal Draft Interface opens after accepting call
- ✅ Auto-save every 5 seconds + on call completion
- ✅ Draft is saved if form is partially filled
- ✅ Draft is NOT saved if form is empty
- ✅ On next call from same number — option to continue draft is offered
- ✅ Draft is deleted after 7 days if not converted to Deal

### 4.4. Deal Creation

- ✅ With [Save & Create Deal] — required fields are checked
- ✅ Deal is created with auto-filled Job Source, Metro Area from Twilio
- ✅ Contact Person is saved separately from main Contact
- ✅ Call recording is automatically added to timeline
- ✅ Dispatcher is redirected to Deal page after creation

### 4.5. Spam Handling

- ✅ [Mark as Spam] — Deal is NOT created, number is logged
- ✅ On next call from spam number — warning in pop-up
- ✅ Number is NOT blocked automatically (possibility of error)
