# TR-22: Inventory - Receiving Stock

**Epic:** Inventory Management System
**Component:** Stock Receiving & Intake Processes
**Version:** 1.0
**Last Updated:** October 29, 2025

---

## 1. Overview

This story covers all workflows for receiving inventory into the system, whether delivered to the central warehouse or directly to technicians. The receiving process is the entry point for all physical inventory into the system and must ensure accuracy, proper documentation, and real-time updates.

**Key Objectives:**
- Barcode-driven receiving to minimize data entry errors
- Support multiple receiving scenarios (warehouse, direct to tech, multiple packages)
- Handle quantity discrepancies and damaged items
- Generate proper documentation (receipts, packing slips)
- Real-time inventory updates across all locations
- Mobile-first experience for technicians

---

## 2. Receiving Scenarios

### 2.1. Scenario Overview

The system must support three primary receiving scenarios:

**Scenario A: Central Warehouse Receipt**
- **Who:** Logistics Manager or designated warehouse staff
- **Where:** Company warehouse/office
- **When:** Bulk orders from suppliers (UHS, Amazon, etc.)
- **Device:** Desktop computer with USB barcode scanner OR mobile device
- **Volume:** Typically 50-500 items per shipment

**Scenario B: Direct to Technician**
- **Who:** Individual Technician
- **Where:** Technician's home or job site
- **When:** Items ordered directly to technician's address
- **Device:** Mobile phone with camera (barcode scanning)
- **Volume:** Typically 1-20 items per shipment

**Scenario C: Emergency/Unexpected Receipt**
- **Who:** Anyone who receives package
- **Where:** Any location
- **When:** Package arrives unexpectedly or to wrong location
- **Device:** Any available (mobile preferred)
- **Volume:** Varies

**Step 7: System Actions (Automatic)**

After clicking "Complete Receipt":

```
System performs:
1. ✅ Update inventory quantities
   - Add 270 units to Central Warehouse
   - Update product stock levels in real-time

2. ✅ Generate receipt document
   - Receipt ID: RCV-2025-001234
   - PDF with itemized list
   - Includes photos of damaged items
   - Electronic signature/timestamp

3. ✅ Create discrepancy report
   - Lists all issues
   - Assigns follow-up tasks
   - Tracks to resolution

4. ✅ Send notifications
   - Email to: Owner, Manager, Logistics
   - Subject: "Stock Received - 270 units - 2 issues"
   - Includes summary and link to receipt

5. ✅ Update Purchase Order (if linked)
   - Mark items as received
   - Close PO if fully received
   - Or keep open for backordered items

6. ✅ Log transaction
   - Audit trail entry
   - Timestamp, user, location
   - Before/after quantities

7. ✅ Update dashboard
   - Real-time stock levels refresh
   - Alerts update (low stock resolved?)
   - Reports refresh
```

## 4. Scenario B: Direct to Technician

### 4.1. Workflow Overview

When items are ordered directly to technician's address:

```
Logistics creates order → Supplier ships to tech's home
                                    ↓
                        Package arrives at tech's address
                                    ↓
                        Tech receives SMS/Email notification
                                    ↓
                        Tech opens mobile app → Pending Receipts
                                    ↓
                        Tech scans items using phone camera
                                    ↓
                        System adds to tech's container
                                    ↓
                        Management receives confirmation
```

- ✅ Logistics created "Direct Delivery" order in system

## 6. Barcode Scanning Technology

### 6.1. Desktop/Warehouse Scanning

**Recommended Devices:**
- USB Barcode Scanner (plug-and-play)
- Zebra DS2208, Honeywell Voyager 1200g
- Works like keyboard input
- No special drivers needed

**Setup:**
```
1. Plug USB scanner into computer
2. Open Bitrix24 receiving screen
3. Cursor in barcode field
4. Scan → automatically enters barcode → press Enter
5. Item added to list
```

**Performance:**
- Scan rate: 1-2 items per second
- Range: 0-12 inches from barcode
- Works with printed and screen barcodes
- Supports all common 1D and 2D formats

### 6.2. Mobile Camera Scanning

**Technology:** HTML5 getUserMedia() + ZXing library

**Features:**
- ✅ No app store approval needed (web-based)
- ✅ Works on iOS and Android
- ✅ Auto-focus and auto-capture
- ✅ Flashlight control
- ✅ Vibration feedback on successful scan
- ✅ Works offline (scan → queue → sync when online)

**User Experience:**

```javascript
// Mobile scanning implementation
import { BrowserMultiFormatReader } from '@zxing/library';

const codeReader = new BrowserMultiFormatReader();

// Start scanning
codeReader.decodeFromVideoDevice(null, 'video', (result, err) => {
  if (result) {
    // Barcode detected
    const barcode = result.text;

    // Haptic feedback
    navigator.vibrate(200);

    // Visual feedback
    showSuccessFlash();

    // Lookup product
    lookupProductByBarcode(barcode).then(product => {
      addToReceiptList(product);
    });
  }
});
```

### 7.2. Receipt API Endpoints

```php
// File: /local/api/inventory/receiving.php

namespace Local\Inventory;

class ReceivingManager {

    /**
     * Create new receipt
     * POST /api/inventory/receive/create
     */
    public static function createReceipt($data) {
        global $DB;

        $receiptNumber = self::generateReceiptNumber();
        $userId = $GLOBALS['USER']->GetID();

        $DB->Query("
            INSERT INTO b_inventory_receipt
            (RECEIPT_NUMBER, RECEIPT_TYPE, STORE_ID, SUPPLIER, TRACKING_NUMBER,
             INVOICE_NUMBER, PO_ID, SHIPMENT_DATE, RECEIPT_DATE, TOTAL_ITEMS,
             TOTAL_VALUE, STATUS, NOTES, RECEIVED_BY, DATE_CREATE)
            VALUES
            ('{$receiptNumber}', '{$data['type']}', {$data['store_id']},
             '{$DB->ForSql($data['supplier'])}', '{$DB->ForSql($data['tracking'])}',
             '{$DB->ForSql($data['invoice'])}', " . ($data['po_id'] ?? 'NULL') . ",
             '{$data['shipment_date']}', NOW(), 0, 0, 'draft',
             '{$DB->ForSql($data['notes'])}', {$userId}, NOW())
        ");

        $receiptId = $DB->LastID();

        return [
            'receipt_id' => $receiptId,
            'receipt_number' => $receiptNumber
        ];
    }

    /**
     * Add item to receipt
     * POST /api/inventory/receive/add-item
     */
    public static function addItem($receiptId, $productId, $quantity, $notes = '') {
        global $DB;

        // Get product cost
        $product = \CIBlockElement::GetByID($productId)->Fetch();
        $unitCost = $product['UF_COST_TECH'];
        $totalCost = $unitCost * $quantity;

        // Check if item already in receipt
        $existing = $DB->Query("
            SELECT ID, QUANTITY_RECEIVED FROM b_inventory_receipt_item
            WHERE RECEIPT_ID = {$receiptId} AND PRODUCT_ID = {$productId}
        ")->Fetch();

        if ($existing) {
            // Update quantity
            $newQty = $existing['QUANTITY_RECEIVED'] + $quantity;
            $newTotal = $unitCost * $newQty;

            $DB->Query("
                UPDATE b_inventory_receipt_item
                SET QUANTITY_RECEIVED = {$newQty},
                    TOTAL_COST = {$newTotal},
                    SCANNED_AT = NOW()
                WHERE ID = {$existing['ID']}
            ");

            $itemId = $existing['ID'];
        } else {
            // Insert new item
            $DB->Query("
                INSERT INTO b_inventory_receipt_item
                (RECEIPT_ID, PRODUCT_ID, QUANTITY_RECEIVED, UNIT_COST,
                 TOTAL_COST, NOTES, SCANNED_AT)
                VALUES
                ({$receiptId}, {$productId}, {$quantity}, {$unitCost},
                 {$totalCost}, '{$DB->ForSql($notes)}', NOW())
            ");

            $itemId = $DB->LastID();
        }

        // Update receipt totals
        self::updateReceiptTotals($receiptId);

        return [
            'item_id' => $itemId,
            'quantity' => $quantity,
            'unit_cost' => $unitCost,
            'total_cost' => $totalCost
        ];
    }

    /**
     * Complete receipt (add to inventory)
     * POST /api/inventory/receive/complete
     */
    public static function completeReceipt($receiptId) {
        global $DB;

        // Get receipt details
        $receipt = $DB->Query("
            SELECT * FROM b_inventory_receipt WHERE ID = {$receiptId}
        ")->Fetch();

        if ($receipt['STATUS'] !== 'draft') {
            throw new \Exception("Receipt already completed");
        }

        // Get all items
        $items = $DB->Query("
            SELECT * FROM b_inventory_receipt_item WHERE RECEIPT_ID = {$receiptId}
        ")->fetchAll();

        // Add to inventory
        foreach ($items as $item) {
            self::addToInventory(
                $receipt['STORE_ID'],
                $item['PRODUCT_ID'],
                $item['QUANTITY_RECEIVED']
            );
        }

        // Mark receipt as completed
        $DB->Query("
            UPDATE b_inventory_receipt
            SET STATUS = 'completed',
                DATE_COMPLETE = NOW()
            WHERE ID = {$receiptId}
        ");

        // Generate PDF
        $pdfUrl = self::generateReceiptPDF($receiptId);

        // Send notifications
        self::sendReceiptNotifications($receiptId);

        // Log movement
        self::logInventoryMovement($receiptId, 'receive');

        return [
            'receipt_id' => $receiptId,
            'receipt_number' => $receipt['RECEIPT_NUMBER'],
            'pdf_url' => $pdfUrl,
            'total_items' => $receipt['TOTAL_ITEMS'],
            'total_value' => $receipt['TOTAL_VALUE']
        ];
    }

    /**
     * Report discrepancy
     * POST /api/inventory/receive/report-issue
     */
    public static function reportIssue($receiptId, $issueData) {
        global $DB;
        $userId = $GLOBALS['USER']->GetID();

        $DB->Query("
            INSERT INTO b_inventory_receipt_issue
            (RECEIPT_ID, RECEIPT_ITEM_ID, ISSUE_TYPE, DESCRIPTION,
             QUANTITY_AFFECTED, PHOTOS, DATE_CREATE)
            VALUES
            ({$receiptId}, " . ($issueData['item_id'] ?? 'NULL') . ",
             '{$issueData['type']}', '{$DB->ForSql($issueData['description'])}',
             {$issueData['quantity']}, '{$DB->ForSql(json_encode($issueData['photos']))}',
             NOW())
        ");

        $issueId = $DB->LastID();

        // Mark receipt as having discrepancies
        $DB->Query("
            UPDATE b_inventory_receipt
            SET HAS_DISCREPANCIES = TRUE
            WHERE ID = {$receiptId}
        ");

        // Create follow-up task if needed
        if (!empty($issueData['create_task'])) {
            self::createFollowUpTask($issueId, $issueData);
        }

        // Send alert
        self::sendDiscrepancyAlert($receiptId, $issueId);

        return ['issue_id' => $issueId];
    }

    /**
     * Helper: Add quantity to inventory
     */
    private static function addToInventory($storeId, $productId, $quantity) {
        // Check if product already in this store
        $existing = \Bitrix\Catalog\StoreProductTable::getList([
            'filter' => ['STORE_ID' => $storeId, 'PRODUCT_ID' => $productId],
            'select' => ['ID', 'AMOUNT']
        ])->fetch();

        if ($existing) {
            // Update quantity
            \Bitrix\Catalog\StoreProductTable::update($existing['ID'], [
                'AMOUNT' => $existing['AMOUNT'] + $quantity
            ]);
        } else {
            // Insert new
            \Bitrix\Catalog\StoreProductTable::add([
                'STORE_ID' => $storeId,
                'PRODUCT_ID' => $productId,
                'AMOUNT' => $quantity
            ]);
        }
    }

    /**
     * Helper: Generate unique receipt number
     */
    private static function generateReceiptNumber() {
        $year = date('Y');
        $nextNum = self::getNextReceiptNumber();
        return sprintf('RCV-%s-%06d', $year, $nextNum);
    }

    /**
     * Helper: Update receipt totals
     */
    private static function updateReceiptTotals($receiptId) {
        global $DB;

        $result = $DB->Query("
            SELECT COUNT(*) as item_count, SUM(TOTAL_COST) as total_value
            FROM b_inventory_receipt_item
            WHERE RECEIPT_ID = {$receiptId}
        ")->Fetch();

        $DB->Query("
            UPDATE b_inventory_receipt
            SET TOTAL_ITEMS = {$result['item_count']},
                TOTAL_VALUE = {$result['total_value']}
            WHERE ID = {$receiptId}
        ");
    }

    /**
     * Helper: Send notifications
     */
    private static function sendReceiptNotifications($receiptId) {
        // Implementation: Email to management
        \CEvent::Send('INVENTORY_RECEIVED', 's1', [
            'RECEIPT_ID' => $receiptId
        ]);
    }
}
```
