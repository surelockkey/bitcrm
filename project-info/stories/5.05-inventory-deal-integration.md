# TR-24: Inventory - Deal Integration

**Epic:** Inventory Management System
**Component:** Using Inventory Items in Deals/Jobs
**Version:** 1.0
**Last Updated:** October 31, 2025

---

## 1. Overview

This story covers how technicians and office staff add inventory items to deals (jobs), how inventory is automatically deducted from containers, how the 3-tier pricing model works in deals, and how this integrates with commission calculations.

**Key Objectives:**
- Barcode-first approach for adding items to deals
- Automatic inventory deduction from technician's container
- Support 3-tier pricing (Cost Company, Cost Tech, Price Client)
- Price adjustment within configured limits
- Validate item availability before adding
- Insufficient inventory handling
- Integration with commission reports
- Mobile and desktop workflows
- Real-time inventory updates

---

## 2. Core Concepts

### 2.1. When Inventory is Deducted

**IMPORTANT:** Inventory is deducted **when item is added to deal**, NOT when deal is invoiced/paid.

```
Timeline:

1. Technician arrives at job
2. Uses parts from van
3. Adds items to deal via mobile app
   → ✅ INVENTORY DEDUCTED IMMEDIATELY
4. Completes job
5. Customer pays
6. Commission calculated (using Cost Tech)
```

**Why deduct immediately?**
- ✅ Reflects real-time van inventory
- ✅ Prevents technician from "forgetting" to add items
- ✅ Accurate inventory tracking at all times
- ✅ Other techs know what's truly available

### 2.2. Pricing in Deals

Three prices exist for each item:

```
┌─────────────────────────────────────────────┐
│ LOCK-RES-KW-001 - Kwikset Deadbolt          │
├─────────────────────────────────────────────┤
│ Cost (Company):  $15.00  [hidden from tech] │
│ Cost (Tech):     $18.00  [what tech pays]   │
│ Price (Client):  $45.00  [what client pays] │
└─────────────────────────────────────────────┘
```

**In a deal:**
- **Technician sees:** Cost ($18) and Price ($45)
- **Commission calculation uses:** Cost Tech ($18)
- **Client invoice shows:** Price ($45)
- **Company analysis (internal):** Company profit = ($18 - $15) + commission share

### 2.3. Container Restriction

Items can ONLY be added from:
1. **Technician's own container** (primary)
2. **Central warehouse** (if user is Logistics/Manager/Owner)

**You CANNOT add items from another technician's container.**

## 5. Inventory Deduction Logic

### 5.1. Automatic Deduction Flow

```
User adds item to deal
  ↓
System checks:
1. Is item in tech's container?
   ├─ YES → Continue
   └─ NO → Mark as PENDING
  ↓
2. Sufficient quantity?
   ├─ YES → Continue
   └─ NO → Show warning, allow partial/pending
  ↓
3. Deduct from container immediately
  ↓
4. Create inventory movement record
  ↓
5. Update deal with item details
  ↓
6. Show confirmation to user
```

### 5.2. Inventory Movement Record

Every time an item is added to a deal, a movement record is created:

```sql
INSERT INTO b_inventory_movement (
  PRODUCT_ID,
  SOURCE_STORE_ID,        -- Tech's container
  DEST_TYPE,              -- 'deal'
  DEST_ID,                -- Deal ID
  QUANTITY,
  MOVEMENT_TYPE,          -- 'deal_usage'
  UNIT_COST,              -- Cost Tech
  UNIT_PRICE,             -- Price Client
  MOVED_BY,               -- User ID
  DATE_CREATE
) VALUES (
  12345,                  -- Product
  89,                     -- Tech #5's container
  'deal',
  98765,                  -- Deal #98765
  1,                      -- Qty
  'deal_usage',
  18.00,                  -- Cost Tech
  45.00,                  -- Price Client
  5,                      -- User who added
  NOW()
);
```

## 6. Commission Integration

### 6.1. Parts Cost in Commission Calculation

When calculating commission, the system uses **Cost Tech** (not Cost Company):

```
Commission Calculation for Deal #12345:

Revenue (Total):        $350.00

Deductions:
  Tax:                   $28.00
  Parts (Cost Tech):     $63.00  ← Uses Cost for Tech
  Credit Card Fee:       $10.50
                       ──────────
Profit Pool:            $248.50

Tech Share (40%):       $99.40
Company Share (60%):    $149.10
```

**Parts Detail:**
```
Kwikset Deadbolt × 1:    $18.00  (Cost Tech)
Key Blanks × 5:          $4.00   (Cost Tech)
Batteries × 2:           $0.60   (Cost Tech)
August Lock × 1:         $85.00  (Cost Tech)
Schlage Commercial × 1:  $25.00  (Cost Tech) [PENDING]
                        ────────
Total Parts:             $132.60

But Schlage is PENDING (not fulfilled yet),
so commission uses only fulfilled items:
                        ────────
Parts for Commission:    $63.00
```

### 6.2. Company Internal Analysis

Separately, company tracks true profit using Cost Company:

```
Company Internal Profit Analysis:

Parts used (fulfilled only):
  Kwikset: Sold $45, Tech paid $18, Company paid $15 → Profit: $3
  Keys: Sold $12.50, Tech paid $4, Company paid $3 → Profit: $1
  Batteries: Sold $2, Tech paid $0.60, Company paid $0.40 → Profit: $0.20
  August: Sold $120, Tech paid $85, Company paid $80 → Profit: $5
                                           Parts Profit: $9.20

Company share from commission:             $149.10
                                         ──────────
Company Total Profit:                     $158.30
```

## 9. Technical Implementation

### 9.1. Database Schema

```sql
-- Inventory movements (tracking all inventory changes)
CREATE TABLE IF NOT EXISTS b_inventory_movement (
  ID INT PRIMARY KEY AUTO_INCREMENT,
  MOVEMENT_NUMBER VARCHAR(50) UNIQUE,  -- MOV-2025-123456
  PRODUCT_ID INT NOT NULL,
  SOURCE_STORE_ID INT NULL,            -- NULL if external source
  DEST_STORE_ID INT NULL,              -- NULL if dest is deal/disposal
  DEST_TYPE ENUM('store', 'deal', 'disposal', 'adjustment') NOT NULL,
  DEST_ID INT NULL,                    -- Deal ID if DEST_TYPE = 'deal'
  QUANTITY DECIMAL(18,4) NOT NULL,
  MOVEMENT_TYPE ENUM('transfer', 'receive', 'deal_usage', 'return', 'adjustment', 'stocktake') NOT NULL,
  UNIT_COST DECIMAL(10,2),             -- Cost Tech at time of movement
  UNIT_PRICE DECIMAL(10,2),            -- Price Client (if deal_usage)
  REFERENCE_TYPE VARCHAR(50),          -- 'transfer', 'receipt', 'deal'
  REFERENCE_ID INT,                    -- ID of transfer/receipt/deal
  NOTES TEXT,
  MOVED_BY INT NOT NULL,
  DATE_CREATE DATETIME NOT NULL,
  FOREIGN KEY (PRODUCT_ID) REFERENCES b_iblock_element(ID),
  FOREIGN KEY (SOURCE_STORE_ID) REFERENCES b_catalog_store(ID),
  FOREIGN KEY (DEST_STORE_ID) REFERENCES b_catalog_store(ID),
  INDEX idx_movement_number (MOVEMENT_NUMBER),
  INDEX idx_product (PRODUCT_ID),
```

```php
// File: /local/api/inventory/deal_integration.php

namespace Local\Inventory;

class DealInventoryManager {

    /**
     * Add product to deal from inventory
     * POST /api/inventory/deal/{dealId}/add-product
     */
    public static function addProductToDeal($dealId, $data) {
        global $DB;
        $userId = $GLOBALS['USER']->GetID();

        // Validate deal exists and user has access
        $deal = \CCrmDeal::GetByID($dealId);
        if (!$deal) {
            throw new \Exception("Deal not found");
        }

        $productId = $data['product_id'];
        $quantity = $data['quantity'];
        $sourceStoreId = $data['source_store_id']; // Tech's container
        $priceOverride = $data['price_override'] ?? null;

        // Get product details
        $product = \CIBlockElement::GetByID($productId)->Fetch();
        $costTech = $product['UF_COST_TECH'];
        $costCompany = $product['UF_COST_COMPANY'];
        $priceDefault = $product['UF_PRICE_CLIENT'];

        // Validate price adjustment
        if ($priceOverride) {
            self::validatePriceAdjustment($priceDefault, $priceOverride);
        }
        $finalPrice = $priceOverride ?? $priceDefault;

        // Check availability in tech's container
        $available = self::getStoreStock($sourceStoreId, $productId);

        $inventoryStatus = 'fulfilled';
        $quantityToDeduct = $quantity;

        if ($available < $quantity) {
            if ($available > 0) {
                // Partial fulfillment
                $quantityToDeduct = $available;
                $inventoryStatus = 'partial';
            } else {
                // No stock - mark as pending
                $quantityToDeduct = 0;
                $inventoryStatus = 'pending';
            }
        }

        // Deduct inventory if available
        if ($quantityToDeduct > 0) {
            self::deductFromStore($sourceStoreId, $productId, $quantityToDeduct);

            // Create movement record
            $movementId = self::createMovementRecord([
                'product_id' => $productId,
                'source_store_id' => $sourceStoreId,
                'dest_type' => 'deal',
                'dest_id' => $dealId,
                'quantity' => $quantityToDeduct,
                'movement_type' => 'deal_usage',
                'unit_cost' => $costTech,
                'unit_price' => $finalPrice,
                'moved_by' => $userId
            ]);
        }

        // Add product to deal
        $productRow = [
            'OWNER_TYPE' => 'D',
            'OWNER_ID' => $dealId,
            'PRODUCT_ID' => $productId,
            'PRODUCT_NAME' => $product['NAME'],
            'PRICE' => $finalPrice,
            'QUANTITY' => $quantity,
            'CUSTOMIZED' => 'Y',
            'UF_INVENTORY_SOURCE' => 'tech_container',
            'UF_SOURCE_STORE_ID' => $sourceStoreId,
            'UF_COST_TECH' => $costTech,
            'UF_COST_COMPANY' => $costCompany,
            'UF_INVENTORY_STATUS' => $inventoryStatus,
            'UF_MOVEMENT_ID' => $movementId ?? null,
            'UF_PRICE_ADJUSTED' => $priceOverride ? true : false,
            'UF_PRICE_ADJUST_REASON' => $data['price_reason'] ?? null
        ];

        $rowId = \CCrmProductRow::Add($productRow);

        // Update deal total
        self::updateDealTotal($dealId);

        // Send notification if pending
        if ($inventoryStatus === 'pending') {
            self::notifyPendingItem($dealId, $productId, $userId);
        }

        return [
            'row_id' => $rowId,
            'product_id' => $productId,
            'quantity' => $quantity,
            'quantity_fulfilled' => $quantityToDeduct,
            'status' => $inventoryStatus,
            'price' => $finalPrice,
            'movement_id' => $movementId ?? null
        ];
    }

    /**
     * Remove product from deal (return to inventory)
     * DELETE /api/inventory/deal/{dealId}/product/{rowId}
     */
    public static function removeProductFromDeal($dealId, $rowId, $reason = null) {
        global $DB;
        $userId = $GLOBALS['USER']->GetID();

        // Get product row
        $row = \CCrmProductRow::GetByID($rowId);
        if (!$row || $row['OWNER_ID'] != $dealId) {
            throw new \Exception("Product row not found");
        }

        // Return inventory if it was fulfilled
        if ($row['UF_INVENTORY_STATUS'] === 'fulfilled') {
            $sourceStoreId = $row['UF_SOURCE_STORE_ID'];
            $productId = $row['PRODUCT_ID'];
            $quantity = $row['QUANTITY'];

            // Return to store
            self::returnToStore($sourceStoreId, $productId, $quantity);

            // Create return movement record
            self::createMovementRecord([
                'product_id' => $productId,
                'dest_store_id' => $sourceStoreId,
                'dest_type' => 'store',
                'quantity' => $quantity,
                'movement_type' => 'return',
                'unit_cost' => $row['UF_COST_TECH'],
                'reference_type' => 'deal_removal',
                'reference_id' => $dealId,
                'notes' => "Returned from deal #{$dealId}. Reason: {$reason}",
                'moved_by' => $userId
            ]);
        }

        // Delete product row
        \CCrmProductRow::Delete($rowId);

        // Update deal total
        self::updateDealTotal($dealId);

        return [
            'row_id' => $rowId,
            'returned_to_store' => $row['UF_INVENTORY_STATUS'] === 'fulfilled',
            'quantity_returned' => $row['QUANTITY']
        ];
    }

    /**
     * Mark pending item as fulfilled
     * POST /api/inventory/deal/{dealId}/product/{rowId}/fulfill
     */
    public static function fulfillPendingItem($dealId, $rowId, $data) {
        global $DB;
        $userId = $GLOBALS['USER']->GetID();

        // Get product row
        $row = \CCrmProductRow::GetByID($rowId);
        if ($row['UF_INVENTORY_STATUS'] !== 'pending') {
            throw new \Exception("Item is not pending");
        }

        $sourceStoreId = $row['UF_SOURCE_STORE_ID'];
        $productId = $row['PRODUCT_ID'];
        $quantity = $row['QUANTITY'];

        // Check if now available
        $available = self::getStoreStock($sourceStoreId, $productId);
        if ($available < $quantity) {
            throw new \Exception("Still insufficient inventory. Available: {$available}, Needed: {$quantity}");
        }

        // Deduct from store
        self::deductFromStore($sourceStoreId, $productId, $quantity);

        // Create movement record
        $movementId = self::createMovementRecord([
            'product_id' => $productId,
            'source_store_id' => $sourceStoreId,
            'dest_type' => 'deal',
            'dest_id' => $dealId,
            'quantity' => $quantity,
            'movement_type' => 'deal_usage',
            'unit_cost' => $row['UF_COST_TECH'],
            'unit_price' => $row['PRICE'],
            'notes' => "Fulfilled pending item. Source: {$data['fulfillment_source']}",
            'moved_by' => $userId
        ]);

        // Update product row
        \CCrmProductRow::Update($rowId, [
            'UF_INVENTORY_STATUS' => 'fulfilled',
            'UF_MOVEMENT_ID' => $movementId
        ]);

        return [
            'row_id' => $rowId,
            'status' => 'fulfilled',
            'movement_id' => $movementId
        ];
    }

    /**
     * Update product quantity in deal
     * PATCH /api/inventory/deal/{dealId}/product/{rowId}
     */
    public static function updateProductQuantity($dealId, $rowId, $newQuantity) {
        $row = \CCrmProductRow::GetByID($rowId);
        $oldQuantity = $row['QUANTITY'];
        $diff = $newQuantity - $oldQuantity;

        if ($diff > 0) {
            // Increasing - need to deduct more
            $sourceStoreId = $row['UF_SOURCE_STORE_ID'];
            $productId = $row['PRODUCT_ID'];

            $available = self::getStoreStock($sourceStoreId, $productId);
            if ($available < $diff) {
                throw new \Exception("Insufficient inventory for increase");
            }

            self::deductFromStore($sourceStoreId, $productId, $diff);

            // Create movement for increase
            self::createMovementRecord([
                'product_id' => $productId,
                'source_store_id' => $sourceStoreId,
                'dest_type' => 'deal',
                'dest_id' => $dealId,
                'quantity' => $diff,
                'movement_type' => 'deal_usage',
                'unit_cost' => $row['UF_COST_TECH'],
                'unit_price' => $row['PRICE'],
                'notes' => "Quantity increased from {$oldQuantity} to {$newQuantity}",
                'moved_by' => $GLOBALS['USER']->GetID()
            ]);

        } else if ($diff < 0) {
            // Decreasing - return to store
            $returnQty = abs($diff);
            self::returnToStore($row['UF_SOURCE_STORE_ID'], $row['PRODUCT_ID'], $returnQty);

            // Create return movement
            self::createMovementRecord([
                'product_id' => $row['PRODUCT_ID'],
                'dest_store_id' => $row['UF_SOURCE_STORE_ID'],
                'dest_type' => 'store',
                'quantity' => $returnQty,
                'movement_type' => 'return',
                'notes' => "Quantity decreased from {$oldQuantity} to {$newQuantity}",
                'moved_by' => $GLOBALS['USER']->GetID()
            ]);
        }

        // Update row
        \CCrmProductRow::Update($rowId, ['QUANTITY' => $newQuantity]);
        self::updateDealTotal($dealId);

        return ['new_quantity' => $newQuantity];
    }

    /**
     * Validate price adjustment within limits
     */
    private static function validatePriceAdjustment($defaultPrice, $newPrice) {
        $maxIncrease = $defaultPrice * 1.15; // +15%
        $maxDecrease = $defaultPrice * 0.85; // -15%

        if ($newPrice > $maxIncrease || $newPrice < $maxDecrease) {
            throw new \Exception("Price adjustment exceeds ±15% limit. Allowed range: {$maxDecrease} - {$maxIncrease}");
        }
    }

    /**
     * Get commission-relevant parts cost for deal
     * Used by commission calculation system
     */
    public static function getDealPartsCost($dealId) {
        $rows = \CCrmProductRow::LoadRows('D', $dealId);

        $totalCost = 0;
        foreach ($rows as $row) {
            // Only include fulfilled items
            if ($row['UF_INVENTORY_STATUS'] === 'fulfilled') {
                $totalCost += $row['UF_COST_TECH'] * $row['QUANTITY'];
            }
        }

        return $totalCost;
    }

    /**
     * Helper: Get stock level for store
     */
    private static function getStoreStock($storeId, $productId) {
        $stock = \Bitrix\Catalog\StoreProductTable::getList([
            'filter' => ['STORE_ID' => $storeId, 'PRODUCT_ID' => $productId],
            'select' => ['AMOUNT']
        ])->fetch();

        return $stock['AMOUNT'] ?? 0;
    }

    /**
     * Helper: Deduct from store
     */
    private static function deductFromStore($storeId, $productId, $quantity) {
        $stock = \Bitrix\Catalog\StoreProductTable::getList([
            'filter' => ['STORE_ID' => $storeId, 'PRODUCT_ID' => $productId],
            'select' => ['ID', 'AMOUNT']
        ])->fetch();

        if ($stock) {
            \Bitrix\Catalog\StoreProductTable::update($stock['ID'], [
                'AMOUNT' => $stock['AMOUNT'] - $quantity
            ]);
        }
    }

    /**
     * Helper: Return to store
     */
    private static function returnToStore($storeId, $productId, $quantity) {
        $stock = \Bitrix\Catalog\StoreProductTable::getList([
            'filter' => ['STORE_ID' => $storeId, 'PRODUCT_ID' => $productId],
            'select' => ['ID', 'AMOUNT']
        ])->fetch();

        if ($stock) {
            \Bitrix\Catalog\StoreProductTable::update($stock['ID'], [
                'AMOUNT' => $stock['AMOUNT'] + $quantity
            ]);
        }
    }

    /**
     * Helper: Create movement record
     */
    private static function createMovementRecord($data) {
        global $DB;

        $movementNumber = self::generateMovementNumber();

        $fields = array_merge([
            'MOVEMENT_NUMBER' => $movementNumber,
            'DATE_CREATE' => date('Y-m-d H:i:s')
        ], $data);

        $DB->Query("
            INSERT INTO b_inventory_movement
            (" . implode(',', array_keys($fields)) . ")
            VALUES
            ('" . implode("','", array_values($fields)) . "')
        ");

        return $DB->LastID();
    }

    /**
     * Helper: Update deal total
     */
    private static function updateDealTotal($dealId) {
        $rows = \CCrmProductRow::LoadRows('D', $dealId);
        $total = 0;
        foreach ($rows as $row) {
            $total += $row['PRICE'] * $row['QUANTITY'];
        }

        \CCrmDeal::Update($dealId, ['OPPORTUNITY' => $total]);
    }

    /**
     * Helper: Generate movement number
     */
    private static function generateMovementNumber() {
        $year = date('Y');
        $nextNum = rand(100000, 999999); // Simplified
        return sprintf('MOV-%s-%06d', $year, $nextNum);
    }
}
```
