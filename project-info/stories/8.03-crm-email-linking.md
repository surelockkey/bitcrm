# Story 8.03: CRM - Email Linking & Deal Creation

---

## Overview

This story covers the integration between Gmail (Google Workspace) and Bitrix24 CRM, enabling dispatchers to manually link emails to existing client records (Company/Contact) and create Deals directly from email threads. This is a selective, manual linking process designed to prevent CRM noise while maintaining full context for important client communications.

---

## Business Context

**Current Pain Point:**
- Emails from clients arrive in Gmail, not visible in CRM
- Dispatcher must manually copy email content into Deal notes
- No link between email thread and Deal timeline
- Work Orders arrive via email, require manual processing

**Solution:**
Selective email linking workflow where dispatcher identifies relevant client emails, links them to CRM entities, and creates Deals with full email context preserved.

**Expected Outcome:**
- Email history visible in Deal timeline
- Deal creation from email in 1 click
- Work Order attachments auto-linked to Deal
- No CRM noise from irrelevant emails

---

## User Personas

### Dispatcher
- **Goal:** Quickly create Deals from client emails with Work Orders
- **Pain:** Switching between Gmail and Bitrix, manual data copying
- **Need:** One-click Deal creation from email, automatic attachment linking

### Manager (Platinum Accounts)
- **Goal:** Track all communication with government/commercial clients
- **Pain:** Email history scattered, Work Orders lost in inbox
- **Need:** Centralized email history per client, searchable Work Orders

## Functional Requirements

### 1. Gmail Integration Architecture

**Integration Approach:**

Two implementation options for Gmail integration:

**Option A: Email Client Inside Bitrix24**
```
User opens Gmail inbox inside Bitrix24 interface
  ↓
Bitrix24 iframe/widget displays Gmail
  ↓
User selects email → Clicks "Link to Client"
  ↓
Bitrix24 interface overlays Gmail
  ↓
User searches CRM, selects Company/Contact
  ↓
Email linked to CRM entity
```

**Option B: Bitrix24 Widget Inside Gmail** (Recommended)
```
User opens Gmail in browser
  ↓
Bitrix24 browser extension/widget appears in sidebar
  ↓
User selects email → Widget shows CRM search
  ↓
User searches, selects Company/Contact
  ↓
Email linked via Bitrix24 API
  ↓
User can create Deal directly from widget
```

**Recommendation:** Option B (widget inside Gmail) provides better UX as dispatchers already work in Gmail daily.

### 5. Work Order Attachment Handling

**Automatic Detection:**

```
Email has attachment with filename patterns:
  - WO-*.pdf
  - Work Order*.pdf
  - *WorkOrder*.pdf
  - *.pdf (with "Work Order" in email subject)
  ↓
System auto-detects:
  "This email appears to contain a Work Order"
  ↓
When creating Deal:
  - Attachment auto-linked as Work Order
  - WO number extracted (if in filename: WO-2025-11-005)
  - Stored in Deal field: UF_WORK_ORDER_FILE
  - Added to central Work Order registry (Epic 9)
```

**Manual Upload:**

```
If attachment not auto-detected:
  Dispatcher can manually designate:
    [This is a Work Order] checkbox
    ↓
  System treats as WO attachment
```

### 7. Selective Linking (No Auto-Sync)

**Critical Requirement:** System does NOT auto-sync all emails to CRM.

**Reasoning:**
- Prevent CRM noise (newsletters, spam, internal emails)
- Dispatcher manually chooses relevant emails
- Only business-critical client communications linked

**Workflow:**

```
Dispatcher inbox has 50 emails today:
  - 10 client emails (relevant)
  - 20 newsletters (irrelevant)
  - 15 internal team emails (irrelevant)
  - 5 spam (irrelevant)
  ↓
Dispatcher manually links ONLY 10 client emails
  ↓
CRM contains ONLY relevant business communications
  ↓
Clean timeline, no noise
```

### 2. Bitrix24 Email Linker Component

```php
// File: /local/components/bitrix/crm.email.linker/class.php

namespace Bitrix\Crm\Component;

use Bitrix\Main\Loader;
use Bitrix\Crm\ActivityTable;

class EmailLinkerComponent extends \CBitrixComponent {

    /**
     * Link email to CRM entity (Company or Contact)
     */
    public function linkEmailToEntity($emailData, $entityType, $entityId) {
        Loader::includeModule('crm');

        // Step 1: Create Activity
        $activityId = ActivityTable::add([
            'OWNER_TYPE_ID' => $this->getOwnerTypeId($entityType),
            'OWNER_ID' => $entityId,
            'TYPE_ID' => \CCrmActivityType::Email,
            'SUBJECT' => $emailData['subject'],
            'DESCRIPTION' => $emailData['body'],
            'PROVIDER_ID' => 'GMAIL',
            'PROVIDER_TYPE_ID' => 'EMAIL',
            'ASSOCIATED_ENTITY_ID' => $emailData['message_id'],
            'COMPLETED' => 'Y',
            'DIRECTION' => \CCrmActivityDirection::Incoming,
            'START_TIME' => $emailData['date'],
            'END_TIME' => $emailData['date']
        ])->getId();

        // Step 2: Store attachments
        if (!empty($emailData['attachments'])) {
            foreach ($emailData['attachments'] as $attachment) {
                $this->storeAttachment($activityId, $attachment);
            }
        }

        // Step 3: Label email in Gmail (for tracking)
        $this->addGmailLabel($emailData['message_id'], 'Bitrix24-Linked');

        return $activityId;
    }

    /**
     * Create Deal from linked email
     */
    public function createDealFromEmail($emailData, $dealData) {
        Loader::includeModule('crm');

        // Create Deal
        $deal = new \CCrmDeal();
        $dealId = $deal->Add([
            'TITLE' => $dealData['title'] ?? $emailData['subject'],
            'COMPANY_ID' => $dealData['company_id'],
            'CONTACT_ID' => $dealData['contact_id'] ?? null,
            'UF_CRM_JOB_TYPE' => $dealData['job_type'],
            'ADDRESS' => $dealData['address'] ?? '',
            'COMMENTS' => $emailData['body'],
            'UF_CRM_JOB_SOURCE' => 'Email',
            'STAGE_ID' => 'SUBMITTED',
            'ASSIGNED_BY_ID' => $GLOBALS['USER']->GetID()
        ]);

        if (!$dealId) {
            throw new \Exception('Failed to create deal: ' . $deal->LAST_ERROR);
        }

        // Link email Activity to Deal
        ActivityTable::update($emailData['activity_id'], [
            'OWNER_TYPE_ID' => \CCrmOwnerType::Deal,
            'OWNER_ID' => $dealId
        ]);

        // If Work Order attachment present, link to Deal
        if ($this->isWorkOrderAttachment($emailData['attachments'])) {
            $this->linkWorkOrderToDeal($dealId, $emailData['attachments']);
        }

        return $dealId;
    }

    /**
     * Check if attachment is a Work Order
     */
    private function isWorkOrderAttachment($attachments) {
        foreach ($attachments as $file) {
            $filename = strtolower($file['filename']);
            if (
                strpos($filename, 'wo-') === 0 ||
                strpos($filename, 'workorder') !== false ||
                strpos($filename, 'work order') !== false ||
                strpos($filename, 'work_order') !== false
            ) {
                return true;
            }
        }
        return false;
    }

    /**
     * Store attachment in Bitrix24
     */
    private function storeAttachment($activityId, $attachment) {
        // Download file from Gmail
        $fileContent = $this->downloadGmailAttachment($attachment['id']);

        // Upload to Bitrix24 storage
        $fileArray = [
            'name' => $attachment['filename'],
            'type' => $attachment['mimeType'],
            'content' => $fileContent,
            'MODULE_ID' => 'crm'
        ];

        $fileId = \CFile::SaveFile($fileArray, 'crm');

        // Link file to Activity
        ActivityTable::update($activityId, [
            'STORAGE_ELEMENT_IDS' => [$fileId]
        ]);

        return $fileId;
    }

    /**
     * Get CRM owner type ID
     */
    private function getOwnerTypeId($entityType) {
        return match($entityType) {
            'contact' => \CCrmOwnerType::Contact,
            'company' => \CCrmOwnerType::Company,
            'deal' => \CCrmOwnerType::Deal,
            default => throw new \Exception('Invalid entity type')
        };
    }

    /**
     * Add label to Gmail message (for tracking)
     */
    private function addGmailLabel($messageId, $labelName) {
        // Gmail API call to add label
        $gmailService = $this->getGmailService();
        $gmailService->users_messages->modify('me', $messageId, [
            'addLabelIds' => [$this->getLabelId($labelName)]
        ]);
    }

    /**
     * Get Gmail service instance
     */
    private function getGmailService() {
        // Initialize Gmail API client
        // (implementation depends on Google API PHP client library)
        return new \Google_Service_Gmail($this->getGoogleClient());
    }
}
```

### 3. Database Schema

```sql
-- Email Activity stored in standard Bitrix24 table
b_crm_activity
  ID
  OWNER_TYPE_ID (Contact=3, Company=4, Deal=2)
  OWNER_ID
  TYPE_ID (Email=4)
  SUBJECT
  DESCRIPTION (email body)
  PROVIDER_ID ('GMAIL')
  PROVIDER_TYPE_ID ('EMAIL')
  ASSOCIATED_ENTITY_ID (Gmail Message ID)
  COMPLETED ('Y')
  DIRECTION (Incoming=2, Outgoing=1)
  START_TIME (email date)
  STORAGE_ELEMENT_IDS (attachment file IDs, JSON array)

-- Gmail Message ID tracking (for deduplication)
CREATE TABLE b_crm_gmail_link (
    ID INT PRIMARY KEY AUTO_INCREMENT,
    GMAIL_MESSAGE_ID VARCHAR(255) UNIQUE NOT NULL,
    CRM_ACTIVITY_ID INT NOT NULL,
    CRM_ENTITY_TYPE ENUM('contact','company','deal'),
    CRM_ENTITY_ID INT NOT NULL,
    DATE_LINKED DATETIME NOT NULL,
    LINKED_BY INT NOT NULL,
    INDEX idx_message (GMAIL_MESSAGE_ID),
    INDEX idx_entity (CRM_ENTITY_TYPE, CRM_ENTITY_ID)
);
```
