# Story 4.01: Interactive Dispatch Map

**Epic:** [EPIC 4: Scheduling & Dispatching](../epics/EPIC-4-Scheduling.md)
**Story Points:** 8
**Priority:** High
**Complexity:** High (Custom map integration)

---

## User Story

**As a** Shift Manager/Dispatcher
**I want** to see all unassigned jobs and technician locations on an interactive map
**So that** I can quickly visualize workload distribution and make informed assignment decisions without switching between multiple tools

---

## Business Context

**Current Pain Point:** Dispatchers currently use Workiz + Google Sheets + Google Maps in separate windows. They manually copy addresses, search for technicians, and track availabillity in the spreadsheet. This workflow is slow and error-prone.

**Desired Outcome:** Single unified map interface where dispatcher can:
- See all unassigned jobs at a glance
- View technician real-time locations
- Click job → see recommendations → assign in <10 seconds

**Success Metric:** Reduce average job assignment time from 3-5 minutes to <1 minute

## Technical Requirements

### Map Provider Options

**Option A: Google Maps JavaScript API (Recommended)**
- Pros: Rich features, accurate geocoding, familiar UI
- Cons: Cost ($7 per 1,000 requests after free tier)
- Integration: Load via script tag, use `google.maps` API

**Option B: Bitrix24 Native Maps**
- Pros: No additional cost, built-in integration
- Cons: Limited customization, may lack features
- Integration: Use Bitrix24 REST API `crm.map.show`

**Decision:** Evaluate Bitrix24 maps first. If insufficient, fall back to Google Maps.

### Geocoding Strategy

**Address to Coordinates:**
- Use Google Geocoding API when Deal address is created/updated
- Cache coordinates in `UF_LATITUDE` and `UF_LONGITUDE` fields
- Re-geocode if address changes
- Fallback: If geocoding fails, use metro area center + show warning

```javascript
async function geocodeAddress(address) {
  // Check cache first
  const cached = await db.query(
    "SELECT latitude, longitude FROM b_crm_deal WHERE address = ?",
    [address]
  );
  if (cached) return cached;

  // Call Google Geocoding API
  const response = await fetch(
    `https://maps.googleapis.com/maps/api/geocode/json?address=${encodeURIComponent(address)}&key=${API_KEY}`
  );
  const data = await response.json();

  if (data.status === "OK") {
    const { lat, lng } = data.results[0].geometry.location;
    return { lat, lng };
  } else {
    // Fallback to metro area center
    return METRO_AREA_CENTER;
  }
}
```

## Edge Cases & Error Handling

### Geocoding Failures

**Problem:** Address cannot be geocoded (invalid, incomplete, etc.)

**Solution:**
1. Show warning on Deal: "Address could not be geocoded. Job will not appear on dispatch map."
2. Place pin at metro area center with red border (indicates approximation)
3. Allow manual lat/lng override by dispatcher

### High Job Density

**Problem:** >100 jobs in same area, map becomes cluttered

**Solution:**
1. Enable pin clustering (group nearby jobs into numbered circles)
2. Clicking cluster zooms in to expand
3. List view toggle: Show jobs in table format instead of map

### Technician Location Unavailable

**Problem:** Technician has GPS disabled or location is stale

**Solution:**
1. Show technician icon at last known location with gray overlay
2. Tooltip: "Last seen: 45 minutes ago"
3. Exclude from ETA calculations in recommendation algorithm

### Map Load Failure

**Problem:** Google Maps API fails to load (network issue, API key issue)

**Solution:**
1. Show error message: "Map failed to load. Please refresh the page."
2. Fallback: Display jobs in list view with addresses
3. Log error to monitoring system for investigation
