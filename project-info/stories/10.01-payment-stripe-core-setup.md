## 1. System Overview

### 1.1. Purpose

Implement a comprehensive payment processing system based on **Bitrix24 Self-Hosted** + **Stripe** that provides:

- Payment acceptance through multiple channels (Credit Card, ACH, Tap to Pay, Apple/Google Pay)
- Automatic service fee calculation for credit card payments
- Refund and dispute processing with automatic logging
- Sunbit financing integration for installment payments
- Automatic late payment fee calculation for overdue invoices
- 100% transaction logging in CRM

### 1.2. Current Infrastructure (Migrating From)

- **Workiz:** CRM + invoicing + payments (backend: Stripe)
- **Transaction Volume:** ~10,000+ transactions/year

## 2. Payment Methods

### 2.1. Credit Card Payments

**Requirements:**
- Support for all major cards: Visa, Mastercard, Amex, Discover
- Accept payments through:
  - **Apple Pay / Google Pay** (for online and mobile)
  - **Tap to Pay on iPhone** (technician accepts payment on-site)
  - **Manual Card Entry** (through web form)
  - **Payment Link** (client pays via link from email/SMS)

**Implementation:**
- Stripe Payment Intents API for all card payments
- Stripe Terminal SDK for Tap to Pay on mobile app
- Automatic service fee calculation on card payment

**Flow Diagram:**

```
Client Ready to Pay
  ↓
Payment Method Selection:
  ├─ Online (Payment Link)
  │   └─ Stripe Checkout → Card/Apple Pay/Google Pay
  │
  ├─ On-site (Tap to Pay)
  │   └─ Tech iPhone → NFC Reader → Card tap
  │
  └─ Phone Order (Manual Entry)
      └─ Dispatcher enters card details
  ↓
Stripe processes payment
  ↓
Webhook → Bitrix24
  ↓
Deal updated: Payment Status = "Paid"
Timeline entry: "💳 Payment received: $XXX"
```

### 2.2. ACH Bank Transfers

**Requirements:**
- Support ACH Direct Debit (client authorizes debit)
- Support ACH Credit (client initiates transfer from bank)
- Display ACH transaction status (pending → succeeded/failed)

**Implementation:**
- Stripe ACH API
- Webhook for status updates after clearing period (3-5 days)

**Status Flow:**

```
Day 1:
  Client authorizes ACH debit
  Status: "Pending"

Day 3-5:
  ACH clearing period
  Status: "Processing"

Day 5:
  Bank confirms transaction
  Status: "Succeeded"

  OR

  Bank rejects transaction
  Status: "Failed"
  Reason: "Insufficient funds"
```

**Bitrix Deal Updates:**


```php
// File: /local/php_interface/include/offline_payment_handler.php

use Bitrix\Sale\Payment;
use Bitrix\Sale\Order;

function registerOfflinePayment($dealId, $amount, $method, $notes) {
    global $USER;

    // Get order associated with deal
    $order = Order::loadByAccountNumber($dealId);

    if (!$order) {
        throw new \Exception('Order not found for deal');
    }

    // Create payment
    $payment = Payment::create($order->getPaymentCollection());
    $payment->setFields([
        'SUM' => $amount,
        'CURRENCY' => 'USD',
        'PAID' => 'Y',
        'DATE_PAID' => new \Bitrix\Main\Type\DateTime(),
        'PAY_SYSTEM_ID' => getOfflinePaySystemId($method),
        'IS_RETURN' => 'N',
        'PS_STATUS' => 'Paid',
        'PS_STATUS_DESCRIPTION' => $notes
    ]);

    $result = $payment->save();

    if ($result->isSuccess()) {
        // Update deal
        \CCrmDeal::Update($dealId, [
            'UF_CRM_PAYMENT_STATUS' => 'Paid',
            'UF_CRM_PAYMENT_METHOD' => $method,
            'STAGE_ID' => 'WON'
        ]);

        // Add timeline entry
        \Bitrix\Crm\Timeline\CommentController::getInstance()->onCreate([
            'ENTITY_TYPE_ID' => \CCrmOwnerType::Deal,
            'ENTITY_ID' => $dealId,
            'AUTHOR_ID' => $USER->GetID(),
            'COMMENT' => "💵 Offline payment received: $" . number_format($amount, 2) .
                        "\nMethod: " . $method .
                        "\nNotes: " . $notes
        ]);

        return $payment->getId();
    }

    throw new \Exception('Failed to create payment: ' . implode(', ', $result->getErrorMessages()));
}

function getOfflinePaySystemId($method) {
    // Map payment method to pay system ID
    $mapping = [
        'Cash' => 1,
        'Check' => 2,
        'Cash by Referral' => 3
    ];

    return $mapping[$method] ?? 1;
}
```

### 2.4. Installments (Sunbit)

**Requirements:**
- Integration with existing Sunbit account
- Client applies for financing directly from Invoice
- After approval from Sunbit:
  - Invoice in Bitrix24 automatically marked as "Paid"
  - Deal transitions to `Done` status
  - Sunbit disburses full amount to company (minus their commission)

**Implementation:**
- Sunbit API integration (separate documentation: TR-15)
- Webhook from Sunbit → Bitrix24 for status updates

**Preview (detailed in TR-15):**

```
Client sees Invoice with total > $500
  ↓
[Financing Options] button displayed
  ↓
Client clicks → Opens Sunbit application
  ↓
Application submitted (2 minutes)
  ↓
Sunbit processes application (approval time varies)
  ↓
WHILE AWAITING SUNBIT DECISION:
  Payment Status = "Pending (Sunbit)"
  Invoice remains unpaid in Bitrix24
  ↓
IF APPROVED:
  Sunbit webhook → Bitrix24
  Deal: Payment Method = "Sunbit (Installments)"
  Payment Status = "Paid"
  ↓
Next business day: Sunbit disburses funds
```

**Webhook Handler Implementation:**

```php
// File: /local/api/stripe/webhook.php

<?php
require_once($_SERVER['DOCUMENT_ROOT'] . '/bitrix/modules/main/include/prolog_before.php');

use Bitrix\Main\Context;

// Set error logging
error_log("Stripe webhook received at " . date('Y-m-d H:i:s'));

// Get webhook payload
$payload = @file_get_contents('php://input');
$sig_header = $_SERVER['HTTP_STRIPE_SIGNATURE'] ?? '';
$endpoint_secret = \COption::GetOptionString('stripe', 'webhook_secret');

// Load Stripe library
require_once(__DIR__ . '/../../vendor/stripe-php/init.php');
\Stripe\Stripe::setApiKey(\COption::GetOptionString('stripe', 'secret_key'));

try {
    // Verify webhook signature
    $event = \Stripe\Webhook::constructEvent(
        $payload,
        $sig_header,
        $endpoint_secret
    );

    error_log("Stripe webhook verified: " . $event->type);

    // Route to appropriate handler
    switch ($event->type) {
        case 'payment_intent.succeeded':
            handlePaymentSucceeded($event->data->object);
            break;

        case 'payment_intent.payment_failed':
            handlePaymentFailed($event->data->object);
            break;

        case 'payment_intent.processing':
            handlePaymentProcessing($event->data->object);
            break;

        case 'charge.dispute.created':
            handleDisputeCreated($event->data->object);
            break;

        case 'charge.dispute.closed':
            handleDisputeClosed($event->data->object);
            break;

        case 'charge.refunded':
            handleRefundCreated($event->data->object);
            break;

        default:
            error_log("Unhandled webhook event type: " . $event->type);
    }

    http_response_code(200);
    echo json_encode(['status' => 'success']);

} catch (\Stripe\Exception\SignatureVerificationException $e) {
    // Invalid signature
    error_log("Stripe webhook signature verification failed: " . $e->getMessage());
    http_response_code(400);
    exit();

} catch (\Exception $e) {
    error_log("Stripe webhook error: " . $e->getMessage());
    http_response_code(500);
    exit();
}

/**
 * Handle successful payment
 */
function handlePaymentSucceeded($paymentIntent) {
    $dealId = $paymentIntent->metadata->bitrix_deal_id ?? null;

    if (!$dealId) {
        error_log("No deal ID in payment intent metadata");
        return;
    }

    $amount = $paymentIntent->amount / 100; // Convert cents to dollars
    $paymentMethod = $paymentIntent->payment_method_types[0] ?? 'unknown';

    // Update Deal
    \CCrmDeal::Update($dealId, [
        'UF_CRM_PAYMENT_STATUS' => 'Paid',
        'UF_CRM_STRIPE_PI_ID' => $paymentIntent->id,
        'UF_CRM_PAYMENT_METHOD' => ucfirst($paymentMethod),
        'STAGE_ID' => 'WON'
    ]);

    // Add timeline entry
    \Bitrix\Crm\Timeline\CommentController::getInstance()->onCreate([
        'ENTITY_TYPE_ID' => \CCrmOwnerType::Deal,
        'ENTITY_ID' => $dealId,
        'COMMENT' => "💳 Payment received: $" . number_format($amount, 2) .
                    "\nMethod: " . ucfirst($paymentMethod) .
                    "\nStripe ID: " . $paymentIntent->id
    ]);

    // Create payment record in Bitrix
    createBitrixPayment($dealId, $amount, $paymentIntent->id);

    error_log("Payment succeeded for deal #$dealId: $$amount");
}

/**
 * Handle failed payment
 */
function handlePaymentFailed($paymentIntent) {
    $dealId = $paymentIntent->metadata->bitrix_deal_id ?? null;

    if (!$dealId) {
        return;
    }

    $errorMessage = $paymentIntent->last_payment_error->message ?? 'Unknown error';

    // Update Deal
    \CCrmDeal::Update($dealId, [
        'UF_CRM_PAYMENT_STATUS' => 'Failed'
    ]);

    // Add timeline entry
    \Bitrix\Crm\Timeline\CommentController::getInstance()->onCreate([
        'ENTITY_TYPE_ID' => \CCrmOwnerType::Deal,
        'ENTITY_ID' => $dealId,
        'COMMENT' => "❌ Payment FAILED\nReason: " . $errorMessage .
                    "\nStripe ID: " . $paymentIntent->id
    ]);

    // Notify dispatcher
    $deal = \CCrmDeal::GetByID($dealId);
    $assignedUserId = $deal['ASSIGNED_BY_ID'];

    \CIMNotify::Add([
        'TO_USER_ID' => $assignedUserId,
        'NOTIFY_TYPE' => \IM_NOTIFY_SYSTEM,
        'NOTIFY_MODULE' => 'crm',
        'NOTIFY_MESSAGE' => "⚠️ Payment FAILED for Deal #$dealId. Reason: $errorMessage"
    ]);

    error_log("Payment failed for deal #$dealId: $errorMessage");
}
```
