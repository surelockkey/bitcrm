# Story 4.03: Mobile App for Technicians

**Epic:** [EPIC 4: Scheduling & Dispatching](../epics/EPIC-4-Scheduling.md)
**Story Points:** 15
**Priority:** High
**Complexity:** High (Mobile + GPS + Real-time sync)

---

## User Story

**As a** Technician
**I want** a mobile app that shows my assigned jobs in sequence, enables GPS tracking, and lets me update job status
**So that** I can navigate efficiently, share my location with dispatchers, and keep job status updated in real-time

**As a** Shift Manager/Dispatcher
**I want** to see technician locations in real-time on the dispatch map
**So that** I can monitor job progress and optimize route assignments

---

## Business Context

**Current Pain Point:** Technicians manually check jobs across multiple systems (Workiz mobile + calls from dispatcher). No real-time location visibility for dispatchers. Manual status updates via phone calls are slow and disruptive.

**Desired Outcome:**
- Technician sees all jobs in sequence [1], [2], [3] on mobile app
- One-tap navigation to next job
- Quick status updates (Accept → In Progress → Done)
- Real-time GPS location shared with dispatch map
- Offline support for areas with poor connectivity

**Success Metrics:**
- Reduce status update time from 2-3 minutes (phone call) to <10 seconds (button tap)
- Enable real-time location visibility for 95%+ of work hours
- Reduce "where is the technician?" calls from dispatcher by 80%

## Technical Requirements

### Mobile App Technology Stack

**Primary Approach: Bitrix24 Mobile App**
- **Platform:** Bitrix24 native mobile app (iOS + Android)
- **Customization:** Bitrix24 Mobile SDK for custom modules
- **Push Notifications:** Bitrix24 built-in push notification service
- **GPS Tracking:** Native mobile geolocation APIs via Bitrix24 REST API
- **Offline Storage:** Bitrix24 mobile app cache + local database
- **Navigation:** Deep links to Google Maps / Apple Maps from mobile app

**Why Bitrix24 Mobile:**
- ✅ No need to build separate mobile app from scratch
- ✅ Single Sign-On (SSO) with Bitrix24 account
- ✅ Native integration with Bitrix24 CRM, Calendar, Tasks
- ✅ Built-in push notifications and user authentication
- ✅ Automatic updates via App Store / Google Play
- ✅ Lower development and maintenance costs

**Custom Module Development:**
- Use Bitrix24 REST API for backend communication
- Develop custom UI screens using Bitrix24 Mobile SDK
- Extend existing mobile app with "Technician Job Management" module
- Leverage Bitrix24 mobile components (lists, forms, buttons)

**Fallback Option (if Bitrix24 Mobile insufficient):**
- Build standalone React Native app with Bitrix24 REST API integration
- Only if Bitrix24 Mobile SDK cannot support required features
- Decision point: After evaluating Bitrix24 Mobile SDK capabilities

### API Endpoints

```javascript
// Get technician's assigned jobs
GET /api/scheduling/technician/{user_id}/assigned-jobs
  Response: {
    jobs: [{
      id: "12345",
      deal_id: "67890",
      sequence_number: 1,
      address: "123 Main St, Atlanta, GA",
      client_name: "Jane Doe",
      client_phone: "+1-555-1234",
      job_type: "Lock Installation",
      scheduled_time: "2025-01-15T09:00:00Z",
      status: "assigned",
      estimated_value: 250,
      notes: "Front door lock, bring deadbolt",
      required_items: ["Deadbolt Lock", "Keys (x3)"]
    }]
  }

// Update job status
POST /api/scheduling/job/{job_id}/update-status
  Body: {
    status: "in_progress" | "completed",
    timestamp: "2025-01-15T09:15:00Z",
    user_id: "tech_123"
  }
  Response: {
    success: true,
    deal_updated: true,
    new_status: "in_progress"
  }

// Start GPS tracking
POST /api/scheduling/location/start
  Body: {
    user_id: "tech_123",
    device_id: "device_abc"
  }
  Response: {
    tracking_enabled: true,
    update_interval: 30
  }

// Send location update
POST /api/scheduling/location/update
  Body: {
    user_id: "tech_123",
    latitude: 33.7490,
    longitude: -84.3880,
    accuracy: 10, // meters
    timestamp: "2025-01-15T09:15:30Z"
  }
  Response: {
    success: true
  }

// Stop GPS tracking
POST /api/scheduling/location/stop
  Body: {
    user_id: "tech_123"
  }
  Response: {
    tracking_enabled: false
  }
```

```sql
  ID (int primary key)

```

### Background Location Service

**iOS Implementation:**
```javascript
// React Native - iOS Background Location
import BackgroundGeolocation from 'react-native-background-geolocation';

BackgroundGeolocation.configure({
  desiredAccuracy: BackgroundGeolocation.DESIRED_ACCURACY_HIGH,
  distanceFilter: 50, // meters
  stopOnTerminate: false,
  startOnBoot: true,
  interval: 30000, // 30 seconds
  fastestInterval: 30000,
});

BackgroundGeolocation.on('location', (location) => {
  sendLocationToServer(location);
});
```

**Android Implementation:**
```javascript
// Android Foreground Service (to prevent OS from killing GPS)
import ForegroundService from 'react-native-foreground-service';

ForegroundService.startService({
  channelId: 'locksmith_gps',
  id: 1,
  title: 'GPS Tracking Active',
  message: 'Sharing location with dispatcher',
  icon: 'ic_gps',
});
```

## Edge Cases & Error Handling

### GPS Tracking Failures

**Problem:** GPS signal unavailable (indoor job, basement, parking garage)

**Solution:**
1. Use last known location
2. Show "GPS signal weak" indicator
3. Attempt to get coarse location from cell tower
4. Queue location updates, send when signal restored

### Location Permission Denied

**Problem:** User denies location permission

**Solution:**
1. Show in-app message: "Location permission required for GPS tracking"
2. Provide [Open Settings] button to re-enable permission
3. Allow app to function without GPS (limited functionality)
4. Notify dispatcher: "Technician GPS disabled"

### Status Update Conflicts

**Problem:** Technician marks job "In Progress" but dispatcher reassigned it

**Solution:**
1. Server returns 409 Conflict error
2. App shows: "This job was reassigned. Refreshing job list..."
3. Force refresh job list from server
4. Remove conflicted job from technician's list

### Battery Optimization Killing Background Service

**Problem:** Android battery optimizer kills GPS background service

**Solution:**
1. Request battery optimization exemption on app install
2. Show notification: "GPS Tracking Active" (prevents service kill)
3. Use Android Foreground Service for GPS tracking
4. Detect when service stopped, show in-app warning

### Offline Mode Limitations

**Problem:** Technician offline for extended period (>1 hour), many queued updates

**Solution:**
1. Store up to 100 queued updates locally
2. When online, batch-sync all updates in chronological order
3. If server rejects update (stale data), show conflict resolution dialog
4. Allow technician to review and confirm/discard stale updates
