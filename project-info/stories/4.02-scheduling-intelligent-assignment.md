# Story 4.02: Intelligent Job Assignment

**Epic:** [EPIC 4: Scheduling & Dispatching](../epics/EPIC-4-Scheduling.md)
**Story Points:** 13
**Priority:** High
**Complexity:** Medium (Filtering logic + integrations)

---

## User Story

**As a** Shift Manager/Dispatcher
**I want** the system to show me only qualified technicians for each job (matching skills and metro area)
**So that** I can quickly assign jobs to the right technicians without manually checking who has the required skills and is in the right area

---

## Business Context

**Current Pain Point:** Dispatchers manually evaluate 15-20 technicians for each job assignment:
1. Open technician list in Workiz
2. Check if tech has required job type skill (not tracked consistently)
3. Check if tech is assigned to the right metro area
4. Call/text tech to confirm availability

This process takes 3-5 minutes per assignment and often results in suboptimal matches (wrong skills or wrong area).

**Desired Outcome:**
- System shows ONLY technicians who meet basic criteria:
  - ✅ Has required job type skill
  - ✅ Assigned to job's metro area
- Dispatcher sees filtered list of 5-10 qualified techs
- Dispatcher clicks [Assign] button → Done in 30 seconds

**Success Metrics:**
- Assignment time: <30 seconds (vs 3-5 minutes currently)
- Zero "wrong skill/area" assignments
- Dispatcher confidence in system filtering

## Acceptance Criteria

### 1. Simple Filtering Logic

**Job Requirements:**
- Job Type (single value): "Locksmith", "Installation", "Rekey", "Lockout", etc.
  - Job has exactly ONE job type
  - Job Type IS a skill from predefined list
- Metro Area (single value): "Atlanta", "North GA", "East Atlanta", etc.
- Required Items (optional, rarely used): ["KB-100", "KB-200"]

**Technician Must Match:**
1. **Job Type Match (REQUIRED):**
   - Tech's skills list contains the job's job type
   - Job Type = "Lockout" → Check if tech has "Lockout" in their skills
   - If tech's skills include this job type → SHOW tech ✅
   - If tech's skills DON'T include this job type → EXCLUDE tech ❌

2. **Metro Area Match (REQUIRED):**
   - Tech's assigned metro areas include job's metro area
   - If job is in "Atlanta" → Tech must have "Atlanta" in their areas
   - If tech not in area → EXCLUDE from list

3. **Items Check (OPTIONAL - show warning if missing):**
   - If job has required items, check tech's container inventory
   - If missing → Still show tech, but display warning icon
   - Don't exclude tech based on inventory alone

**Example:**

```
Job:
  - Type: "Lockout"  ← This is ONE skill from the skills list
  - Metro Area: "Atlanta"
  - Items: ["KB-100"] (optional)

Tech A:
  - Skills: ["Locksmith", "Lockout", "Rekey"]  ← Has "Lockout"
  - Areas: ["Atlanta", "North GA"]
  - Inventory: [KB-100: 3 units]
  → SHOW (has "Lockout" skill + in "Atlanta" area) ✅

Tech B:
  - Skills: ["Lockout", "Installation"]  ← Has "Lockout"
  - Areas: ["Atlanta"]
  - Inventory: [No KB-100]
  → SHOW (has skill + in area, show warning about missing item) ⚠️

Tech C:
  - Skills: ["Locksmith", "Rekey"]  ← Does NOT have "Lockout"
  - Areas: ["Atlanta"]
  → HIDE (missing "Lockout" skill) ❌

Tech D:
  - Skills: ["Lockout"]  ← Has "Lockout"
  - Areas: ["North GA", "East Atlanta"]  ← NOT in "Atlanta"
  → HIDE (wrong metro area) ❌
```

**Key Point:** Job Type is NOT a combination of skills. It's a single skill value. The job either requires "Lockout" OR "Rekey" OR "Installation", never multiple at once.

### 3. One-Click Assignment Workflow

**When dispatcher clicks [Assign to John]:**

```
Step 1: Validate Assignment
  → Check if job still unassigned (concurrent edit check)
  → If validation fails, show error and refresh list

Step 2: Update Deal
  → Set Deal field "ASSIGNED_TO" = tech_user_id
  → Change Deal status: "SUBMITTED" → "ASSIGNED"
  → Calculate sequence number (see Time-Based Sequencing below)

Step 3: Send Notifications
  → Push notification to technician's mobile app
  → SMS backup: "New job assigned: Lockout at 123 Main St. Scheduled 2:00 PM"
  → Notification includes: Job details, address, client phone, sequence number

Step 4: Update Map
  → Job pin changes color (red → tech's color)
  → Add sequence number badge [3] to pin
  → Technician's "current jobs" count updates: [2 of 4] → [3 of 5]

Step 5: Log Activity
  → Create activity log entry:
    - "Job #12345 assigned to John Smith by Dispatcher Jane"
    - Timestamp, job type, metro area

Step 6: Close Sidebar
  → Show success toast: "Job #12345 assigned to John Smith ✅"
  → Sidebar closes automatically after 2 seconds
```

**Error Handling:**
- If assignment fails (network error, validation error), show error message
- Allow dispatcher to retry or choose different technician
- Don't close sidebar until success confirmed

### 4. Time-Based Job Sequencing

**Automatic Sequencing Logic:**

When job is assigned, system calculates sequence number based on scheduled time:

```javascript
function calculateSequenceNumber(technicianId, scheduledTime) {
  // Get all assigned jobs for this technician (status: ASSIGNED or IN_PROGRESS)
  const assignedJobs = getAssignedJobs(technicianId);

  // Sort by scheduled time (earliest first)
  assignedJobs.sort((a, b) => a.scheduled_time - b.scheduled_time);

  // Find position for new job
  let sequenceNumber = 1;
  for (const job of assignedJobs) {
    if (scheduledTime < job.scheduled_time) {
      break; // Insert before this job
    }
    sequenceNumber++;
  }

  // Update sequence numbers for all jobs (renumber if needed)
  renumberTechnicianJobs(technicianId);

  return sequenceNumber;
}
```

**Example:**

Technician John has 3 existing jobs:
- [1] Job A - 9:00 AM
- [2] Job B - 11:00 AM
- [3] Job C - 3:00 PM

New job assigned with scheduled time: 1:00 PM

System calculates:
- New job scheduled after Job B (11 AM) and before Job C (3 PM)
- Assign sequence number [3]
- Renumber Job C: [3] → [4]

Final sequence:
- [1] Job A - 9:00 AM
- [2] Job B - 11:00 AM
- [3] **New Job - 1:00 PM** ← Inserted here
- [4] Job C - 3:00 PM

## Technical Requirements

### Filtering Algorithm Implementation

```javascript
function getQualifiedTechnicians(job) {
  // Get all active technicians
  const allTechnicians = await db.query(
    "SELECT * FROM b_user WHERE ACTIVE='Y' AND UF_ROLE='technician'"
  );

  const qualified = [];
  const excluded = {
    noSkill: [],
    wrongArea: []
  };

  for (const tech of allTechnicians) {
    const techSkills = tech.UF_JOB_TYPES || []; // Multi-select field
    const techAreas = tech.UF_SERVICE_AREAS || []; // Multi-select field

    // Check 1: Job Type Match (REQUIRED)
    const hasRequiredSkill = techSkills.includes(job.job_type);
    if (!hasRequiredSkill) {
      excluded.noSkill.push(tech);
      continue; // Skip this tech
    }

    // Check 2: Metro Area Match (REQUIRED)
    const inServiceArea = techAreas.includes(job.metro_area);
    if (!inServiceArea) {
      excluded.wrongArea.push(tech);
      continue; // Skip this tech
    }

    // Tech qualifies! Add to list
    // Check inventory (optional - for display only)
    const inventory = await getTechnicianInventory(tech.id);
    const hasRequiredItems = job.required_items?.every(itemSku =>
      inventory[itemSku] && inventory[itemSku].quantity > 0
    );

    qualified.push({
      id: tech.id,
      name: tech.name,
      skills: techSkills,
      areas: techAreas,
      current_job_count: await getAssignedJobCount(tech.id),
      has_required_items: hasRequiredItems || job.required_items?.length === 0,
      missing_items: hasRequiredItems ? [] : job.required_items
    });
  }

  // Sort by current job count (ascending) - techs with fewer jobs first
  qualified.sort((a, b) => a.current_job_count - b.current_job_count);

  return {
    qualified: qualified,
    excluded: excluded,
    total_evaluated: allTechnicians.length
  };
}
```

## Edge Cases & Error Handling

### No Qualified Technicians

**Problem:** No technicians match job type + metro area criteria

**Solution:**
1. Show message: "⚠️ No technicians match this job's requirements."
2. Breakdown:
   - "Job Type: Lockout - 0 technicians have this skill"
   - "Metro Area: Atlanta - 5 technicians in this area"
3. Automatically show [All Technicians] list for manual override
4. Dispatcher can assign to ANY technician with confirmation

### Concurrent Assignment Conflicts

**Problem:** Two dispatchers assign same technician to overlapping time slots

**Solution:**
1. Lock job during assignment (optimistic locking)
2. If another dispatcher assigns first, show error: "This job was just assigned by [Dispatcher Name]"
3. Refresh job list automatically

### Technician Declines Assignment

**Problem:** Technician opens mobile app and clicks [Decline Job]

**Solution:**
1. Job reverts to "UNASSIGNED" status
2. Notification sent to dispatcher: "John Smith declined Job #12345. Reason: [reason]"
3. Job reappears on dispatch map as red pin
4. Dispatcher can reassign to different technician

### Sequence Number Conflicts

**Problem:** Multiple jobs assigned simultaneously, sequence numbers collide

**Solution:**
1. Use database transaction with row locking
2. Renumber all jobs for technician after each assignment
3. Send updated sequence numbers to mobile app

### Manual Override Tracking

**Problem:** Need to track when dispatcher assigns job to unqualified technician

**Solution:**
1. Set `MANUAL_OVERRIDE = true` in assignment record
2. Log override reason in audit log
3. Show badge on job card: "⚠️ Manual Assignment"
4. Generate weekly report of manual overrides for management review
