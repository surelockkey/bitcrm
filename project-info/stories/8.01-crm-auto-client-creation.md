# Story 8.01: CRM - Automatic Client Creation from Calls

---

## Overview

This story covers the automatic creation of Company and Contact entities in Bitrix24 CRM when a Deal is created from an inbound call. The system must intelligently detect if a client already exists (by phone number) and create new entities only when necessary, while maintaining data integrity and preventing duplicates.

---

## Business Context

**Current Pain Point:**
When a dispatcher receives a call and creates a Deal, they must manually search for and create client records, which:
- Wastes 30-60 seconds per call
- Causes data entry errors
- Creates duplicate clients if search fails
- Interrupts conversation flow with client

**Solution:**
Automatic client creation with phone number deduplication, seamlessly integrated into the call-to-Deal workflow from [Story 1.01: Inbound Call Routing](1.01-telephony-inbound-calls.md).

**Expected Outcome:**
Zero manual client search/creation, automatic deduplication, and seamless Deal-to-Client linking.

---

## User Personas

### Dispatcher
- **Goal:** Quickly create Deal during active call without client lookup delays
- **Pain:** Manual client search interrupts call flow, unsure if client already exists
- **Need:** Automatic client detection and creation, zero friction

### Manager
- **Goal:** Ensure clean client database, no duplicates
- **Pain:** Duplicate clients created during busy periods
- **Need:** Automatic deduplication by phone, audit trail of client creation

## Functional Requirements

### 1. Phone Number Lookup Logic

**Trigger:** When dispatcher creates Deal from active call (see [Story 1.01: Deal Creation Flow](1.01-telephony-inbound-calls.md#3-deal-creation-flow-during-call))

**Lookup Sequence:**
```
Step 1: Extract phone number from call metadata
  Phone: +1 (555) 123-4567
  ↓
Step 2: Search CRM for existing Contact
  Query: b_crm_contact WHERE PHONE = '+15551234567'
  (normalized format: strip spaces, dashes, parentheses)
  ↓
Step 3A: Contact FOUND
  → Load existing Contact
  → Display name in Deal form (read-only)
  → Link Deal to existing Contact
  → Skip client creation
  ↓
Step 3B: Contact NOT FOUND
  → Mark as "New Client"
  → Display empty form fields
  → Prepare for auto-creation on Deal save
```

**Phone Normalization Rules:**
```javascript
// Input formats accepted:
+1 (555) 123-4567
555-123-4567
5551234567
1-555-123-4567

// Normalized output:
+15551234567

// Logic:
1. Strip: spaces, dashes, parentheses
2. Add +1 if missing (US default)
3. Store in normalized format
4. Display in friendly format: (555) 123-4567
```

```
│ Last Name:  [Smith______________]         │ ← REQUIRED
│ Company:    [Smith Consulting___]         │ ← OPTIONAL
│ Email:      [john@example.com___]         │ ← OPTIONAL
│ Address:    [123 Main St________]         │ ← For Deal, not Contact
│                                            │

└────────────────────────────────────────────┘
```

**Auto-Creation Logic:**

```
On [Save & Create Deal] clicked:
  ↓
If phone NOT found in CRM:
  ↓
  CREATE CONTACT:
    First Name: "John"
    Last Name: "Smith"
    Phone: +15551234567 (normalized)
    Email: john@example.com (if provided)
    Company ID: [created company ID, if company name provided]
    Source: "Phone Call"
    Created By: [Dispatcher User ID]
    Date Created: NOW()
  ↓
  If Company name provided ("Smith Consulting"):
    ↓
    CREATE COMPANY:
      Title: "Smith Consulting"
      Phone: +15551234567 (same as contact)
      Client Type: [NULL, will be set later by dispatcher/manager]
      Created By: [Dispatcher User ID]
      Date Created: NOW()
    ↓
    LINK Contact → Company:
      Contact.COMPANY_ID = Company.ID
  ↓
  CREATE DEAL:
    Contact ID: [new contact ID]
    Company ID: [new company ID, if created]
    Phone: +15551234567
    Address: "123 Main St" (Deal-specific field)
    Job Type: [from form]
    Job Source: [from call metadata]
    Stage: SUBMITTED
    Created By: [Dispatcher User ID]
  ↓
Success: Deal created with auto-created client
```

### 4. Client Type Assignment

**Rule:** Client Type defaults to **Residential** during auto-creation.

**Reasoning:**
- Most clients are residential (majority use case)
- Prevents NULL values that could break workflows
- Manager can change type if needed after review

**Workflow:**
```
Step 1: Auto-create client (Company + Contact)
  Client Type: "Residential" (default)
  ↓
Step 2: Manager reviews new clients daily (if needed)
  Opens "New Clients" report
  Changes type if applicable:
    - Residential (default, no change needed)
    - Commercial
    - Government
  ↓
Step 3: Type determines features
  Commercial/Government → Platinum workflow enabled
  Residential → Standard workflow
```

**UI Indicator:**
```
Company Card Header:
┌────────────────────────────────────┐
│ ABC Supply Co.                     │
│ Client Type: Residential (default) │
│ [Change Type: Commercial ▼]       │
└────────────────────────────────────┘
```

### 5. Duplicate Prevention

**Primary Key:** Phone number (normalized)

**Duplicate Detection:**
```sql
-- Before creating Contact
SELECT ID FROM b_crm_contact
WHERE PHONE = '+15551234567'
LIMIT 1;

-- If found: Use existing
-- If not found: Create new
```

**Edge Case: Multiple people, same phone**

**Example:** Husband and wife share phone number

```
Scenario:
  First call: John Doe calls → Contact created
  Second call: Jane Doe calls (same phone)

System behavior:
  Pop-up shows: "Existing Client: John Doe"
  Dispatcher notices: "But this is Jane Doe (wife)"

Solution:
  Dispatcher uses "Contact Name for this Deal" field:
    Client (in CRM): John Doe [read-only]
    Contact Name for this Deal: Jane Doe

Result:
  - Deal linked to John Doe (Contact)
  - Deal notes: "Contact: Jane Doe (wife)"
  - Manager can manually create separate Contact later
```

**Note:** Advanced contact relationship management (multiple contacts per phone) is out of scope for initial implementation.

### 2. Phone Normalization Function

```php
// File: /local/php_interface/include/crm_phone_normalizer.php

namespace Local\Crm;

class PhoneNormalizer {

    /**
     * Normalize phone number to E.164 format
     * Input: "(555) 123-4567", "555-123-4567", "5551234567"
     * Output: "+15551234567"
     */
    public static function normalize($phone) {
        // Strip all non-digits
        $digits = preg_replace('/[^0-9]/', '', $phone);

        // If starts with 1, keep it; otherwise add 1 (US default)
        if (strlen($digits) == 10) {
            $digits = '1' . $digits;
        }

        // Add + prefix
        return '+' . $digits;
    }

    /**
     * Format for display
     * Input: "+15551234567"
     * Output: "(555) 123-4567"
     */
    public static function formatDisplay($phone) {
        $digits = preg_replace('/[^0-9]/', '', $phone);

        // US format: (555) 123-4567
        if (strlen($digits) == 11 && $digits[0] == '1') {
            $digits = substr($digits, 1); // remove leading 1
        }

        if (strlen($digits) == 10) {
            return sprintf(
                "(%s) %s-%s",
                substr($digits, 0, 3),
                substr($digits, 3, 3),
                substr($digits, 6, 4)
            );
        }

        return $phone; // fallback
    }
}
```

### 3. Client Lookup Logic

```php
// File: /local/php_interface/include/crm_client_lookup.php

namespace Local\Crm;

use Bitrix\Crm\ContactTable;
use Bitrix\Crm\CompanyTable;

class ClientLookup {

    /**
     * Find existing contact by phone number
     * Returns Contact ID if found, null otherwise
     */
    public static function findContactByPhone($phone) {
        $normalized = PhoneNormalizer::normalize($phone);

        // Search in Contact phone fields
        $contact = ContactTable::getList([
            'filter' => ['PHONE' => $normalized],
            'select' => ['ID', 'NAME', 'LAST_NAME', 'COMPANY_ID'],
            'limit' => 1
        ])->fetch();

        if ($contact) {
            return [
                'found' => true,
                'contact_id' => $contact['ID'],
                'name' => $contact['NAME'] . ' ' . $contact['LAST_NAME'],
                'company_id' => $contact['COMPANY_ID'] ?? null
            ];
        }

        return ['found' => false];
    }

    /**
     * Find existing company by exact name match
     */
    public static function findCompanyByName($companyName) {
        if (empty($companyName)) {
            return ['found' => false];
        }

        $company = CompanyTable::getList([
            'filter' => ['TITLE' => $companyName],
            'select' => ['ID', 'TITLE'],
            'limit' => 1
        ])->fetch();

        if ($company) {
            return [
                'found' => true,
                'company_id' => $company['ID'],
                'title' => $company['TITLE']
            ];
        }

        return ['found' => false];
    }
}
```

### 4. Auto-Creation Handler

```php
// File: /local/php_interface/include/crm_auto_create_client.php

namespace Local\Crm;

use Bitrix\Crm\ContactTable;
use Bitrix\Crm\CompanyTable;
use Bitrix\Crm\DealTable;

class AutoCreateClient {

    /**
     * Main entry point: Create client entities from Deal form data
     *
     * @param array $formData - Data from Deal creation form
     * @return array - ['contact_id' => int, 'company_id' => int|null]
     */
    public static function createFromDealData($formData) {
        $phone = PhoneNormalizer::normalize($formData['phone']);

        // Check if contact already exists
        $lookup = ClientLookup::findContactByPhone($phone);
        if ($lookup['found']) {
            return [
                'contact_id' => $lookup['contact_id'],
                'company_id' => $lookup['company_id'],
                'created' => false
            ];
        }

        // Create new client
        $companyId = null;

        // Step 1: Create Company (if company name provided)
        if (!empty($formData['company_name'])) {
            $companyId = self::createCompany($formData);
        }

        // Step 2: Create Contact
        $contactId = self::createContact($formData, $companyId);

        return [
            'contact_id' => $contactId,
            'company_id' => $companyId,
            'created' => true
        ];
    }

    /**
     * Create Company entity
     */
    private static function createCompany($formData) {
        global $USER;

        $phone = PhoneNormalizer::normalize($formData['phone']);

        // Check if company with this name already exists
        $existing = ClientLookup::findCompanyByName($formData['company_name']);
        if ($existing['found']) {
            return $existing['company_id']; // use existing
        }

        $company = new \CCrmCompany();
        $companyId = $company->Add([
            'TITLE' => $formData['company_name'],
            'PHONE' => [
                ['VALUE' => $phone, 'VALUE_TYPE' => 'WORK']
            ],
            'EMAIL' => !empty($formData['email']) ? [
                ['VALUE' => $formData['email'], 'VALUE_TYPE' => 'WORK']
            ] : [],
            'UF_CRM_CLIENT_TYPE' => 'residential', // default to residential
            'ASSIGNED_BY_ID' => $USER->GetID(),
            'OPENED' => 'Y'
        ]);

        if (!$companyId) {
            throw new \Exception('Failed to create company: ' . $company->LAST_ERROR);
        }

        return $companyId;
    }

    /**
     * Create Contact entity
     */
    private static function createContact($formData, $companyId = null) {
        global $USER;

        $phone = PhoneNormalizer::normalize($formData['phone']);

        $contact = new \CCrmContact();
        $contactId = $contact->Add([
            'NAME' => $formData['first_name'],
            'LAST_NAME' => $formData['last_name'],
            'PHONE' => [
                ['VALUE' => $phone, 'VALUE_TYPE' => 'WORK']
            ],
            'EMAIL' => !empty($formData['email']) ? [
                ['VALUE' => $formData['email'], 'VALUE_TYPE' => 'WORK']
            ] : [],
            'COMPANY_ID' => $companyId,
            'UF_SOURCE' => 'Phone Call',
            'ASSIGNED_BY_ID' => $USER->GetID(),
            'OPENED' => 'Y'
        ]);

        if (!$contactId) {
            throw new \Exception('Failed to create contact: ' . $contact->LAST_ERROR);
        }

        return $contactId;
    }
}
```

### 5. Deal Creation Integration

```php
// File: /local/php_interface/include/deal_create_with_client.php

namespace Local\Crm;

use Bitrix\Crm\DealTable;

class DealCreateWithClient {

    /**
     * Create Deal with automatic client creation
     * This is called from telephony Deal creation form (Story 1.01)
     *
     * @param array $dealData - Deal form data
     * @return int - Deal ID
     */
    public static function create($dealData) {
        global $USER;

        // Step 1: Auto-create or find client
        $client = AutoCreateClient::createFromDealData([
            'phone' => $dealData['phone'],
            'first_name' => $dealData['first_name'],
            'last_name' => $dealData['last_name'],
            'company_name' => $dealData['company_name'] ?? null,
            'email' => $dealData['email'] ?? null
        ]);

        // Step 2: Create Deal
        $deal = new \CCrmDeal();
        $dealId = $deal->Add([
            'TITLE' => self::generateDealTitle($dealData),
            'CONTACT_ID' => $client['contact_id'],
            'COMPANY_ID' => $client['company_id'],
            'PHONE' => [
                ['VALUE' => $dealData['phone'], 'VALUE_TYPE' => 'WORK']
            ],
            'UF_CONTACT_PERSON' => $dealData['contact_name_for_deal'] ?? null,
            'ADDRESS' => self::formatAddress($dealData),
            'UF_CRM_JOB_TYPE' => $dealData['job_type'],
            'UF_CRM_JOB_SOURCE' => $dealData['job_source'] ?? 'Phone Call',
            'UF_CRM_GCLID' => $dealData['gclid'] ?? null,
            'UF_CRM_METRO_AREA' => $dealData['metro_area'] ?? null,
            'STAGE_ID' => 'SUBMITTED',
            'COMMENTS' => $dealData['notes'] ?? '',
            'ASSIGNED_BY_ID' => $USER->GetID(),
            'OPENED' => 'Y'
        ]);

        if (!$dealId) {
            throw new \Exception('Failed to create deal: ' . $deal->LAST_ERROR);
        }

        // Step 3: Link call recording to Deal timeline (if call_id provided)
        if (!empty($dealData['call_id'])) {
            self::linkCallToDeal($dealData['call_id'], $dealId);
        }

        return $dealId;
    }

    /**
     * Generate Deal title
     */
    private static function generateDealTitle($data) {
        $jobType = $data['job_type'] ?? 'Service';
        $contactName = $data['contact_name_for_deal'] ??
                       ($data['first_name'] . ' ' . $data['last_name']);

        return $jobType . ' for ' . $contactName;
    }

    /**
     * Format address from separate fields
     */
    private static function formatAddress($data) {
        return sprintf(
            "%s, %s, %s %s",
            $data['street'] ?? '',
            $data['city'] ?? '',
            $data['state'] ?? '',
            $data['zip'] ?? ''
        );
    }

    /**
     * Link call recording to Deal timeline
     */
    private static function linkCallToDeal($callId, $dealId) {
        // Add timeline activity
        \CCrmActivity::Add([
            'OWNER_TYPE_ID' => \CCrmOwnerType::Deal,
            'OWNER_ID' => $dealId,
            'TYPE_ID' => \CCrmActivityType::Call,
            'SUBJECT' => 'Inbound call',
            'PROVIDER_ID' => 'VOXIMPLANT_CALL',
            'PROVIDER_TYPE_ID' => 'CALL',
            'ASSOCIATED_ENTITY_ID' => $callId,
            'COMPLETED' => 'Y'
        ]);
    }
}
```

## Acceptance Criteria

### Functional Acceptance

- ✅ **Phone lookup:** System searches CRM by normalized phone before Deal creation
- ✅ **Existing client:** If found, Deal links to existing Contact (no duplicate)
- ✅ **New client:** If not found, Contact auto-created on Deal save
- ✅ **Company creation:** If company name provided, Company entity created and linked
- ✅ **Company reuse:** If company name matches existing, Contact linked to existing Company
- ✅ **Required fields:** First Name, Last Name, Phone validated before save
- ✅ **Phone normalization:** All formats normalized to +1XXXXXXXXXX
- ✅ **Source tracking:** Job Source and gclid captured from call metadata
- ✅ **Timeline integration:** Call recording linked to Deal timeline
- ✅ **No duplicates:** Same phone number never creates duplicate Contact

### UI/UX Acceptance

- ✅ **Clear indicators:** "Existing Client" vs. "New Client" clearly displayed
- ✅ **Read-only fields:** Client lookup result NOT editable during call
- ✅ **Validation errors:** Clear error messages for missing required fields
- ✅ **Success feedback:** Confirmation shown after Deal + Client created
- ✅ **Manager reminder:** Warning shown to set Client Type for new clients

### Data Integrity Acceptance

- ✅ **Phone uniqueness:** No duplicate Contacts with same phone
- ✅ **Relationship integrity:** Contact → Company FK maintained
- ✅ **Deal linking:** Deal always linked to valid Contact
- ✅ **Audit trail:** All creations logged with user ID and timestamp
