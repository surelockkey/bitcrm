## 1. Overview

### 1.1. Purpose

Implement Tap to Pay functionality for technicians to accept contactless payments on-site using their iPhone (iOS) or Android phone. This eliminates the need for external card readers and enables:
- Contactless NFC card payments
- Apple Pay / Google Pay acceptance
- Instant payment processing at customer location
- Automatic sync with Bitrix24 Deal
- Digital receipt generation

### 1.2. Business Requirements

**Why Tap to Pay?**

Traditional payment acceptance requires:
- ❌ External card reader hardware ($50-300 per device)
- ❌ Reader charging and maintenance
- ❌ Bluetooth pairing issues
- ❌ Lost/broken readers

**Tap to Pay Benefits:**
- ✅ No additional hardware (uses iPhone built-in NFC)
- ✅ Lower fees: 2.9% + $0.30 (same as online)
- ✅ Faster checkout (2-3 seconds)
- ✅ Professional appearance
- ✅ Works offline (transactions sync when online)

**Supported Devices:**
- **iOS:** iPhone XS or newer (iOS 15.4+)
- **Android:** NFC-enabled devices with Android 9.0+
- Both devices must have Bitrix24 Mobile App installed

```swift
import StripeTerminal

class BitrixStripeTokenProvider: NSObject, ConnectionTokenProvider {
    static let shared = BitrixStripeTokenProvider()

    private override init() {
        super.init()
    }

    func fetchConnectionToken(_ completion: @escaping ConnectionTokenCompletionBlock) {
        // Call Bitrix24 API to get Stripe Terminal connection token
        let url = "\(BitrixConfig.baseURL)/local/api/stripe/terminal/connection-token"

        NetworkManager.shared.post(url: url) { result in
            switch result {
            case .success(let data):
                if let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any],
                   let token = json["secret"] as? String {
                    completion(token, nil)
                } else {
                    let error = NSError(domain: "com.bitrix.stripe",
                                      code: 1001,
                                      userInfo: [NSLocalizedDescriptionKey: "Invalid token response"])
                    completion(nil, error)
                }
            case .failure(let error):
                completion(nil, error)
            }
        }
    }
}
```

#### 3.1.5. Process Payment

File: `/bitrix-mobile-ios/ViewControllers/PaymentViewController.swift` (continued)

```swift
extension PaymentViewController {

    func processPayment(dealId: Int, amount: Double) {
        guard connectedReader != nil else {
            showError("No reader connected")
            return
        }

        // Show loading
        updateUI(status: "Creating payment intent...")

        // Step 1: Create Payment Intent on server
        createPaymentIntent(dealId: dealId, amount: amount) { paymentIntentSecret in

            // Step 2: Retrieve Payment Intent from Stripe
            Terminal.shared.retrievePaymentIntent(clientSecret: paymentIntentSecret) { retrievedIntent, error in
                if let error = error {
                    self.showError("Failed to retrieve payment intent: \(error.localizedDescription)")
                    return
                }

                guard let paymentIntent = retrievedIntent else {
                    self.showError("No payment intent returned")
                    return
                }

                // Step 3: Collect payment method (NFC tap)
                self.updateUI(status: "Ready to tap card")

                Terminal.shared.collectPaymentMethod(paymentIntent) { collectResult, error in
                    if let error = error {
                        self.showError("Failed to collect payment: \(error.localizedDescription)")
                        return
                    }

                    guard let collectedIntent = collectResult else {
                        self.showError("No payment intent after collection")
                        return
                    }

                    // Step 4: Process the payment
                    self.updateUI(status: "Processing payment...")

                    Terminal.shared.processPayment(collectedIntent) { processedIntent, error in
                        if let error = error {
                            self.showError("Payment failed: \(error.localizedDescription)")
                            return
                        }

                        guard let finalIntent = processedIntent else {
                            self.showError("No payment intent after processing")
                            return
                        }

                        // Step 5: Confirm payment on server
                        self.confirmPayment(paymentIntentId: finalIntent.stripeId, dealId: dealId)
                    }
                }
            }
        }
    }

    // MARK: - Server API Calls

    func createPaymentIntent(dealId: Int, amount: Double, completion: @escaping (String) -> Void) {
        let url = "\(BitrixConfig.baseURL)/local/api/stripe/terminal/create-payment-intent"

        let params: [String: Any] = [
            "deal_id": dealId,
            "amount": amount,
            "currency": "usd",
            "payment_method_types": ["card_present"],
            "capture_method": "automatic"
        ]

        NetworkManager.shared.post(url: url, parameters: params) { result in
            switch result {
            case .success(let data):
                if let json = try? JSONSerialization.jsonObject(with: data) as? [String: Any],
                   let clientSecret = json["client_secret"] as? String {
                    completion(clientSecret)
                } else {
                    self.showError("Invalid payment intent response")
                }
            case .failure(let error):
                self.showError("API error: \(error.localizedDescription)")
            }
        }
    }

    func confirmPayment(paymentIntentId: String, dealId: Int) {
        let url = "\(BitrixConfig.baseURL)/local/api/stripe/terminal/confirm-payment"

        let params: [String: Any] = [
            "payment_intent_id": paymentIntentId,
            "deal_id": dealId
        ]

        NetworkManager.shared.post(url: url, parameters: params) { result in
            switch result {
            case .success(_):
                // Payment successful!
                self.showSuccess(dealId: dealId, amount: self.paymentAmount)
            case .failure(let error):
                self.showError("Failed to confirm payment: \(error.localizedDescription)")
            }
        }
    }

    // MARK: - UI Updates

    func showSuccess(dealId: Int, amount: Double) {
        updateUI(status: "✅ Payment Successful")

        // Show success screen
        let successVC = PaymentSuccessViewController()
        successVC.dealId = dealId
        successVC.amount = amount
        successVC.modalPresentationStyle = .fullScreen
        present(successVC, animated: true)

        // Sync Deal data
        syncDeal(dealId: dealId)
    }

    func updateUI(status: String) {
        DispatchQueue.main.async {
            self.statusLabel.text = status
        }
    }

    func showError(_ message: String) {
        DispatchQueue.main.async {
            let alert = UIAlertController(title: "Payment Error",
                                        message: message,
                                        preferredStyle: .alert)
            alert.addAction(UIAlertAction(title: "OK", style: .default))
            self.present(alert, animated: true)
        }
    }
}
```

#### 3.2.3. Token Provider Implementation

File: `/bitrix-mobile-android/app/src/main/java/com/bitrix/mobile/payment/BitrixTokenProvider.kt`

```kotlin
import com.stripe.stripeterminal.external.callable.Cancelable
import com.stripe.stripeterminal.external.callable.ConnectionTokenCallback
import com.stripe.stripeterminal.external.callable.ConnectionTokenProvider
import okhttp3.*
import org.json.JSONObject
import java.io.IOException

class BitrixTokenProvider : ConnectionTokenProvider {

    override fun fetchConnectionToken(callback: ConnectionTokenCallback) {
        // Call Bitrix24 API to get connection token
        val url = "${BitrixConfig.BASE_URL}/local/api/stripe/terminal/connection-token"

        val request = Request.Builder()
            .url(url)
            .post(RequestBody.create(null, ByteArray(0)))
            .addHeader("Authorization", "Bearer ${BitrixConfig.getAccessToken()}")
            .build()

        OkHttpClient().newCall(request).enqueue(object : Callback {
            override fun onFailure(call: Call, e: IOException) {
                callback.onFailure(
                    com.stripe.stripeterminal.external.models.ConnectionTokenException(
                        "Failed to fetch connection token: ${e.message}"
                    )
                )
            }

            override fun onResponse(call: Call, response: Response) {
                try {
                    val responseBody = response.body?.string()
                    val json = JSONObject(responseBody ?: "")
                    val token = json.getString("secret")

                    callback.onSuccess(token)
                } catch (e: Exception) {
                    callback.onFailure(
                        com.stripe.stripeterminal.external.models.ConnectionTokenException(
                            "Failed to parse token: ${e.message}"
                        )
                    )
                }
            }
        })
    }
}
```

```javascript
            Alert.alert('Update Required', 'Please update to iOS 15.4 or later to use Tap to Pay');
            return;
        }
    }

    // Navigate to payment screen
    navigate('PaymentScreen', {
        dealId: dealId,
        amount: amount,
        paymentMethod: 'tap_to_pay'
    });
}
```

## 7. Troubleshooting

### 7.1. Common Issues

**Issue 1: "Tap to Pay not available on this device"**

**Cause:** iPhone model too old (earlier than XS)

**Solution:**
1. Check device: Settings → General → About → Model Name
2. Upgrade to iPhone XS or newer
3. Alternative: Use external card reader

---

**Issue 2: "Failed to connect to reader"**

**Cause:** Location permissions not granted

**Solution:**
1. iOS: Settings → Privacy → Location Services → Bitrix24 → Always
2. Android: Settings → Apps → Bitrix24 → Permissions → Location → Allow
3. Restart app

---

**Issue 3: "Connection token failed"**

**Cause:** Bitrix server unreachable or auth issue

**Solution:**
1. Check internet connection
2. Verify user is logged in to Bitrix24 app
3. Check server logs: `/local/logs/stripe-terminal.log`

---

**Issue 4: "Payment processing timeout"**

**Cause:** Poor network connection

**Solution:**
1. Switch from WiFi to cellular (or vice versa)
2. Move to area with better signal
3. Use offline mode (if amount <$100)
