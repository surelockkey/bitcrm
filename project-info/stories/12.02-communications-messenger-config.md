# Story 12.02: Messenger Configuration

**Epic:** [EPIC 12: Communications & Corporate Messenger Migration](../epics/EPIC-12-Communications.md)

---

## User Story

**As a** IT Administrator
**I want** to configure Bitrix24 messenger with all necessary features (chat types, file sharing, audio/video calls, call recording)
**So that** employees can communicate effectively with all required capabilities

---

## Acceptance Criteria

### 1. Enable Bitrix24 Messenger for All Users

**Given** Bitrix24 messenger module is installed
**When** configuring messenger access
**Then** the system must:

- ✅ Enable messenger module for all employee accounts
- ✅ Verify messenger access for each user role (Dispatcher, Technician, Manager, Admin)
- ✅ Configure default messenger settings (notifications, sounds, desktop app)
- ✅ Set messenger as default communication tool
- ✅ Verify mobile app messenger functionality

**Configuration Steps:**
```php
// Enable messenger for all users
$userGroups = [
    'DISPATCHER',
    'TECHNICIAN',
    'SHIFT_MANAGER',
    'ADMINISTRATOR'
];

foreach ($userGroups as $groupName) {
    $group = \CGroup::GetList(
        $by = 'NAME',
        $order = 'ASC',
        ['NAME' => $groupName]
    )->Fetch();

    if ($group) {
        // Enable IM module for group
        \COption::SetOptionString('im', "access_group_{$group['ID']}", 'Y');
    }
}
```

### 2. Configure Chat Types

**Given** messenger is enabled
**When** configuring chat types
**Then** the system must support:

- ✅ **Private Chats (1-on-1):** Enabled for all users
- ✅ **Group Chats:** Enabled with member management
- ✅ **Department Channels:** Configured for each office (Ukraine, Georgia, USA)
- ✅ **Activity Stream/Feed:** Enabled for company-wide announcements
- ✅ **Chat Creation Permissions:** Restricted based on role (see Epic 11)

**Chat Type Configuration:**

**Private Chats:**
```php
// Enable private chats
\COption::SetOptionString('im', 'allow_p2p_chat', 'Y');

// Allow all users to initiate private chats
\COption::SetOptionString('im', 'p2p_chat_restricted', 'N');
```

```php
// Enable group chats
```

### 3. Configure File Sharing and Storage

**Given** messenger is enabled
**When** configuring file storage
**Then** the system must:

- ✅ Link to Bitrix24 Drive as primary storage
- ✅ Set file upload limits (per file and per user)
- ✅ Configure file retention policies
- ✅ Enable file preview/online viewing
- ✅ Configure automatic file organization
- ✅ Set file access permissions

**File Storage Configuration:**

**Bitrix24 Drive Integration:**
```php
// Enable Disk module
\Bitrix\Main\Loader::includeModule('disk');

// Configure default storage for messenger files
\COption::SetOptionString('im', 'disk_storage_enabled', 'Y');

// Create dedicated folder structure for messenger files
$driver = \Bitrix\Disk\Driver::getInstance();
$storage = $driver->getStorageByUserId(1); // Admin storage

if ($storage) {
    $rootFolder = $storage->getRootObject();

    // Create "Messenger Files" folder
    $messengerFolder = $rootFolder->addSubFolder([
        'NAME' => 'Messenger Files',
        'CREATED_BY' => 1
    ]);

    // Create subfolders by chat type
    $subfolders = ['Private Chats', 'Group Chats', 'Department Channels'];
    foreach ($subfolders as $folderName) {
        $messengerFolder->addSubFolder([
            'NAME' => $folderName,
            'CREATED_BY' => 1
        ]);
    }
}
```

**File Upload Limits:**
```php
// Set file upload limits
\COption::SetOptionString('im', 'max_file_size', '104857600'); // 100 MB per file

// Set storage quota per user (if applicable)
\COption::SetOptionString('disk', 'default_quota', '10737418240'); // 10 GB per user
```

**File Retention Policies:**
```php
// Configure automatic cleanup of old files
\COption::SetOptionString('im', 'file_retention_days', '365'); // Keep files for 1 year

// Exclude important file types from automatic cleanup
\COption::SetOptionString('im', 'file_retention_exclude_types',
    'pdf,docx,xlsx,contract,invoice'
);
```

**File Preview Configuration:**
```php
// Enable online file preview
\COption::SetOptionString('disk', 'allow_file_preview', 'Y');

// Configure supported preview formats
$previewFormats = [
    'images' => ['jpg', 'jpeg', 'png', 'gif', 'bmp', 'webp'],
    'documents' => ['pdf', 'doc', 'docx', 'xls', 'xlsx', 'ppt', 'pptx'],
    'text' => ['txt', 'csv', 'json', 'xml', 'log']
];

\COption::SetOptionString('disk', 'preview_formats',
    json_encode($previewFormats)
);
```

### 5. Configure Call Recording

**Given** audio/video calls are enabled

- ✅ Enable automatic call recording (audio and video)

- ✅ Set recording access permissions (admin and participants)
- ✅ Configure retention period (30 days accessible, then archive)
- ✅ Enable recording transcription (if available)
- ✅ Configure recording notifications

**Call Recording Configuration:**

**Enable Recording:**
```php
// Enable automatic call recording
\COption::SetOptionString('im', 'call_recording_enabled', 'Y');

// Record all calls automatically
\COption::SetOptionString('im', 'call_recording_automatic', 'Y');

// Notify participants that call is being recorded
\COption::SetOptionString('im', 'call_recording_notification', 'Y');
```

**Recording Storage:**
```php

if ($storage) {
```

**Recording Access Permissions:**


**Automated Recording Archival Script:**
```php
<?php
/**
 * Scheduled task: Archive old call recordings
 * Run daily via cron
 */

require_once($_SERVER["DOCUMENT_ROOT"]."/bitrix/modules/main/include/prolog_before.php");

use Bitrix\Main\Loader;
use Bitrix\Disk\Driver;
use Bitrix\Disk\File;

Loader::includeModule('disk');

class CallRecordingArchiver
{
    public function archiveOldRecordings()
    {
        $retentionDays = \COption::GetOptionInt('im', 'recording_retention_active_days', 30);
        $cutoffDate = new DateTime();
        $cutoffDate->modify("-{$retentionDays} days");

        // Get recordings folder
        $driver = Driver::getInstance();
        $storage = $driver->getStorageByUserId(1);
        $recordingsFolder = $storage->getFolderByPath('/Call Recordings');

        if (!$recordingsFolder) {
            return;
        }

        // Get archive folder (create if not exists)
        $archiveFolder = $storage->getFolderByPath('/Call Recordings Archive');
        if (!$archiveFolder) {
            $rootFolder = $storage->getRootObject();
            $archiveFolder = $rootFolder->addSubFolder([
                'NAME' => 'Call Recordings Archive',
                'CREATED_BY' => 1
            ]);
        }

        // Find old recordings
        $files = $recordingsFolder->getChildren();
        $archivedCount = 0;

        foreach ($files as $file) {
            if (!($file instanceof File)) {
                continue;
            }

            $createDate = $file->getCreateTime();

            if ($createDate < $cutoffDate) {
                // Move to archive
                $file->moveTo($archiveFolder);
                $archivedCount++;
            }
        }

        echo "Archived {$archivedCount} recordings\n";
    }

    public function deleteExpiredArchives()
    {
        $archiveRetentionDays = \COption::GetOptionInt('im', 'recording_archive_retention_days', 365);
        $cutoffDate = new DateTime();
        $cutoffDate->modify("-{$archiveRetentionDays} days");

        // Get archive folder
        $driver = Driver::getInstance();
        $storage = $driver->getStorageByUserId(1);
        $archiveFolder = $storage->getFolderByPath('/Call Recordings Archive');

        if (!$archiveFolder) {
            return;
        }

        // Find expired archives
        $files = $archiveFolder->getChildren();
        $deletedCount = 0;

        foreach ($files as $file) {
            if (!($file instanceof File)) {
                continue;
            }

            $createDate = $file->getCreateTime();

            if ($createDate < $cutoffDate) {
                // Delete permanently
                $file->delete(1); // Delete by admin (user ID 1)
                $deletedCount++;
            }
        }

        echo "Deleted {$deletedCount} expired archives\n";
    }
}

// Run archival tasks
$archiver = new CallRecordingArchiver();
$archiver->archiveOldRecordings();
$archiver->deleteExpiredArchives();
```

```bash

```

## Risks & Mitigation

| Risk | Impact | Mitigation Strategy |
|------|--------|---------------------|
| **Call quality issues** | High - poor UX | - Test with different network conditions<br>- Configure adaptive bitrate<br>- Provide TURN server for NAT traversal<br>- Monitor call quality metrics |
| **Recording storage costs** | Medium - budget | - Set clear retention policies<br>- Automatic archival after 30 days<br>- Automatic deletion after 1 year<br>- Monitor storage usage |
| **Mobile app configuration** | Medium - user access | - Test mobile app on iOS and Android<br>- Verify push notifications work<br>- Document mobile setup process<br>- Provide employee training |
| **Permission configuration errors** | Medium - security | - Test each role's permissions<br>- Verify group creation restrictions<br>- Test recording access controls<br>- Audit permissions before go-live |
