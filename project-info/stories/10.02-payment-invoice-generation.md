# TR-11: Payment Integration - Invoice Generation & Payment Links

**Epic:** Payment Processing System
**Component:** Invoice Generation and Payment Link Management
**Version:** 1.0
**Last Updated:** January 16, 2025

---

## 1. Overview

### 1.1. Purpose

Implement comprehensive invoice generation system with:
- Professional PDF invoices matching Workiz design
- Automatic payment link generation
- Digital signature capture
- Multi-channel delivery (email, SMS, download)
- Integration with Stripe payment processing

### 1.2. Document Types

| Type | When Created | Client Action | Format |
|------|--------------|---------------|--------|
| **Estimate** | Before job | Review & Approve with signature | Web page (Client Portal) |
| **Invoice (Net-0)** | Job completion | Sign & Pay immediately | Web page (Client Portal) |
| **Invoice (Net-30)** | Job completion | Sign & Pay within 30 days | Web page (Client Portal) |
| **Invoice (Net-60)** | Job completion | Sign & Pay within 60 days | Web page (Client Portal) |

**Important:** Both Estimates and Invoices are delivered as **web pages** (Client Portal URLs), NOT as PDF attachments. PDF is generated only for download after client signs/pays.

## 3. Client Portal (Web Pages for Estimate & Invoice)

### 3.1. Architecture

**Unlike Workiz (SaaS client portal), Bitrix24 implementation uses:**
- Custom web pages hosted on Bitrix24 server
- Public URLs (no authentication required)
- Unique secure tokens for access control

```
Client receives link:
https://bitrix24.yourdomain.com/client-portal/invoice/abc123def456

Where:
  abc123def456 = Unique secure token (UUID)

Security:
  - Token is unique per document (Estimate or Invoice)
  - Token cannot be guessed (cryptographically secure)
  - No authentication required (client doesn't need account)
  - Token expires after document is signed/paid (optional)
```

### 3.4. Payment Modal

**Payment Method Selection Logic (Based on User Role):**

| User Role | Available Payment Methods | Default Selected | Can Select Multiple? |
|-----------|--------------------------|------------------|---------------------|
| **Technician** | Credit Card only | ✅ Credit Card | No (only one option) |
| **Dispatcher/Admin** | Credit Card + ACH + Sunbit | ✅ Credit Card | Yes (multiple allowed) |

**Why different permissions?**
- **Technicians:** Simplified workflow, CC provides instant payment confirmation
- **Dispatchers/Admins:** Full control, can offer multiple payment options to client

**Payment Method Costs:**

| Method | Company Fee | Processing Time | Client Experience |
|--------|-------------|-----------------|-------------------|
| **Credit Card** | 2.9% + $0.30 | Instant | Easy (Stripe hosted page) |
| **ACH** | 0.8% (max $5) | 3-5 business days | Medium (bank account entry) |
| **Sunbit** | 5% | Instant (after approval) | Complex (financing application) |

**When Dispatcher selects multiple methods:**
- Client sees all selected options in Client Portal
- Client chooses which method to use
- Example: If Dispatcher selects CC + ACH, client can choose either

**Sunbit Financing:**
- ❌ NOT offered in client portal directly
- ✅ Only dispatcher can send Sunbit financing link manually
- 5% commission (only used when client needs installment plan)

---

**Client Portal Payment Options Matrix:**

| Who Sent Invoice | Dispatcher Selected | Database Value | Client Sees in Portal | Pre-Selected? |
|------------------|---------------------|----------------|-----------------------|---------------|
| **Technician** | (Automatic) | `'card'` | Credit Card only | ✅ Yes |
| **Dispatcher** | CC only | `'card'` | Credit Card only | ✅ Yes |
| **Dispatcher** | CC + ACH | `'card,ach'` | Both CC and ACH | ❌ No (must choose) |
| **Dispatcher** | CC + Sunbit | `'card,sunbit'` | Both CC and Sunbit | ❌ No (must choose) |
| **Dispatcher** | ACH + Sunbit | `'ach,sunbit'` | Both ACH and Sunbit | ❌ No (must choose) |
| **Dispatcher** | All three | `'card,ach,sunbit'` | All three options | ❌ No (must choose) |

**Key Insight:**
- Single method = Pre-selected, client proceeds immediately
- Multiple methods = None pre-selected, client must choose one

### 3.4.3. Technical Implementation: Role-Based Payment Method Selection

**Permission Check (PHP):**

```php
// File: /local/php_interface/include/payment_permissions.php

namespace Local\Payment;

class PaymentPermissions {

    /**
     * Check if user can select ACH and Sunbit payment methods
     */
    public static function canSelectAdvancedPaymentMethods($userId) {
        global $USER;

        // Get user groups
        $userGroups = \CUser::GetUserGroup($userId);

        // Admin group ID (usually 1)
        if (in_array(1, $userGroups)) {
            return true;
        }

        // Dispatcher group ID (configure in Bitrix24)
        $dispatcherGroupId = \COption::GetOptionInt('payment', 'dispatcher_group_id', 5);
        if (in_array($dispatcherGroupId, $userGroups)) {
            return true;
        }

        // Technicians and other users: only CC
        return false;
    }

    /**
     * Get available payment methods for user
     */
    public static function getAvailablePaymentMethods($userId) {
        $methods = [
            'credit_card' => [
                'id' => 'card',
                'name' => 'Credit Card / Debit Card',
                'enabled' => true, // Always available
                'default' => true
            ]
        ];

        // Add ACH and Sunbit for dispatchers/admins
        if (self::canSelectAdvancedPaymentMethods($userId)) {
            $methods['ach'] = [
                'id' => 'ach',
                'name' => 'ACH Bank Transfer',
                'enabled' => true,
                'default' => false
            ];

            $methods['sunbit'] = [
                'id' => 'sunbit',
                'name' => 'Sunbit Financing',
                'enabled' => true,
                'default' => false
            ];
        }

        return $methods;
    }
}
```

```php
function sendInvoice() {
    const selectedMethods = [];
    document.querySelectorAll('input[name="payment_methods[]"]:checked').forEach(cb => {
        selectedMethods.push(cb.value);
    });

    if (selectedMethods.length === 0) {
        alert('Please select at least one payment method');
        return;
    }

    // Send invoice with selected payment methods
    BX.ajax.runAction('crm.invoice.send', {
        data: {
            invoice_id: <?= $invoice['ID'] ?>,
            payment_methods: selectedMethods,
            send_email: document.querySelector('[name="send_email"]').checked,
            send_sms: document.querySelector('[name="send_sms"]').checked
        }
    }).then(
        response => {
            alert('Invoice sent successfully!');
            closeDialog();
            location.reload();
        },
        error => {
            alert('Error: ' + error.errors[0].message);
        }
    );
}
</script>
```

```php
        $canSelectAdvanced = \Local\Payment\PaymentPermissions::canSelectAdvancedPaymentMethods($USER->GetID());

        // Filter payment methods based on permissions
        $allowedMethods = ['card']; // Always allowed
        if ($canSelectAdvanced) {
            $allowedMethods[] = 'ach';
            $allowedMethods[] = 'sunbit';
        }

        $validMethods = array_intersect($paymentMethods, $allowedMethods);

        if (empty($validMethods)) {
            throw new \Exception('No valid payment methods selected');
        }

        // Save to invoice
        $DB->Query("
            UPDATE b_crm_invoice
            SET AVAILABLE_PAYMENT_METHODS = '" . implode(',', $validMethods) . "'
            WHERE ID = {$invoiceId}
        ");

        // Generate Client Portal URL
        $portalUrl = $this->generateClientPortalUrl($invoiceId);

        // Send email/SMS
        if ($sendEmail) {
            $this->sendEmailWithInvoiceLink($invoiceId, $portalUrl);
        }

        if ($sendSms) {
            $this->sendSmsWithInvoiceLink($invoiceId, $portalUrl);
        }

        // Add timeline entry
        $methodsList = implode(', ', array_map('ucfirst', $validMethods));
        \Bitrix\Crm\Timeline\CommentController::getInstance()->onCreate([
            'ENTITY_TYPE_ID' => \CCrmOwnerType::Deal,
            'ENTITY_ID' => $invoice['DEAL_ID'],
            'COMMENT' => "Invoice sent with payment methods: $methodsList"
        ]);

        return ['success' => true, 'portal_url' => $portalUrl];
    }
}
```

```php
function getDocumentByToken($token, $type) {
    global $DB;

    $tableName = $type === 'estimate' ? 'b_crm_estimate' : 'b_crm_invoice';

    $result = $DB->Query("
        SELECT * FROM {$tableName}
```

**Role-Based Payment Method Flow Explanation:**

The Client Portal dynamically displays payment methods based on what the Dispatcher/Admin selected when sending the invoice:

- Database: `AVAILABLE_PAYMENT_METHODS = 'card'`
- Client Portal shows: Single payment method (Credit Card only, pre-selected)
- No choice for client - proceeds directly with CC

**Scenario 2: Dispatcher selects CC + ACH**

- Client Portal shows: Two payment options (neither pre-selected)

- Client must choose one before continuing

**Scenario 3: Dispatcher selects all three (CC + ACH + Sunbit)**
- Database: `AVAILABLE_PAYMENT_METHODS = 'card,ach,sunbit'`
- Client Portal shows: All three payment options (none pre-selected)
- Recommendation badges based on amount
- Client must choose one before continuing

**Key Technical Points:**
1. The `AVAILABLE_PAYMENT_METHODS` field is set when Dispatcher clicks "Send Invoice"
2. Client Portal reads this field and only renders the allowed payment methods
3. Amount-based recommendations are still displayed to guide client choice
4. Sunbit only appears if explicitly selected by Dispatcher (never shown by default)

### 4.5. Technical Implementation: Document Attachment

```php
// File: /local/php_interface/include/document_manager.php

namespace Local\Payment;

class DocumentManager {

    /**
     * Generate PDF after client signs invoice
     */
    public static function generateSignedPDF($invoiceId) {
        // Get invoice data
        $invoice = self::getInvoiceData($invoiceId);

        // Generate PDF with signature
        $pdfGenerator = new InvoicePDFGenerator();
        $pdfContent = $pdfGenerator->generate($invoice);

        $fileName = $invoice['INVOICE_NUMBER'] . '_signed.pdf';

        // Storage 1: Attach to Deal
        $dealFileId = self::attachToDeal(
            $invoice['DEAL_ID'],
            $pdfContent,
            $fileName,
            'Invoice'
        );

        // Storage 2: Save to centralized Drive folder
        $driveFileId = self::saveToDrive(
            $pdfContent,
            $fileName,
            'Invoices',
            $invoice['DATE_CREATE']
        );

        // Update invoice record with file IDs
        global $DB;
        $DB->Query("
            UPDATE b_crm_invoice
            SET PDF_FILE_ID_DEAL = {$dealFileId},
                PDF_FILE_ID_DRIVE = {$driveFileId},
                PDF_GENERATED_AT = NOW()
            WHERE ID = {$invoiceId}
        ");

        // Add timeline entry
        \Bitrix\Crm\Timeline\DocumentController::getInstance()->onCreate([
            'ENTITY_TYPE_ID' => \CCrmOwnerType::Deal,
            'ENTITY_ID' => $invoice['DEAL_ID'],
            'AUTHOR_ID' => 0, // System
            'DOCUMENT_ID' => $dealFileId,
            'COMMENT' => "✅ Invoice #{$invoice['INVOICE_NUMBER']} signed by client. PDF generated."
        ]);

        return [
            'deal_file_id' => $dealFileId,
            'drive_file_id' => $driveFileId
        ];
    }

    /**
     * Attach document to Deal
     */
    private static function attachToDeal($dealId, $content, $fileName, $category) {
        // Get Deal's storage
        $storage = \Bitrix\Disk\Driver::getInstance()->getStorageByUserId(1); // Admin storage

        // Create file
        $folder = $storage->getRootObject();
        $file = $folder->addFile([
            'NAME' => $fileName,
            'CONTENT' => $content
        ]);

        // Attach to Deal
        \CCrmDeal::AddFileRelation($dealId, $file->getId(), [
            'CATEGORY' => $category // 'Estimate' or 'Invoice'
        ]);

        return $file->getId();
    }

    /**
     * Save to centralized Drive folder
     */
    private static function saveToDrive($content, $fileName, $category, $date) {
        // Parse date
        $year = date('Y', strtotime($date));
        $month = date('m-F', strtotime($date)); // "01-January"

        // Get or create folder structure
        $storage = \Bitrix\Disk\Driver::getInstance()->getStorageByGroupId(COMPANY_STORAGE_ID);
        $rootFolder = $storage->getRootObject();

        // Navigate/create: Documents → Invoices → 2025 → 01-January
        $documentsFolder = self::getOrCreateFolder($rootFolder, 'Documents');
        $categoryFolder = self::getOrCreateFolder($documentsFolder, $category);
        $yearFolder = self::getOrCreateFolder($categoryFolder, $year);
        $monthFolder = self::getOrCreateFolder($yearFolder, $month);

        // Upload file
        $file = $monthFolder->addFile([
            'NAME' => $fileName,
            'CONTENT' => $content
        ]);

        return $file->getId();
    }

    /**
     * Get or create folder
     */
    private static function getOrCreateFolder($parentFolder, $name) {
        // Try to find existing
        $folder = $parentFolder->getChild(['NAME' => $name]);

        if (!$folder) {
            // Create new
            $folder = $parentFolder->addSubFolder([
                'NAME' => $name,
                'CREATED_BY' => 1
            ]);
        }

        return $folder;
    }
}
```

### 5.2. Client Portal Experience (Overdue Invoice)

### 5.3. Late Fee Calculation Rules

**State-Specific Configuration:**

The system supports configurable late fee rules on a per-state basis:

| Configuration Parameter | Description | Example Values |
|------------------------|-------------|----------------|
| **Late Fee Rate** | Percentage charged per month | 1.0%, 1.5%, 2.0% |
| **Grace Period** | Days before late fees start | 0, 5, 10, 15 days |
| **Maximum Fee** | Cap on total late fees | $25, $50, No limit, 18% annually |
| **Calculation Method** | How fees are calculated | Daily prorated, Monthly |

**Flexible by State:**
- Each state has different legal requirements for late payment fees
- System administrator can configure different rates per state
- Grace periods can be adjusted based on state regulations
- Maximum fee caps can be set to comply with state laws

**Formula (General):**

```
Late Fee = Invoice Amount × Rate × (Days Overdue - Grace Period) / 30

Example with configurable values:
  Invoice: $481.50
  Rate: [Configured per state]
  Days overdue: 10 days
  Grace period: 5 days

  Chargeable days: 10 - 5 = 5 days
  Fee = $481.50 × [State Rate] × (5/30)

  If max fee is set, Fee = min(Calculated Fee, Max Fee)
```

**Implementation Notes:**
- Late fee configuration stored in `b_option` table with state-specific keys
- Default values can be set for states without specific configuration
- Fees are calculated dynamically when client opens invoice portal page
- Daily prorated calculation ensures fair charging

### 5.4. Technical Implementation

**Client Portal: Calculate on Page Load**

```php
// File: /local/public/client-portal/views/invoice.php

<?php
// Calculate late fee when page loads
$document = applyLateFeeIfNeeded($document);

function applyLateFeeIfNeeded($invoice) {
    // Check if invoice is past due
    $dueDate = strtotime($invoice['DUE_DATE']);
    $today = time();

    if ($today <= $dueDate) {
        // Not overdue yet
        return $invoice;
    }

    // Check if already paid
    if ($invoice['PAYMENT_STATUS'] === 'Paid') {
        // Already paid, no late fee
        return $invoice;
    }

    // Calculate days overdue
    $daysOverdue = floor(($today - $dueDate) / 86400); // 86400 = seconds in a day

    // Get grace period for state
    $gracePeriod = getGracePeriod($invoice['CLIENT_STATE']);

    if ($daysOverdue <= $gracePeriod) {
        // Within grace period
        return $invoice;
    }

    // Calculate late fee
    $lateFeeRate = getLateFeeRate($invoice['CLIENT_STATE']); // 1.5% = 0.015
    $maxLateFee = getMaxLateFee($invoice['CLIENT_STATE']); // $50 or null

    $originalAmount = $invoice['SUBTOTAL'] + $invoice['TAX'];
    $daysChargeable = $daysOverdue - $gracePeriod;

    $lateFee = $originalAmount * $lateFeeRate * ($daysChargeable / 30);

    // Apply max fee cap if exists
    if ($maxLateFee && $lateFee > $maxLateFee) {
        $lateFee = $maxLateFee;
    }

    // Update invoice in database
    updateLateFee($invoice['ID'], $lateFee, $daysOverdue);

    // Update document array
    $invoice['LATE_FEE'] = $lateFee;
    $invoice['DAYS_OVERDUE'] = $daysOverdue;
    $invoice['TOTAL'] = $originalAmount + $lateFee;
    $invoice['STATUS'] = 'Past Due';

    return $invoice;
}

function updateLateFee($invoiceId, $lateFee, $daysOverdue) {
    global $DB;

    $DB->Query("
        UPDATE b_crm_invoice
        SET LATE_FEE = " . round($lateFee, 2) . ",
            DAYS_OVERDUE = {$daysOverdue},
            LATE_FEE_CALCULATED_AT = NOW(),
            STATUS = 'Past Due'
        WHERE ID = {$invoiceId}
    ");

    // Add timeline entry in Deal
    $invoice = $DB->Query("SELECT DEAL_ID, INVOICE_NUMBER FROM b_crm_invoice WHERE ID = {$invoiceId}")->Fetch();

    \Bitrix\Crm\Timeline\CommentController::getInstance()->onCreate([
        'ENTITY_TYPE_ID' => \CCrmOwnerType::Deal,
        'ENTITY_ID' => $invoice['DEAL_ID'],
        'AUTHOR_ID' => 0, // System
        'COMMENT' => "⚠️ Late payment fee applied to Invoice #{$invoice['INVOICE_NUMBER']}: $" . number_format($lateFee, 2) . " ({$daysOverdue} days overdue)"
    ]);
}
```

**Technical Implementation:**
- When creating invoice/estimate, use Deal ID as the document number
- No need for separate sequential numbering system
- Simplifies database queries (JOIN on Deal ID)

```php
namespace Local\Payment;

use Dompdf\Dompdf;
use Dompdf\Options;

class InvoicePDFGenerator {

    /**
     * Generate PDF from invoice data
     */
    public static function generatePDF($dealId) {
        // Get deal data
        $deal = \CCrmDeal::GetByID($dealId);

        if (!$deal) {
            throw new \Exception('Deal not found');
        }

        // Prepare invoice data
        $invoiceData = self::prepareInvoiceData($deal);

        // Render HTML template
        $html = self::renderTemplate($invoiceData);

        // Generate PDF
        $options = new Options();
        $options->set('isHtml5ParserEnabled', true);
        $options->set('isRemoteEnabled', true); // For loading images
        $options->set('defaultFont', 'Helvetica');

        $dompdf = new Dompdf($options);
        $dompdf->loadHtml($html);
        $dompdf->setPaper('letter', 'portrait');
        $dompdf->render();

        // Get PDF content
        $pdfContent = $dompdf->output();

        // Save to Bitrix Drive
        $fileName = "Invoice_{$invoiceData['INVOICE_NUMBER']}.pdf";
        $fileId = self::saveToDrive($pdfContent, $fileName, $dealId);

        // Attach to Deal timeline
        self::attachToTimeline($dealId, $fileId, $invoiceData['INVOICE_NUMBER']);

        return [
            'file_id' => $fileId,
            'file_path' => self::getFilePath($fileId),
            'download_url' => self::getDownloadUrl($fileId)
        ];
    }

    /**
     * Prepare invoice data from Deal
     */
    private static function prepareInvoiceData($deal) {
        // Get contact info
        $contact = \CCrmContact::GetByID($deal['CONTACT_ID']);

        // Get products/services
        $items = self::getLineItems($deal['ID']);

        // Calculate totals
        $subtotal = array_sum(array_column($items, 'TOTAL'));
        $taxRate = $deal['UF_CRM_TAX_RATE'] ?? 7.0;
        $tax = $subtotal * ($taxRate / 100);
        $serviceFee = $deal['UF_CRM_SERVICE_FEE'] ?? 0;
        $lateFee = $deal['UF_CRM_LATE_FEE'] ?? 0;
        $total = $subtotal + $tax + $serviceFee + $lateFee;

        // Calculate due date
        $invoiceDate = $deal['UF_CRM_INVOICE_DATE'] ?? new \Bitrix\Main\Type\DateTime();
        $terms = $deal['UF_CRM_TERMS_OF_PAYMENT'] ?? 'Net-0 (Immediate)';
        $dueDate = self::calculateDueDate($invoiceDate, $terms);

        return [
            'INVOICE_NUMBER' => $deal['UF_CRM_INVOICE_NUMBER'],
            'INVOICE_DATE' => $invoiceDate->format('M d, Y'),
            'DUE_DATE' => $dueDate->format('M d, Y'),
            'CLIENT_NAME' => $contact['NAME'] . ' ' . $contact['LAST_NAME'],
            'CLIENT_ADDRESS' => $contact['ADDRESS'],
            'CLIENT_CITY' => $contact['ADDRESS_CITY'],
            'CLIENT_STATE' => $contact['ADDRESS_PROVINCE'],

            'SERVICE_FEE' => $serviceFee,

            'TOTAL' => $total,
```

```php
// File: /local/php_interface/include/stripe_payment_link.php

namespace Local\Payment;

class StripePaymentLinkGenerator {

    /**
     * Create Stripe payment link for invoice
     */
    public static function createPaymentLink($dealId) {
        // Get deal data
        $deal = \CCrmDeal::GetByID($dealId);
        $invoiceNumber = $deal['UF_CRM_INVOICE_NUMBER'];

        // Get line items
        $lineItems = self::prepareLineItems($dealId);

        // Initialize Stripe
        \Stripe\Stripe::setApiKey(\COption::GetOptionString('stripe', 'secret_key'));

        try {
            // Create payment link
            $paymentLink = \Stripe\PaymentLink::create([
                'line_items' => $lineItems,
                'after_completion' => [
                    'type' => 'redirect',
                    'redirect' => [
                        'url' => 'https://bitrix24.yourdomain.com/payment/success?deal=' . $dealId
                    ]
                ],
                'metadata' => [
                    'bitrix_deal_id' => $dealId,
                    'invoice_number' => $invoiceNumber
                ],
                'invoice_creation' => [
                    'enabled' => true,
                    'invoice_data' => [
                        'description' => "Invoice #$invoiceNumber",
                        'metadata' => [
                            'bitrix_deal_id' => $dealId,
                            'invoice_number' => $invoiceNumber
                        ]
                    ]
                ],
                'payment_method_types' => ['card', 'us_bank_account'],
                'phone_number_collection' => [
                    'enabled' => true
                ],
                'custom_text' => [
                    'submit' => [
                        'message' => 'Thank you for your payment!'
                    ]
                ]
            ]);

            // Save payment link to Deal
            \CCrmDeal::Update($dealId, [
                'UF_CRM_PAYMENT_LINK' => $paymentLink->url,
                'UF_CRM_STRIPE_PAYMENT_LINK_ID' => $paymentLink->id
            ]);

            // Add timeline entry
            \Bitrix\Crm\Timeline\CommentController::getInstance()->onCreate([
                'ENTITY_TYPE_ID' => \CCrmOwnerType::Deal,
                'ENTITY_ID' => $dealId,
                'COMMENT' => "💳 Payment link created\n" . $paymentLink->url
            ]);

            return $paymentLink->url;

        } catch (\Stripe\Exception\ApiErrorException $e) {
            error_log("Stripe payment link creation failed: " . $e->getMessage());
            throw new \Exception('Failed to create payment link: ' . $e->getMessage());
        }
    }

    /**
     * Prepare line items for Stripe
     */
    private static function prepareLineItems($dealId) {
        $items = [];

        // Get products from Deal
        $dbRes = \CCrmProductRow::GetList(
            ['SORT' => 'ASC'],
            ['OWNER_ID' => $dealId, 'OWNER_TYPE' => 'D'],
            false,
            false,
            ['*']
        );

        while ($row = $dbRes->Fetch()) {
            // Create or get Stripe price
            $priceId = self::getOrCreateStripePrice(
                $row['PRODUCT_NAME'],
                $row['PRICE'] * 100, // Convert to cents
                'usd'
            );

            $items[] = [
                'price' => $priceId,
                'quantity' => $row['QUANTITY']
            ];
        }

        return $items;
    }

    /**
     * Get or create Stripe price for product
     */
    private static function getOrCreateStripePrice($productName, $amountCents, $currency) {
        \Stripe\Stripe::setApiKey(\COption::GetOptionString('stripe', 'secret_key'));

        try {
            // Try to find existing price
            $prices = \Stripe\Price::all([
                'lookup_keys' => [self::generateLookupKey($productName, $amountCents)]
            ]);

            if (count($prices->data) > 0) {
                return $prices->data[0]->id;
            }

            // Create new price
            $price = \Stripe\Price::create([
                'unit_amount' => $amountCents,
                'currency' => $currency,
                'product_data' => [
                    'name' => $productName
                ],
                'lookup_key' => self::generateLookupKey($productName, $amountCents)
            ]);

            return $price->id;

        } catch (\Exception $e) {
            error_log("Failed to create Stripe price: " . $e->getMessage());
            throw $e;
        }
    }

    /**
     * Generate lookup key for product
     */
    private static function generateLookupKey($productName, $amountCents) {
        $slug = strtolower(preg_replace('/[^a-z0-9]+/', '_', $productName));
        return $slug . '_' . $amountCents;
    }
}
```

```html
    isDrawing = true;
    [lastX, lastY] = [e.offsetX, e.offsetY];

function draw(e) {
    if (!isDrawing) return;

    ctx.beginPath();
    ctx.moveTo(lastX, lastY);
    ctx.lineTo(e.offsetX, e.offsetY);
    ctx.strokeStyle = '#000';
    ctx.lineWidth = 2;
    ctx.lineCap = 'round';
    ctx.stroke();

    [lastX, lastY] = [e.offsetX, e.offsetY];
}

function stopDrawing() {
    isDrawing = false;
}

function handleTouchStart(e) {
    e.preventDefault();
    const touch = e.touches[0];
    const rect = canvas.getBoundingClientRect();
    lastX = touch.clientX - rect.left;
    lastY = touch.clientY - rect.top;
    isDrawing = true;
}

function handleTouchMove(e) {
    if (!isDrawing) return;
    e.preventDefault();

    const touch = e.touches[0];
    const rect = canvas.getBoundingClientRect();
    const x = touch.clientX - rect.left;
    const y = touch.clientY - rect.top;

    ctx.beginPath();
    ctx.moveTo(lastX, lastY);
    ctx.lineTo(x, y);
    ctx.strokeStyle = '#000';
    ctx.lineWidth = 2;
    ctx.lineCap = 'round';
    ctx.stroke();

    lastX = x;
    lastY = y;
}
```
