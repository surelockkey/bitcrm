# Story 8.02: CRM - Duplicate Detection & Merging

---

## Overview

This story covers the detection and merging of duplicate client records (Companies and Contacts) in Bitrix24 CRM. The system provides managers with tools to identify potential duplicates, review them, and merge records while preserving complete history (deals, calls, emails, invoices) on the primary record.

---

## Business Context

**Current Pain Point:**
- Multiple dispatchers create clients during busy periods
- Phone number typos create duplicates (e.g., "555-1234" vs. "5551234")
- Same company name variations ("ABC Supply" vs. "ABC Supply Co.")
- Manual cleanup wastes manager time

**Solution:**
Intelligent duplicate detection with guided merge workflow, preserving complete history and maintaining data integrity.

**Expected Outcome:**
Clean client database, no orphaned records, complete history preserved after merge.

---

## User Personas

### Manager
- **Goal:** Maintain clean CRM, merge duplicates weekly
- **Pain:** Manual duplicate search time-consuming, afraid of losing data
- **Need:** Automated duplicate detection, safe merge with history preservation

### Administrator
- **Goal:** Keep database clean, prevent future duplicates
- **Pain:** No visibility into duplicate creation patterns
- **Need:** Duplicate prevention rules, audit trail of merges

### Dispatcher
- **Goal:** Find correct client quickly
- **Pain:** Multiple records for same client cause confusion
- **Need:** Single source of truth per client

### 1. Duplicate Detection Logic

**Detection Triggers:**

```
AUTOMATIC DETECTION (background job, runs nightly):
  ↓
Scan all Contacts:
  - Compare normalized phone numbers
  - Compare email addresses
  - Compare full names (fuzzy match)
  ↓
Scan all Companies:
  - Compare normalized company names
  - Compare phone numbers
  - Compare email addresses
  ↓
Generate "Potential Duplicates" list
  ↓
Notify Manager role users
```

**Detection Rules:**

**Rule 1: Exact Phone Match**
```
Contact A: Phone = "+15551234567"
Contact B: Phone = "+1 (555) 123-4567"

Normalized: Both = "+15551234567"

Result: HIGH CONFIDENCE duplicate
```

**Rule 2: Exact Email Match**
```
Contact A: Email = "john@example.com"
Contact B: Email = "JOHN@example.com"

Normalized: Both = "john@example.com"

Result: HIGH CONFIDENCE duplicate
```

**Rule 3: Fuzzy Name Match + Partial Phone**
```
Contact A:
  Name: "John Smith"
  Phone: "+15551234567"

Contact B:
  Name: "John Smyth"  ← typo
  Phone: "+15551234000"  ← last 4 digits different

Fuzzy match: 85% similar (Levenshtein distance)
Phone area code: SAME (555)

Result: MEDIUM CONFIDENCE duplicate
```

**Rule 4: Company Name Variations**
```
Company A: "ABC Supply Co."
Company B: "ABC Supply"

Normalized:
  - Strip: "Co.", "Inc.", "LLC", punctuation
  - Lowercase
  - Result: "abc supply" = "abc supply"

Result: HIGH CONFIDENCE duplicate
```

### 3. Merge Process Logic

**Merge Flow:**

```
Step 1: Manager selects PRIMARY record
  Record A = Primary (will be kept)
  Record B = Duplicate (will be deleted)
  ↓
Step 2: System validates merge eligibility
  Check:
    - Both records are same entity type (Contact/Company)
    - Neither record is locked
    - User has "Manager" role permission
  ↓
Step 3: Backup snapshot created
  Store JSON snapshot of both records
  Store in: b_crm_merge_log table
  ↓
Step 4: Move all related entities
  From Record B → Record A:
    - Deals: UPDATE CONTACT_ID/COMPANY_ID
    - Activities (calls, emails): UPDATE OWNER_ID
    - Invoices: UPDATE CLIENT_ID
    - Timeline items: UPDATE ENTITY_ID
    - Notes: COPY to Record A
  ↓
Step 5: Merge field values
  Strategy: UNION (combine unique values)

  Example:
    Record A phones: [555-123-4567]
    Record B phones: [(555) 123-4567, 555-999-8888]

    Result: [555-123-4567, 555-999-8888]
    (duplicates removed after normalization)
  ↓
Step 6: Delete duplicate record
  DELETE FROM b_crm_contact WHERE ID = Record B
  (cascading deletes handled by FK constraints)
  ↓
Step 7: Log merge action
  INSERT INTO b_crm_merge_log:
    - Primary ID
    - Duplicate ID
    - Merged by User ID
    - Timestamp
    - Snapshot (JSON)
  ↓
Step 8: Notify completion
  Show success message
  Update "Potential Duplicates" count
```

### 4. Field Merge Strategy

**Strategy by Field Type:**

**Phone Numbers:**
```
Strategy: UNION (keep all unique)

Record A: [555-123-4567]
Record B: [(555) 123-4567, 555-999-8888]

Normalized:
  - 555-123-4567 → +15551234567
  - (555) 123-4567 → +15551234567 (duplicate)
  - 555-999-8888 → +15559998888

Result: [+15551234567, +15559998888]
Display: ["(555) 123-4567", "(555) 999-8888"]
```

**Emails:**
```
Strategy: UNION (keep all unique, case-insensitive)

Record A: [john@example.com]
Record B: [JOHN@example.com, j.smith@example.com]

Normalized (lowercase):
  - john@example.com
  - john@example.com (duplicate)
  - j.smith@example.com

Result: [john@example.com, j.smith@example.com]
```

**Name Fields:**
```
Strategy: PREFER PRIMARY

Record A: First Name = "John", Last Name = "Smith"
Record B: First Name = "John", Last Name = "Smyth"

Result: "John Smith" (from primary)

Manager can manually edit after merge if needed
```

**Company Link:**
```
Strategy: PREFER PRIMARY

Record A: Company ID = 123 (ABC Supply Co.)
Record B: Company ID = NULL

Result: Company ID = 123 (keep primary)
```

**Custom Fields:**
```
Strategy: COALESCE (prefer non-empty)

Record A: Client Type = "Commercial"
Record B: Client Type = NULL

Result: Client Type = "Commercial"

Record A: Notes = "Preferred customer"
Record B: Notes = "VIP account"

Result: Notes = "Preferred customer\n---\nVIP account"
(concatenate with separator)
```

## Technical Implementation

### 1. Database Schema

```sql
-- Merge Log Table
CREATE TABLE b_crm_merge_log (
    ID INT PRIMARY KEY AUTO_INCREMENT,
    ENTITY_TYPE ENUM('contact', 'company'),
    PRIMARY_ID INT NOT NULL,              -- Record that was kept
    DUPLICATE_ID INT NOT NULL,            -- Record that was deleted
    MERGED_BY INT NOT NULL,               -- User ID
    DATE_MERGED DATETIME NOT NULL,
    SNAPSHOT_JSON LONGTEXT,               -- JSON backup of both records
    ROLLBACK_POSSIBLE ENUM('Y','N') DEFAULT 'Y',
    DATE_ROLLBACK_EXPIRES DATETIME,       -- 30 days from merge
    INDEX idx_primary (PRIMARY_ID, ENTITY_TYPE),
    INDEX idx_duplicate (DUPLICATE_ID, ENTITY_TYPE),
    INDEX idx_date (DATE_MERGED)
);

-- Duplicate Detection Queue
CREATE TABLE b_crm_duplicate_queue (
    ID INT PRIMARY KEY AUTO_INCREMENT,
    ENTITY_TYPE ENUM('contact', 'company'),
    RECORD_A_ID INT NOT NULL,
    RECORD_B_ID INT NOT NULL,
    CONFIDENCE ENUM('high', 'medium', 'low'),
    MATCH_REASON VARCHAR(255),            -- "phone_match", "email_match", etc.
    STATUS ENUM('pending', 'reviewed', 'merged', 'not_duplicate'),
    REVIEWED_BY INT,
    DATE_DETECTED DATETIME NOT NULL,
    DATE_REVIEWED DATETIME,
    INDEX idx_status (STATUS),
    INDEX idx_confidence (CONFIDENCE),
    INDEX idx_date (DATE_DETECTED)
);
```

### 2. Duplicate Detection Algorithm

```php
// File: /local/php_interface/include/crm_duplicate_detector.php

namespace Local\Crm;

use Bitrix\Crm\ContactTable;
use Bitrix\Crm\CompanyTable;

class DuplicateDetector {

    /**
     * Detect all potential duplicates (run nightly via cron)
     */
    public static function detectAll() {
        self::detectContactDuplicates();
        self::detectCompanyDuplicates();
    }

    /**
     * Detect duplicate Contacts
     */
    private static function detectContactDuplicates() {
        global $DB;

        // Find Contacts with identical normalized phone
        $sql = "
            SELECT c1.ID as ID_A, c2.ID as ID_B, 'phone_match' as REASON
            FROM b_crm_contact c1
            INNER JOIN b_crm_contact c2
              ON c1.ID < c2.ID
              AND REPLACE(REPLACE(REPLACE(c1.PHONE, ' ', ''), '-', ''), '(', '') =
                  REPLACE(REPLACE(REPLACE(c2.PHONE, ' ', ''), '-', ''), '(', '')
            WHERE c1.PHONE IS NOT NULL
              AND c2.PHONE IS NOT NULL
        ";

        $result = $DB->Query($sql);
        while ($row = $result->Fetch()) {
            self::addToQueue('contact', $row['ID_A'], $row['ID_B'], 'high', $row['REASON']);
        }

        // Find Contacts with identical email
        $sql = "
            SELECT c1.ID as ID_A, c2.ID as ID_B, 'email_match' as REASON
            FROM b_crm_contact c1
            INNER JOIN b_crm_contact c2
              ON c1.ID < c2.ID
              AND LOWER(c1.EMAIL) = LOWER(c2.EMAIL)
            WHERE c1.EMAIL IS NOT NULL
              AND c2.EMAIL IS NOT NULL
        ";

        $result = $DB->Query($sql);
        while ($row = $result->Fetch()) {
            self::addToQueue('contact', $row['ID_A'], $row['ID_B'], 'high', $row['REASON']);
        }

        // Fuzzy name match (medium confidence)
        self::detectFuzzyNameDuplicates();
    }

    /**
     * Detect duplicate Companies
     */
    private static function detectCompanyDuplicates() {
        global $DB;

        // Normalize company names and find matches
        $sql = "
            SELECT c1.ID as ID_A, c2.ID as ID_B, 'name_match' as REASON
            FROM b_crm_company c1
            INNER JOIN b_crm_company c2
              ON c1.ID < c2.ID
              AND LOWER(
                    REPLACE(
                      REPLACE(
                        REPLACE(c1.TITLE, 'Co.', ''),
                        'Inc.', ''
                      ),
                      'LLC', ''
                    )
                  ) = LOWER(
                    REPLACE(
                      REPLACE(
                        REPLACE(c2.TITLE, 'Co.', ''),
                        'Inc.', ''
                      ),
                      'LLC', ''
                    )
                  )
        ";

        $result = $DB->Query($sql);
        while ($row = $result->Fetch()) {
            self::addToQueue('company', $row['ID_A'], $row['ID_B'], 'high', $row['REASON']);
        }
    }

    /**
     * Fuzzy name matching using Levenshtein distance
     */
    private static function detectFuzzyNameDuplicates() {
        // Get all contacts
        $contacts = ContactTable::getList([
            'select' => ['ID', 'NAME', 'LAST_NAME', 'PHONE']
        ])->fetchAll();

        // Compare each pair
        foreach ($contacts as $i => $contactA) {
            foreach (array_slice($contacts, $i + 1) as $contactB) {
                $fullNameA = $contactA['NAME'] . ' ' . $contactA['LAST_NAME'];
                $fullNameB = $contactB['NAME'] . ' ' . $contactB['LAST_NAME'];

                $distance = levenshtein($fullNameA, $fullNameB);
                $maxLength = max(strlen($fullNameA), strlen($fullNameB));
                $similarity = 1 - ($distance / $maxLength);

                // If >80% similar AND same area code
                if ($similarity > 0.80) {
                    $areaCodeA = substr(preg_replace('/[^0-9]/', '', $contactA['PHONE']), 1, 3);
                    $areaCodeB = substr(preg_replace('/[^0-9]/', '', $contactB['PHONE']), 1, 3);

                    if ($areaCodeA === $areaCodeB) {
                        self::addToQueue(
                            'contact',
                            $contactA['ID'],
                            $contactB['ID'],
                            'medium',
                            'fuzzy_name_match'
                        );
                    }
                }
            }
        }
    }

    /**
     * Add duplicate pair to review queue
     */
    private static function addToQueue($entityType, $idA, $idB, $confidence, $reason) {
        global $DB;

        // Check if already in queue
        $existing = $DB->Query("
            SELECT ID FROM b_crm_duplicate_queue
            WHERE ENTITY_TYPE = '{$entityType}'
              AND RECORD_A_ID = {$idA}
              AND RECORD_B_ID = {$idB}
              AND STATUS = 'pending'
        ")->Fetch();

        if (!$existing) {
            $DB->Query("
                INSERT INTO b_crm_duplicate_queue (
                    ENTITY_TYPE, RECORD_A_ID, RECORD_B_ID,
                    CONFIDENCE, MATCH_REASON, STATUS, DATE_DETECTED
                ) VALUES (
                    '{$entityType}', {$idA}, {$idB},
                    '{$confidence}', '{$reason}', 'pending', NOW()
                )
            ");
        }
    }
}
```

### 3. Merge Handler

```php
// File: /local/php_interface/include/crm_merge_handler.php

namespace Local\Crm;

use Bitrix\Crm\ContactTable;
use Bitrix\Crm\CompanyTable;
use Bitrix\Crm\DealTable;

class MergeHandler {

    /**
     * Merge two Contact records
     *
     * @param int $primaryId - Contact to keep
     * @param int $duplicateId - Contact to delete
     * @return bool
     */
    public static function mergeContacts($primaryId, $duplicateId) {
        global $DB, $USER;

        // Validate permission
        if (!self::hasPermission('merge')) {
            throw new \Exception('Permission denied: merge contacts');
        }

        // Start transaction
        $DB->StartTransaction();

        try {
            // Step 1: Create backup snapshot
            $snapshot = self::createSnapshot('contact', $primaryId, $duplicateId);

            // Step 2: Move all related entities
            self::moveContactDeals($duplicateId, $primaryId);
            self::moveContactActivities($duplicateId, $primaryId);
            self::moveContactInvoices($duplicateId, $primaryId);

            // Step 3: Merge field values
            self::mergeContactFields($primaryId, $duplicateId);

            // Step 4: Log merge
            self::logMerge('contact', $primaryId, $duplicateId, $snapshot);

            // Step 5: Delete duplicate
            $contact = new \CCrmContact();
            $contact->Delete($duplicateId);

            // Step 6: Add timeline entry
            self::addTimelineEntry('contact', $primaryId, $duplicateId);

            // Commit transaction
            $DB->Commit();

            return true;

        } catch (\Exception $e) {
            $DB->Rollback();
            throw $e;
        }
    }

    /**
     * Create JSON snapshot of both records
     */
    private static function createSnapshot($entityType, $primaryId, $duplicateId) {
        if ($entityType === 'contact') {
            $primary = ContactTable::getById($primaryId)->fetch();
            $duplicate = ContactTable::getById($duplicateId)->fetch();
        } else {
            $primary = CompanyTable::getById($primaryId)->fetch();
            $duplicate = CompanyTable::getById($duplicateId)->fetch();
        }

        return json_encode([
            'primary' => $primary,
            'duplicate' => $duplicate,
            'timestamp' => date('Y-m-d H:i:s')
        ]);
    }

    /**
     * Move all Deals from duplicate to primary
     */
    private static function moveContactDeals($fromId, $toId) {
        global $DB;

        $DB->Query("
            UPDATE b_crm_deal
            SET CONTACT_ID = {$toId}
            WHERE CONTACT_ID = {$fromId}
        ");
    }

    /**
     * Move all Activities (calls, emails, meetings)
     */
    private static function moveContactActivities($fromId, $toId) {
        global $DB;

        $DB->Query("
            UPDATE b_crm_activity
            SET OWNER_ID = {$toId}
            WHERE OWNER_TYPE_ID = " . \CCrmOwnerType::Contact . "
              AND OWNER_ID = {$fromId}
        ");
    }

    /**
     * Move all Invoices
     */
    private static function moveContactInvoices($fromId, $toId) {
        global $DB;

        $DB->Query("
            UPDATE b_crm_invoice
            SET UF_CONTACT_ID = {$toId}
            WHERE UF_CONTACT_ID = {$fromId}
        ");
    }

    /**
     * Merge field values (UNION strategy)
     */
    private static function mergeContactFields($primaryId, $duplicateId) {
        $primary = ContactTable::getById($primaryId)->fetch();
        $duplicate = ContactTable::getById($duplicateId)->fetch();

        // Merge phones (union)
        $phones = array_unique(array_merge(
            $primary['PHONE'] ?? [],
            $duplicate['PHONE'] ?? []
        ));

        // Merge emails (union)
        $emails = array_unique(array_merge(
            $primary['EMAIL'] ?? [],
            $duplicate['EMAIL'] ?? []
        ));

        // Update primary record
        $contact = new \CCrmContact();
        $contact->Update($primaryId, [
            'PHONE' => $phones,
            'EMAIL' => $emails
        ]);
    }

    /**
     * Log merge operation
     */
    private static function logMerge($entityType, $primaryId, $duplicateId, $snapshot) {
        global $DB, $USER;

        $DB->Query("
            INSERT INTO b_crm_merge_log (
                ENTITY_TYPE, PRIMARY_ID, DUPLICATE_ID, MERGED_BY,
                DATE_MERGED, SNAPSHOT_JSON, DATE_ROLLBACK_EXPIRES
            ) VALUES (
                '{$entityType}', {$primaryId}, {$duplicateId}, {$USER->GetID()},
                NOW(), '{$DB->ForSql($snapshot)}', DATE_ADD(NOW(), INTERVAL 30 DAY)
            )
        ");
    }

    /**
     * Add timeline entry to primary record
     */
    private static function addTimelineEntry($entityType, $primaryId, $duplicateId) {
        $ownerType = ($entityType === 'contact') ? \CCrmOwnerType::Contact : \CCrmOwnerType::Company;

        \CCrmActivity::Add([
            'OWNER_TYPE_ID' => $ownerType,
            'OWNER_ID' => $primaryId,
            'TYPE_ID' => \CCrmActivityType::Provider,
            'PROVIDER_ID' => 'CRM_MERGE',
            'SUBJECT' => 'Record Merged',
            'DESCRIPTION' => "Record #{$duplicateId} was merged into this record",
            'COMPLETED' => 'Y'
        ]);
    }

    /**
     * Check if user has permission to merge
     */
    private static function hasPermission($action) {
        global $USER;
        // Check if user has "Manager" role
        $userRoles = \CUser::GetUserGroup($USER->GetID());
        return in_array(ROLE_MANAGER_ID, $userRoles);
    }
}
```

### Functional Acceptance

- ✅ **Automatic detection:** Nightly job detects phone/email/name duplicates
- ✅ **Manual reporting:** Dispatchers can report suspected duplicates

- ✅ **History preservation:** All deals, calls, emails moved to primary

- ✅ **Clear comparison:** Side-by-side display of duplicate records

- ✅ **Warning message:** "Cannot be undone" warning displayed

- ✅ **Merge history:** List of past merges accessible
