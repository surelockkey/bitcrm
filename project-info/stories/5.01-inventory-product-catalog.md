# TR-20: Inventory - Product Catalog Setup

**Epic:** Inventory Management System
**Component:** Product Catalog & Item Master Data
**Version:** 1.0
**Last Updated:** October 29, 2025

---

## 1. Overview

This story covers the setup and management of the product catalog in Bitrix24, which serves as the foundation for the entire inventory management system. The catalog includes both physical products (stock-managed) and services (non-stock).

**Key Objectives:**
- Create hierarchical product catalog structure
- Define item attributes and metadata
- Implement barcode management system
- Enable bulk import from supplier data
- Provide price management workflows

## 2. Catalog Structure

### 2.1. Two Separate Catalogs

Bitrix24 will have **TWO distinct catalogs:**

```
    │   ├── Blank Keys
```

**Catalog 2: Services (Non-Stock)**
```

├── ID: "services_catalog"
├── Inventory Tracking: ❌ DISABLED
├── No Categories (flat list)
└── Items:

    ├── Emergency Call Fee
    ├── Mileage Fee
    ├── After Hours Fee
    └── Consultation Fee
```

### 2.2. Why Two Catalogs?

**Reason 1: Inventory Behavior**
- Products → tracked in warehouses, transferred, counted
- Services → no inventory, just pricing

**Reason 2: Different Workflows**
- Products → barcode scanning, receiving, transfers
- Services → only added to invoices, no physical handling

**Reason 3: User Interface**
- Technicians scanning inventory → see only Products
- Creating invoice → can choose from BOTH catalogs

## 3. Product Attributes (Fields)

### 3.1. Required Fields

Every product MUST have:

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| **SKU** | String (50) | Unique identifier | `LOCK-RES-KW-001` |
| **Name** | String (255) | Display name | `Kwikset Residential Deadbolt - Satin Nickel` |
| **Category** | Hierarchy | Product category | `Locks & Cylinders > Residential Locks` |
| **Barcode** | String (255) | Scannable code(s) | `012345678901` |
| **Cost (Company)** | Decimal (10,2) | Company's purchase cost from supplier | `$15.00` |
| **Cost for Tech** | Decimal (10,2) | Tech's buy-in price from company (covers insurance & operations) | `$18.00` |
| **Price (Client)** | Decimal (10,2) | Default selling price to end customer | `$45.00` |
| **Stock Quantity** | Integer | Current available quantity (auto-calculated across all warehouses) | `45` |
| **Active** | Boolean | Is item active? | `Yes` |
| **Type** | Enum | Product or Service | `Product` |

### 3.1.1. Pricing Model Explained

**Three-Tier Pricing Structure:**

```
Cost (Company) = $15.00
   ↓ Company buys from supplier (UHS, Amazon, etc.)

Cost for Tech = $18.00 (+$3.00 markup = 20%)
   ↓ Tech buys from company inventory
   ↓ Markup covers: insurance, overhead, operational costs

Price (Client) = $45.00
   ↓ Tech sells to end customer
   ↓ Tech profit: $45 - $18 = $27.00
   ↓ Company profit: $18 - $15 = $3.00
```

**Important:**
- **Cost (Company):** Hidden from Technicians, visible only to Management/Logistics
- **Cost for Tech:** Visible to Technicians (this is their buy-in price)
- **Price (Client):** Visible to all, editable by Technicians within limits

**Example with Commission Calculation:**

```
Job Total: $350
Parts used by Tech:
  - Lock A: Cost for Tech = $18, Sold to Client = $45
  - Lock B: Cost for Tech = $12, Sold to Client = $30
  Total Parts (Tech Cost): $30
  Total Parts (Client Price): $75

Commission Calculation:
  Revenue: $350
  Tax: $28
  Parts (Tech Cost): $30  ← This is what goes into commission calc
  Credit Card Fee: $10.50
  Tech Share: 40%

  Profit Pool = $350 - $28 - $30 - $10.50 = $281.50
  Tech Commission = $281.50 × 40% = $112.60
  Company Net = $281.50 - $112.60 = $168.90

Company Analysis (internal):
  Company received from Tech for parts: $30
  Company paid supplier for parts: $25 (Cost Company)
  Company profit on parts: $5
  Company total profit: $168.90 + $5 = $173.90
```

### 3.3.1. Pricing Field Visibility & Permissions

**Access Control by Role:**

```php
// File: /local/php_interface/include/product_permissions.php

namespace Local\Inventory;

class ProductPermissions {

    /**
     * Determine which price fields user can see/edit
     */
    public static function getPriceFieldAccess($userId) {
        $user = \CUser::GetByID($userId)->Fetch();
        $userGroups = \CUser::GetUserGroup($userId);

        // Check role
        $isOwner = in_array(OWNER_GROUP_ID, $userGroups);
        $isManager = in_array(MANAGER_GROUP_ID, $userGroups);
        $isLogistics = in_array(LOGISTICS_GROUP_ID, $userGroups);
        $isDispatcher = in_array(DISPATCHER_GROUP_ID, $userGroups);
        $isTech = in_array(TECH_GROUP_ID, $userGroups);

        return [
            'UF_COST_COMPANY' => [
                'visible' => $isOwner || $isManager || $isLogistics,
                'editable' => $isOwner || $isManager || $isLogistics
            ],
            'UF_COST_TECH' => [
                'visible' => true,  // All can see
                'editable' => $isOwner || $isManager || $isLogistics
            ],
            'UF_PRICE_CLIENT' => [
                'visible' => true,  // All can see
                'editable' => true,  // All can edit (with different contexts)
                'edit_in_catalog' => !$isTech,  // Techs can't edit in catalog
                'edit_in_deal' => true  // All can edit during deal/job
            ]
        ];
    }

    /**
     * Filter product data based on user permissions
     */
    public static function filterProductData($productData, $userId) {
        $access = self::getPriceFieldAccess($userId);

        // Hide Cost (Company) from unauthorized users
        if (!$access['UF_COST_COMPANY']['visible']) {
            unset($productData['UF_COST_COMPANY']);
        }

        return $productData;
    }
}
```

## 4. Barcode Management

### 4.1. Supported Barcode Types

System must support **all common barcode formats** without restrictions:

**1D Barcodes (Linear):**
- ✅ UPC-A (12 digits) - most common in USA
- ✅ UPC-E (6 digits) - compact version
- ✅ EAN-13 (13 digits) - European standard
- ✅ EAN-8 (8 digits) - compact European
- ✅ Code 39 - alphanumeric
- ✅ Code 128 - high density, full ASCII
- ✅ ITF (Interleaved 2 of 5) - shipping/distribution
- ✅ Codabar - medical/libraries

**2D Barcodes (Matrix):**
- ✅ QR Code - most versatile, smartphone-friendly
- ✅ Data Matrix - small parts, electronics
- ✅ PDF417 - high data capacity

**Implementation Strategy:**

Use **ZXing library** (Zebra Crossing) - industry standard, supports ALL formats:


### 4.3. Custom Barcode Generation

**Use Case:** Products without manufacturer barcodes

**Generation Rules:**

1. **Format:** Code 128 (supports alphanumeric)
2. **Pattern:** Company prefix + SKU
3. **Example:** `LKS-LOCK-RES-KW-001` → generates Code 128 barcode

**Implementation:**

```php
// File: /local/php_interface/include/barcode_generator.php

namespace Local\Inventory;

use Picqer\Barcode\BarcodeGeneratorPNG;

class BarcodeGenerator {

    /**
     * Generate barcode for product without one
     */
    public static function generateBarcode($productId, $sku) {

        // Pattern: LKS-{SKU}
        $barcodeText = 'LKS-' . $sku;

        // Generate Code 128 image
        $generator = new BarcodeGeneratorPNG();
        $barcodeImage = $generator->getBarcode(
            $barcodeText,
            $generator::TYPE_CODE_128,
            3,  // width factor
            100 // height in pixels
        );

        // Save to Bitrix storage
        $fileName = "barcode_{$productId}.png";
        $filePath = $_SERVER['DOCUMENT_ROOT'] . "/upload/barcodes/{$fileName}";

        file_put_contents($filePath, $barcodeImage);

        // Add barcode to product
        CIBlockElement::SetPropertyValuesEx(
            $productId,
            false,
            ['UF_BARCODE' => $barcodeText]
        );

        return [
            'barcode' => $barcodeText,
            'image_path' => $filePath,
            'image_url' => "/upload/barcodes/{$fileName}"
        ];
    }

    /**
     * Generate printable label (barcode + text)
     */
    public static function generatePrintableLabel($productId) {
        $product = CIBlockElement::GetByID($productId)->Fetch();

        $html = '
        <div style="text-align: center; page-break-after: always;">
            <div style="font-size: 14px; font-weight: bold; margin-bottom: 10px;">
                ' . htmlspecialchars($product['NAME']) . '
            </div>
            <div>
                <img src="/upload/barcodes/barcode_' . $productId . '.png" />
            </div>
            <div style="font-size: 12px; margin-top: 5px;">
                SKU: ' . htmlspecialchars($product['CODE']) . '
            </div>
        </div>
        ';

        return $html;
    }
}
```

## 6. Price Management

### 6.1. Who Can Update Prices

**Permission Levels:**

| Role | Cost (Company) | Cost for Tech | Price (Client) | Bulk Update |
|------|----------------|---------------|----------------|-------------|
| **Owner** | ✅ Edit | ✅ Edit | ✅ Edit | ✅ Yes |
| **Manager** | ✅ Edit | ✅ Edit | ✅ Edit | ✅ Yes |
| **Logistics** | ✅ Edit | ✅ Edit | ✅ Edit | ✅ Yes |
| **Dispatcher** | ❌ Hidden | ✅ View only | ✅ Edit | ❌ No |
| **Technician** | ❌ Hidden | ✅ View only | ✅ Edit* | ❌ No |

**\*Technician Price Edit:** Technicians can edit Price (Client) **during job** within configured limits (e.g., ±15% of default price). They cannot edit catalog prices directly.

**Why this structure:**
- **Cost (Company)** is highly sensitive (true profit margin) → only management/logistics
- **Cost for Tech** is visible to techs (their buy-in price) but only management can change it
- **Price (Client)** can be adjusted by dispatchers and technicians (negotiations, special deals)
- Technicians edit prices on-the-fly during jobs, not in catalog

```php
// Scheduled task: Import photos from FTP folder
// Runs daily at 2 AM

function importProductPhotos() {
    $photoDir = $_SERVER['DOCUMENT_ROOT'] . '/upload/products_import/';
    $files = glob($photoDir . '*.{jpg,jpeg,png}', GLOB_BRACE);

    foreach ($files as $file) {
        $filename = basename($file);

        // Extract SKU from filename
        // Pattern: {SKU}.jpg or {SKU}_2.jpg
        preg_match('/^(.+?)(?:_(\d+))?\.(jpg|jpeg|png)$/i', $filename, $matches);
        $sku = $matches[1];
        $photoIndex = $matches[2] ?? 1;

        // Find product by SKU
        $product = CIBlockElement::GetList(
            [],
            ['CODE' => $sku, 'IBLOCK_ID' => PRODUCTS_CATALOG_ID],
            false,
            false,
            ['ID', 'NAME']
        )->Fetch();

        if ($product) {
            // Upload to Bitrix
            $fileArray = CFile::MakeFileArray($file);
            $fileId = CFile::SaveFile($fileArray, 'catalog');

            if ($photoIndex == 1) {
                // Set as primary photo
                CIBlockElement::SetPropertyValuesEx(
                    $product['ID'],
                    false,
                    ['DETAIL_PICTURE' => $fileId]
                );
            } else {
                // Add to gallery
                // Implementation depends on Bitrix photo gallery setup
            }

            // Archive processed file
            rename($file, $photoDir . 'processed/' . $filename);

            echo "✅ Photo imported for {$sku}\n";
        } else {
            echo "⚠️  Product not found for SKU: {$sku}\n";
        }
    }
}
```

### 8.2. Barcode Lookup Implementation

```php
// File: /local/api/inventory/barcode_lookup.php

<?php
require_once($_SERVER['DOCUMENT_ROOT'] . '/bitrix/modules/main/include/prolog_before.php');

use Bitrix\Main\Application;
use Bitrix\Catalog\StoreProductTable;

$request = Application::getInstance()->getContext()->getRequest();
$barcode = $request->getPost('barcode');
$warehouseId = $request->getPost('warehouse_id');

if (!$barcode) {
    echo json_encode(['success' => false, 'error' => 'Barcode required']);
    exit;
}

// Search product by barcode
$product = findProductByBarcode($barcode);

if (!$product) {
    echo json_encode([
        'success' => false,
        'error' => 'Product not found',
        'barcode' => $barcode
    ]);
    exit;
}

// Get stock levels
$stockLevels = getStockLevels($product['ID'], $warehouseId);

echo json_encode([
    'success' => true,
    'product' => [
        'id' => $product['ID'],
        'sku' => $product['CODE'],
        'name' => $product['NAME'],
        'cost' => $product['CATALOG_PURCHASING_PRICE'],
        'price' => $product['CATALOG_PRICE_1'],
        'barcode_scanned' => $barcode,
        'photo' => $product['DETAIL_PICTURE_URL'],
        'stock_levels' => $stockLevels
    ]
]);

function findProductByBarcode($barcode) {
    // Search in custom barcode field
    $filter = [
        'IBLOCK_ID' => PRODUCTS_CATALOG_ID,
        'ACTIVE' => 'Y',
        'PROPERTY_UF_BARCODE' => $barcode
    ];

    $result = CIBlockElement::GetList(
        [],
        $filter,
        false,
        ['nTopCount' => 1],
        ['ID', 'NAME', 'CODE', 'DETAIL_PICTURE', 'CATALOG_PURCHASING_PRICE', 'CATALOG_PRICE_1']
    )->Fetch();

    return $result ?: null;
}

function getStockLevels($productId, $warehouseId = null) {
    $filter = ['PRODUCT_ID' => $productId];
    if ($warehouseId) {
        $filter['STORE_ID'] = $warehouseId;
    }

    $stores = StoreProductTable::getList([
        'filter' => $filter,
        'select' => ['STORE_ID', 'AMOUNT']
    ])->fetchAll();

    $total = 0;
    $byWarehouse = [];

    foreach ($stores as $store) {
        $total += $store['AMOUNT'];
        $byWarehouse[] = [
            'warehouse_id' => $store['STORE_ID'],
            'quantity' => $store['AMOUNT']
        ];
    }

    return [
        'total' => $total,
        'by_warehouse' => $byWarehouse
    ];
}
```

## 10. Product Variants

### 10.1. Variants Overview

Many products have multiple variations of the same base item:
- **Finishes/Colors:** Satin Nickel, Polished Brass, Oil Rubbed Bronze, etc.
- **Orientation:** Left-handed vs. Right-handed doors
- **Position:** Top mount vs. Bottom mount
- **Sizes:** Different key blank profiles, lock cylinder lengths

### 10.2. Variant Implementation Strategy

**Option A: Separate SKUs (Recommended for Phase 1)**

Each variant is a completely separate product with its own SKU:

```
Base Product: Kwikset Residential Deadbolt

Variants (separate products):
├── LOCK-RES-KW-001-SN  (Satin Nickel)
├── LOCK-RES-KW-001-PB  (Polished Brass)
├── LOCK-RES-KW-001-ORB (Oil Rubbed Bronze)
└── LOCK-RES-KW-001-MB  (Matte Black)

Each variant has:
- Own SKU
- Own barcode
- Own stock levels
- Own pricing (may differ by finish)
- Shared photos (or variant-specific photos)
```

**Advantages:**
- ✅ Simple inventory tracking (each variant tracked separately)
- ✅ Easy barcode scanning (one barcode = one specific variant)
- ✅ Supports different pricing per variant
- ✅ Works with existing Bitrix catalog structure

**Disadvantages:**
- ❌ More SKUs to manage
- ❌ Harder to see "total stock of all variants"

**Option B: Parent-Child Variant System (Future Enhancement)**

One parent product with child variants:

```
Parent: Kwikset Residential Deadbolt (LOCK-RES-KW-001)
├── Variant: Finish
│   ├── Satin Nickel
│   ├── Polished Brass
│   └── Oil Rubbed Bronze
└── Variant: Orientation
    ├── Left-handed
    └── Right-handed

Stock tracked at variant level
Pricing can be shared or per-variant
```

**Implementation for Phase 1: Separate SKUs**

## 11. Automated Price Updates

### 11.1. Scheduled Price Update Process

**Scenario:** Suppliers send updated price lists (UHS, Amazon Business, AliExpress exports)

**Frequency:** Weekly (configurable)
**Schedule:** Sunday 2:00 AM EST (off-hours)

**Workflow:**

```
Step 1: FTP/Email Monitor
  System checks designated folder for new price files
  Supported sources:
  - /upload/price_updates/uhs/
  - /upload/price_updates/amazon/
  - /upload/price_updates/aliexpress/
  ↓
Step 2: Automatic File Processing
  For each new file:
  - Detect format (Excel/CSV)
  - Parse and extract SKU + Price columns
  - Match to existing products by SKU or Supplier SKU
  ↓
Step 3: Generate Comparison Report
  System compares:
  - Current Cost vs. New Cost
  - Calculates impact on margins
  - Identifies significant changes (>15% change)
  ↓
Step 4: Apply Updates (Automatic)
  Updates are applied automatically
  No approval required (notification only)
  ↓
Step 5: Generate & Send Report
  Email report sent to:
  - Logistics Manager
  - Owner
  - Manager

  Report contains:
  - Summary of changes
  - List of significant price increases
  - Total cost impact
  - Margin analysis
```

### 11.4. Technical Implementation

```php
// File: /local/php_interface/include/scheduled_price_update.php

namespace Local\Inventory;

use Bitrix\Main\Mail\Event;

class ScheduledPriceUpdater {

    public static function run() {
        $startTime = microtime(true);
        $results = [];

        // Process each supplier folder
        $suppliers = self::getConfiguredSuppliers();

        foreach ($suppliers as $supplier) {
            if (!$supplier['auto_import']) continue;

            $files = self::getNewPriceFiles($supplier['folder']);

            foreach ($files as $file) {
                $result = self::processPriceFile($file, $supplier['name']);
                $results[] = $result;

                // Archive processed file
                self::archiveFile($file);
            }
        }

        // Generate and send report
        $report = self::generateReport($results);
        self::sendReport($report);

        $duration = microtime(true) - $startTime;
        self::logExecution($duration, $results);

        return $report;
    }

    private static function processPriceFile($filePath, $supplierName) {
        $extension = pathinfo($filePath, PATHINFO_EXTENSION);

        // Parse file based on format
        if (in_array($extension, ['xlsx', 'xls'])) {
            $data = self::parseExcel($filePath);
        } elseif ($extension === 'csv') {
            $data = self::parseCSV($filePath);
        } else {
            throw new \Exception("Unsupported file format: {$extension}");
        }

        $changes = [
            'increases' => [],
            'decreases' => [],
            'no_change' => [],
            'not_found' => []
        ];

        foreach ($data as $row) {
            $sku = $row['sku'] ?? $row['item_code'] ?? null;
            $newCost = floatval($row['cost'] ?? $row['price'] ?? 0);

            if (!$sku || !$newCost) continue;

            // Find product by SKU or Supplier SKU
            $product = self::findProduct($sku, $supplierName);

            if (!$product) {
                $changes['not_found'][] = ['sku' => $sku];
                continue;
            }

            $oldCost = floatval($product['UF_COST']);

            if ($newCost === $oldCost) {
                $changes['no_change'][] = [
                    'sku' => $sku,
                    'cost' => $oldCost
                ];
                continue;
            }

            // Update cost
            CIBlockElement::SetPropertyValuesEx(
                $product['ID'],
                false,
                ['UF_COST' => $newCost]
            );

            // Log price history
            self::logPriceChange(
                $product['ID'],
                'COST',
                $oldCost,
                $newCost,
                'automated_supplier_update',
                $supplierName
            );

            // Categorize change
            $changeData = [
                'sku' => $sku,
                'name' => $product['NAME'],
                'old_cost' => $oldCost,
                'new_cost' => $newCost,
                'change' => $newCost - $oldCost,
                'change_percent' => (($newCost - $oldCost) / $oldCost) * 100,
                'current_price' => floatval($product['CATALOG_PRICE_1']),
                'margin_old' => self::calculateMargin($oldCost, $product['CATALOG_PRICE_1']),
                'margin_new' => self::calculateMargin($newCost, $product['CATALOG_PRICE_1'])
            ];

            if ($newCost > $oldCost) {
                $changes['increases'][] = $changeData;
            } else {
                $changes['decreases'][] = $changeData;
            }
        }

        return [
            'supplier' => $supplierName,
            'file' => basename($filePath),
            'changes' => $changes
        ];
    }

    private static function generateReport($results) {
        // Aggregate all changes
        $totalIncreases = 0;
        $totalDecreases = 0;
        $totalNoChange = 0;
        $significantChanges = [];
        $netCostImpact = 0;

        foreach ($results as $result) {
            $totalIncreases += count($result['changes']['increases']);
            $totalDecreases += count($result['changes']['decreases']);
            $totalNoChange += count($result['changes']['no_change']);

            // Find significant changes (>15%)
            foreach ($result['changes']['increases'] as $change) {
                $netCostImpact += $change['change'];

                if (abs($change['change_percent']) > 15) {
                    $significantChanges[] = $change;
                }
            }

            foreach ($result['changes']['decreases'] as $change) {
                $netCostImpact += $change['change'];
            }
        }

        return [
            'summary' => [
                'total_updated' => $totalIncreases + $totalDecreases,
                'increases' => $totalIncreases,
                'decreases' => $totalDecreases,
                'no_change' => $totalNoChange,
                'net_cost_impact' => $netCostImpact
            ],
            'significant_changes' => $significantChanges,
            'supplier_breakdown' => $results,
            'generated_at' => date('Y-m-d H:i:s')
        ];
    }

    private static function sendReport($report) {
        $recipients = self::getReportRecipients();

        $emailBody = self::formatReportEmail($report);

        Event::send([
            'EVENT_NAME' => 'PRICE_UPDATE_REPORT',
            'LID' => 's1',
            'C_FIELDS' => [
                'EMAIL_TO' => implode(',', $recipients),
                'REPORT_BODY' => $emailBody,
                'DATE' => date('M j, Y')
            ]
        ]);
    }

    private static function calculateMargin($cost, $price) {
        if ($cost == 0) return 0;
        return (($price - $cost) / $cost) * 100;
    }
}

// Schedule: Add to cron or Bitrix agent

    "\\Local\\Inventory\\ScheduledPriceUpdater::run();",
    "catalog",
    "N",
    604800,  // Weekly (7 days in seconds)
    "",
    "Y",
    "2025-11-05 02:00:00"  // Start date
);
```
